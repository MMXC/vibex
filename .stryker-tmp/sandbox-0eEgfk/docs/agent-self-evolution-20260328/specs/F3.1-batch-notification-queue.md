# Spec: F3.1 — 批量心跳通知机制

## 设计目标
减少 Slack/Feishu 通知碎片化，累积 N 个完成事件后发送一条批量摘要。

## 配置
```bash
BATCH_SIZE=5        # 触发批量发送的阈值
BATCH_TIMEOUT=60    # 超时强制发送（秒）
```

## 队列数据结构
```python
# 内存队列（进程内）
queue = []  # List[{"project": str, "stage": str, "agent": str, "time": str}]

# 或文件队列（跨进程）
QUEUE_FILE="/tmp/hb-notify-queue.json"
```

## 触发条件
1. 队列长度 ≥ `BATCH_SIZE` → 立即发送
2. 队列非空且距上次发送 ≥ `BATCH_TIMEOUT` → 定时发送
3. 进程退出时 → 发送剩余队列

## 摘要格式
```
[PM] 🔔 批量完成报告 (N Epic)

✅ 完成项目:
• <project1>/<stage1> — by <agent> — <time>
• <project2>/<stage2> — by <agent> — <time>
...

⏱️ 批量耗时: <N> min
```

## 验证
```bash
# 模拟 5 个完成事件
for i in {1..5}; do
  python3 -c "from notify import queue_event; queue_event('p$i','s$i','dev')"
done

# 验证队列自动发送
# 检查 Slack 消息数量 = 1（而非 5）
```
