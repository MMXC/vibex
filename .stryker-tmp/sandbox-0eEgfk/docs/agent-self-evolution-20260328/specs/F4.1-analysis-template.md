# Spec: F4.1 — analysis.md 模板标准化

## 模板结构（6 个必填节）

```markdown
# Analysis: <项目名>

**Agent**: <agent> | **Date**: <YYYY-MM-DD> | **Status**: <status>

---

## 1. 问题定义
- **当前状态**: ...
- **目标状态**: ...
- **根因**: ...

## 2. 业务场景
<具体描述使用场景>

## 3. JTBD 分析（3-5 条）
| # | As a... | I want to... | So that... |
|---|---------|--------------|------------|
| 1 | ... | ... | ... |

## 4. 技术方案对比（≥2）
### 方案A: <名称>
- 优点: ...
- 缺点: ...
- 复杂度: P1/P2/P3

### 方案B: <名称> (推荐)
- 优点: ...
- 缺点: ...
- 复杂度: P1/P2/P3

## 5. 验收标准（≥4 条）
- [ ] <具体可测试的验收条件>
- [ ] ...

## 6. 风险识别
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| ... | ... | ... | ... |
```

## 放置位置
```
/root/.openclaw/skills/team-tasks/templates/analysis.md.template
```

## 验证脚本
```bash
#!/bin/bash
# 验证所有 analysis.md 合规性
TEMPLATE_KEYS=("问题定义" "业务场景" "JTBD" "技术方案对比" "验收标准" "风险识别")
for f in $(find docs/ -name 'analysis.md'); do
  missing=0
  for key in "${TEMPLATE_KEYS[@]}"; do
    grep -q "$key" "$f" || ((missing++))
  done
  if [ $missing -gt 0 ]; then
    echo "❌ $f 缺少 $missing 个必填节"
  else
    echo "✅ $f 合规"
  fi
done
```
