# 首页骨架屏重构需求分析

**项目**: vibex-homepage-skeleton-redesign
**日期**: 2026-03-14
**状态**: 需求确认中

---

## 1. 用户核心需求

### 需求 A: 移除重复的诊断组件

**用户原话**: "智能需求诊断 这个重复的组件又显示了，需要移除"

**问题**: 诊断组件在移除后又重新出现

---

### 需求 B: 骨架屏固定布局

**用户原话**: "页面布局方面可以考虑骨架屏固定 layout"

**核心概念**:
- 骨架屏固定区域（不可变）
- 区域内组件随业务流程可变化
- 骨架上操作是固定的

**固定骨架结构**:
```
┌─────────────────────────────────────────────────────────────┐
│                     HEADER (固定)                           │
├─────────────────────────────────────────────────────────────┤
│                                              │              │
│    主工作区域 (可拖拽调整)                    │   右侧对话    │
│    ┌────────────────┬────────────────┐      │   (固定)     │
│    │   预览区域      │   需求录入     │      │              │
│    │   (可调整)      │   (可调整)     │      │              │
│    └────────────────┴────────────────┘      │              │
│                                              │              │
├─────────────────────────────────────────────────────────────┤
│  主录入区域操作栏 (固定): 诊断 | 优化 | 清空 | 历史记录 | 示例 │
└─────────────────────────────────────────────────────────────┘
```

---

### 需求 C: 可拖拽互换区域

**用户原话**: "需求录入和实时预览区域可以拖动互换区域"

**功能描述**:
- 预览和录入区域可以拖拽调整大小
- 可以互换位置
- 使用 react-resizable-panels 实现
