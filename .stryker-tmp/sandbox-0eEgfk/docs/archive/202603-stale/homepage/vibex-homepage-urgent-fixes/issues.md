# 首页紧急问题清单

## 用户反馈 (2026-03-14 23:02)

### Bug 类型问题

#### 1. 领域模型生成报错 🔴 严重
```
ErrorBoundary caught an error: TypeError: Cannot read properties of undefined (reading 'length')
```
- **位置**: 首页 → AI分析 → 领域模型生成
- **影响**: 用户无法继续生成领域模型
- **疑似原因**: 某个数组/对象为 undefined

#### 4. 首页加载 React Error #418 🔴 严重
```
Uncaught Error: Minified React error #418
```
- **React Error #418**: Hydration 错误 - 服务端渲染与客户端不一致
- **位置**: 首页加载时
- **影响**: 控制台报错，可能影响功能

#### 5. 面板最大化最小化无效 🔴 严重
- **位置**: 各 panel 区域
- **影响**: 刚实现的面板功能不工作
- **可能原因**: 事件未绑定或状态未更新

#### 7. 示例点击未填入输入框 🟡 中等
- **位置**: "试试这些示例"
- **预期**: 点击示例后，文本应填入"需求录入"输入框
- **实际**: 只在实时预览显示，未填入输入框

### UX 改进问题

#### 2. 按钮交互优化 🟡 中等
- **当前问题**: 点击按钮无反馈
- **期望行为**:
  - 生成中: "继续生成"按钮显示 loading 动画
  - 其他按钮: 点击后提示"生成中，请先停止当前分析"

#### 3. 分析进度条缺失 🟡 中等
- **位置**: AI 分析面板上方
- **期望**: 显示分析进度，防止用户不知道进度

#### 6. Panel 区域大小调整 🟢 低优先级
- **当前问题**: 预设 panel 浪费空间
- **期望**: 加大默认面积 + 允许鼠标拖拽调整边框

### 功能增强问题

#### 8. 流程勾选上下文传递 🔴 重要
- **问题描述**: 用户勾选的内容未传递到下一步分析
- **影响**: 分析结果不符合用户实际需求
- **期望**: 每一步的勾选内容必须包含在下一步的上下文中

#### 9. 步骤切换与工作区恢复 🔴 重要
- **问题描述**: 
  - 已完成步骤不能上下切换
  - 点击到之前步骤后不能恢复工作区
- **影响**: 用户无法查看/修改之前步骤的结果
- **期望**: 
  - 允许切换到任意已完成步骤
  - 切换后工作区显示对应步骤的内容

## 分析要求

1. **定位根因**: 找出每个 Bug 的具体代码位置和原因
2. **评估影响**: 每个问题对用户体验的影响程度
3. **优先级排序**: Bug > UX > 增强
4. **验收标准**: 每个问题必须有可测试的验收标准
