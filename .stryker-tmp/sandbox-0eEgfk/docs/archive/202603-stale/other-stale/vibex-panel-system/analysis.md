# 面板系统需求分析

**项目**: vibex-panel-system
**日期**: 2026-03-14
**用户原话**: "骨架屏做的不好，需要类似其它编辑器panel一样，每个区域都可以调整大小宽高，或者全屏，小窗"

---

## 🚨 红线约束（强制）

| 约束 | 说明 |
|------|------|
| **已有功能不要动** | 不修改现有组件的功能逻辑 |
| **已有集成不要动** | 不修改已有的页面集成代码 |
| **删除的东西不要再加进来** | 不要恢复已删除的组件（如 DiagnosisPanel） |

---

## 1. 问题分析

### 当前骨架屏问题
- 只支持水平拖拽调整（预览/录入区域）
- 不支持垂直调整
- 不支持面板最大化
- 不支持面板最小化/小窗
- 用户体验不如 VS Code/Figma 等编辑器

### 目标效果（类似 VS Code）
```
┌─────────────────────────────────────────────────────────────┐
│  [📁 文件] [🔍 搜索] [📝 编辑] [▶ 运行] [🔧 设置]        │
├────┬──────────────────────────────┬────────────────────────┤
│    │                              │                        │
│ 📁 │      主编辑区               │   🔍 面板               │
│ 文 │      (可调/最大化)          │   (可调/最小化)        │
│ 件 │                              │                        │
│ 树 │                              │   [□] [—] [✕]         │
│    │                              │                        │
│    ├──────────────────────────────┤                        │
│    │      终端/输出               │                        │
│    │      (可调/最小化)          │                        │
└────┴──────────────────────────────┴────────────────────────┘
```

## 2. 功能需求

### F1: 面板可调整大小（宽高）
- 四个边可拖拽
- 四个角可拖拽
- 支持双向调整

### F2: 面板最大化
- 点击最大化按钮 → 面板占满整个容器
- 再次点击 → 恢复原大小

### F3: 面板最小化/折叠
- 点击最小化 → 面板折叠成标题栏
- 再次点击 → 展开面板

### F4: 面板浮动/小窗
- 点击浮动按钮 → 面板弹出到独立窗口
- 可拖拽移动位置
- 关闭时回到主窗口

### F5: 面板持久化
- 保存面板大小/位置到 localStorage
- 下次打开恢复布局

---

## 3. 技术方案

### 方案 A: react-resizable-panels + 自定义扩展
**优点**: 已有基础，扩展方便
**缺点**: 不支持浮动窗口

### 方案 B: allotment + 自定义面板系统
**优点**: 功能强大，支持多场景
**缺点**: 需要重新实现

### 方案 C: 自研面板系统
**优点**: 完全可控，可定制
**缺点**: 开发量大

**推荐**: 方案 A，基于现有代码扩展

## 4. 验收标准

| ID | 功能 | 验收标准 |
|----|------|----------|
| F1 | 调整大小 | expect(panel.style.width).toBeDefined() |
| F2 | 最大化 | expect(panel.classList.contains('maximized')).toBe(true) |
| F3 | 最小化 | expect(panel.classList.contains('minimized')).toBe(true) |
| F4 | 浮动窗口 | expect(floatingWindow).toBeInTheDocument() |
| F5 | 持久化 | expect(localStorage.getItem('panel-state')).not.toBeNull() |
