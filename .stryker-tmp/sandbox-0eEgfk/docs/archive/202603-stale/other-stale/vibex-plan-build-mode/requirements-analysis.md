**项目**: vibex-plan-build-mode
**分析师**: Analyst Agent

## 执行摘要

当前 VibeX 采用**一键生成模式**，用户输入需求后直接进入生成流程，对复杂需求处理准确率较低。引入 **Plan/Build 双模式**后，用户可在规划模式和构建模式间切换，预期提升复杂需求处理准确率 **25%**。

## 3. Plan/Build 双模式设计

### 3.1 模式定义

**Plan 模式**:
- AI 分析需求，展示理解
- 识别关键实体、功能点、潜在问题
- 用户可调整 AI 理解
- 生成规划文档供确认

**Build 模式**:
- 基于 Plan 结果执行生成
- 逐步生成 DDD 各阶段产出
- 支持中途调整

## 4. 功能需求拆分

### 4.1 需求列表

| ID | 需求 | 描述 | 优先级 | 工作量 |
|----|------|------|--------|--------|
| **Plan 模式** |
| P1 | Plan 按钮 | 添加 Plan 按钮到输入区 | P0 | 0.5天 |
| P2 | AI 理解分析 | 分析需求并生成理解结果 | P0 | 2天 |
| P3 | 理解结果展示 | 展示实体、功能点、问题 | P0 | 1天 |
| P4 | 用户调整功能 | 允许用户修改 AI 理解 | P1 | 1天 |
| P5 | 确认/取消 | 确认或取消 Plan 结果 | P0 | 0.5天 |
| **Build 模式** |
| B1 | Build 按钮 | 添加 Build 按钮到输入区 | P0 | 0.5天 |
| B2 | 直接生成逻辑 | 跳过 Plan 直接生成 | P0 | 0.5天 |
| B3 | 从 Plan 构建 | 基于 Plan 结果生成 | P0 | 1天 |
| **状态管理** |
| S1 | 模式状态 | 管理 Plan/Build 状态 | P0 | 0.5天 |
| S2 | Plan 结果缓存 | 缓存 Plan 结果供 Build 使用 | P1 | 0.5天 |

| 模块 | P0 工作量 | P1 工作量 | 总计 |
|------|-----------|-----------|------|

## 6. 技术方案

### 6.1 API 设计

```typescript
// Plan API
POST /api/plan/analyze
Request: { requirement: string }
Response: {
  entities: Entity[];
  features: Feature[];
  questions: Question[];
  suggestedContexts: BoundedContext[];
}

// Build API (现有)
POST /api/ddd/generate
Request: { requirement: string, planResult?: PlanResult }
Response: { boundedContexts: BoundedContext[], ... }
```

### 6.2 状态管理

```typescript
interface PlanBuildState {
  mode: 'plan' | 'build' | null;
  planResult: PlanResult | null;
  isPlanning: boolean;
  isBuilding: boolean;
}

// Zustand Store
const usePlanBuildStore = create<PlanBuildState>((set) => ({
  mode: null,
  planResult: null,
  isPlanning: false,
  isBuilding: false,
  
  startPlan: () => set({ mode: 'plan', isPlanning: true }),
  finishPlan: (result) => set({ planResult: result, isPlanning: false }),
  startBuild: () => set({ mode: 'build', isBuilding: true }),
}));
```
