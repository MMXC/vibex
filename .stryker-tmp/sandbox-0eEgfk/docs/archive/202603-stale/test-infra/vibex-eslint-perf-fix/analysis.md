## 2. 根因分析

### 2.1 lint 命令构成

```bash
# package.json
eslint . --ext .js,.jsx,.ts,.tsx,.mjs,.cjs \
  --ignore-pattern '**/*.yml' \
  --ignore-pattern '**/*.yaml' \
  --ignore-pattern '.github/**' \
  && prettier --check .
```

### 2.2 ESLint 配置分析

```javascript
// eslint.config.mjs
const eslintConfig = defineConfig([
  ...nextVitals,
  ...nextTs,
  globalIgnores([
    // ✅ 正常忽略
    '.next/**', 'node_modules/**', 'coverage/**',
    // ✅ 排除测试
    'tests/**',
    // ⚠️ 但 lint 命令的 --ignore-pattern 不完全生效
  ]),
  // ...
]);
```

### 2.3 性能瓶颈识别

| 瓶颈 | 详情 | 影响 |
|------|------|------|
| **tests/** 未被忽略 | ESLint 仍检查 `tests/e2e/*.spec.ts` | 约 20+ 文件被检查 |
| ESLint 未启用缓存 | 无 `--cache` 参数 | 每次全量扫描 |
| 缺少并行 | 单线程执行 | CPU 未充分利用 |
| Prettier 串行 | 在 ESLint 之后执行 | 叠加耗时 |

**证据**：
```
$ time npm run lint
[warn] tests/e2e/pages/RequirementsPage.ts  ← 不应检查
[warn] tests/e2e/preview-area-v3.spec.ts
... (20+ 个测试文件)
real    1m5.667s
```

## 3. 技术方案

### 3.1 方案 A：添加 --cache + 完善 ignore（推荐）

**修改 `package.json`**:

```json
{
  "scripts": {
    "lint": "eslint . --cache --cache-location node_modules/.cache/eslint/ --ext .js,.jsx,.ts,.tsx,.mjs,.cjs && prettier --check ."
  }
}
```

**修改 `eslint.config.mjs`** — 确保 `tests/**` 完全忽略：

```javascript
// eslint.config.mjs
globalIgnores([
  'tests/**',           // 确认存在
  // 新增明确排除
  'tests/e2e/**',
  'tests/unit/**',
]),
```

**或使用 CLI ignore 文件**:

```bash
# .eslintignore
tests/**
!tests/**/*.test.tsx
```

### 3.2 方案 B：分离 lint 任务

```json
{
  "scripts": {
    "lint:src": "eslint src --cache --cache-location node_modules/.cache/eslint/ --ext .ts,.tsx",
    "lint:check": "prettier --check .",
    "lint": "npm run lint:src && npm run lint:check"
  }
}
```

**优点**: 只检查 src/，排除所有非源码
**缺点**: CI 需要调整

### 3.3 方案 C：并行化（额外优化）

```json
{
  "scripts": {
    "lint": "npm-run-all --parallel lint:eslint lint:prettier",
    "lint:eslint": "eslint src --cache --ext .ts,.tsx",
    "lint:prettier": "prettier --check ."
  }
}
```

### 3.4 推荐组合

**最小改动方案**（推荐）：

```bash
# 只需修改 package.json 中的 lint 脚本
"lint": "eslint . --cache --cache-location node_modules/.cache/eslint/ --ext .js,.jsx,.ts,.tsx,.mjs,.cjs --ignore-pattern 'tests/**' && prettier --check ."
```

预期效果：
- `--cache` → 首次后 20-30s（增量）
- `--ignore-pattern 'tests/**'` → 跳过测试目录
- **总耗时**: 预期 25-35s（首次），10-15s（增量）

## 5. 风险与缓解

| 风险 | 等级 | 缓解 |
|------|------|------|
| `--ignore-pattern` 覆盖不足 | 🟡 中 | 配合 `globalIgnores` 双重保险 |
| CI cache 失效 | 🟢 低 | CI 使用 `npm ci` 后首次仍需时间 |
| Prettier 未缓存 | 🟢 低 | Prettier 本身很快（< 5s） |

## 7. 结论

**根因**: ESLint 未启用缓存 + tests/ 目录未被完全忽略

**推荐修复**:
1. 添加 `--cache --cache-location node_modules/.cache/eslint/`
2. 添加 `--ignore-pattern 'tests/**'`
3. 验证 `tests/e2e/**` 不再被检查

**预期**: 65s → 20-30s（首次），10-15s（增量）
