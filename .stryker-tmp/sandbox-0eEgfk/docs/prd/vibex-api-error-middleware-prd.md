# PRD: API 错误处理中间件

**项目名称**: vibex-api-error-middleware  
**版本**: 1.0  
**创建日期**: 2026-03-04  
**负责人**: PM Agent

---

## 1. 项目目标

统一前端 API 错误处理，减少重复代码 30%，建立统一的错误码映射和用户提示。

---

## 2. 问题分析

### 2.1 当前问题

| 问题 | 描述 | 影响 |
|------|------|------|
| 错误处理重复 | 每个 API 调用独立处理错误 | 代码冗余 |
| 错误提示不一致 | 不同接口错误提示格式不同 | 用户困惑 |
| 错误码未统一 | 后端返回不同格式错误码 | 难以维护 |

### 2.2 目标

- 统一错误处理逻辑
- 建立错误码映射表
- 一致用户体验

## 5. 技术方案

### 5.1 核心接口

```typescript
// types/error.ts
interface ErrorConfig {
  code: string;
  type: 'network' | 'business' | 'server' | 'client';
  message: string;
  userMessage: string;
  retryable: boolean;
}

// api/errorMiddleware.ts
interface ErrorMiddlewareOptions {
  showToast?: boolean;
  retryCount?: number;
  onError?: (error: ErrorConfig) => void;
  customMappings?: Record<string, ErrorConfig>;
}

function withErrorHandling<T>(
  fetchFn: () => Promise<T>,
  options?: ErrorMiddlewareOptions
): Promise<T>
```

### 5.2 使用方式

```typescript
// 方式 1: 包装现有 fetch
const result = await withErrorHandling(
  () => fetch('/api/projects')
)

// 方式 2: 装饰器模式
const api = createApiClient({
  baseUrl: '/api',
  errorHandling: true
})
```
