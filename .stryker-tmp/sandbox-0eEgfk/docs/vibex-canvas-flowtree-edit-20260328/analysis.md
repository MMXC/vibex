## 1. 问题定义

**需求**:
1. 支持新增流程及流程节点
2. 确保流程节点使用标准流程节点样式

**影响**: 当前流程树可能只读或样式不统一，用户无法添加新的流程/节点

---

## 2. 现状分析

### 2.1 功能对比

| 功能 | 当前状态 | 目标状态 |
|------|----------|----------|
| 新增流程 | ❌ 可能缺失 | ✅ 可添加 |
| 新增流程节点 | ❌ 可能缺失 | ✅ 可添加 |
| 节点样式 | ⚠️ 可能不标准 | ✅ 统一标准样式 |

### 2.2 问题定位

| 层级 | 文件/位置 | 说明 |
|------|-----------|------|
| 流程树组件 | `src/components/canvas/flow-tree/` | 树结构渲染 |
| 新增按钮 | 工具栏或右键菜单 | 触发新增操作 |
| 样式定义 | `flow-node.module.css` 或 Tailwind | 节点样式 |
| Store | `useDDDStore.flowData` | 流程数据结构 |

---

## 3. 解决方案

### 3.1 实现方案

**工作量**: ~2-3 小时

**步骤**:
1. **新增流程**: 在流程树工具栏添加「+ 新流程」按钮，绑定 `addFlow()` action
2. **新增节点**: 在流程节点上添加「+ 子节点」按钮或右键菜单
3. **样式统一**: 审查所有 flow-node 相关组件，确保使用统一样式类
4. **数据一致性**: 确保新增的流程/节点数据结构符合 store schema

### 3.2 关键代码（推测）

```typescript
// Store actions
interface FlowTreeStore {
  addFlow: (name: string) => void
  addFlowNode: (flowId: string, nodeData: NodeData) => void
}

// 样式标准
.flowNode {
  @apply rounded-lg border border-blue-300 bg-white px-3 py-2;
}
```
