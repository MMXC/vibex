## 业务场景分析

### VibeX 产品定位

VibeX 是 AI 驱动的原型设计工具，核心用户是产品经理和开发团队。核心流程：

```
需求输入 → 限界上下文 → 领域模型 → 业务流程 → 组件树 → 原型预览 → 导出
```

### 目标用户

| 用户类型 | 核心诉求 |
|----------|---------|
| 产品经理 | 快速原型设计、导出可读文档 |
| 开发团队 | 理解上下文边界、导出代码 |
| 团队协作 | 分享项目结构、无账号也能查看 |

### 当前体验问题

从 PM 提案识别出 6 项体验问题：
- 确认状态视觉区分不清
- 面板状态不持久化
- 导出流程无引导
- 空状态缺乏引导
- 移动端体验缺失
- 导出格式单一

## 技术方案选项

### 方案 A: 按依赖顺序逐步实施（推荐）

| Sprint | 提案 | 工时 | 依赖 | 用户价值 |
|--------|------|------|------|---------|
| Sprint 1 | P-001 + P-002 | 3h | D-E1, D-E2 | 确认状态可视化 + 面板记忆 |
| Sprint 2 | P-003 | 3h | D-003 | 导出向导 |
| Sprint 3 | P-004 | 2h | 无 | 空状态引导 |
| Sprint 4 | P-005 + P-006 | 7h | PWA | 移动端降级 + PRD导出 |

**优点**: 风险分散，每阶段有用户价值交付
**缺点**: 总工期较长（~15h）

### 方案 B: 集中实施

将 P-001~P-004 合并到一个 Sprint 集中开发。

**优点**: 团队专注，交付快
**缺点**: 风险集中，依赖关系复杂

## 验收标准

### P-001 验收标准
- [ ] 未确认节点：黄色虚线边框
- [ ] 已确认节点：绿色实线边框
- [ ] 工具栏有确认状态筛选快捷操作
- [ ] 导出时弹窗提示"未确认节点将被忽略"

### P-002 验收标准
- [ ] 展开/折叠面板 → 刷新页面 → 状态保持
- [ ] localStorage 中有 `vibex-panel-collapsed-*` key
- [ ] 首次访问默认全部展开

### P-003 验收标准
- [ ] 导出按钮点击 → 向导 Step 1/2/3
- [ ] 必填项有红色星号标识
- [ ] 导出过程有进度条
- [ ] 成功/失败有明确反馈

### P-004 验收标准
- [ ] Canvas 空画布显示引导卡片
- [ ] 引导卡片包含快捷操作按钮
- [ ] 有历史数据时引导卡片隐藏

### P-005 验收标准
- [ ] 移动端访问 Canvas → 显示友好降级提示
- [ ] 提供"查看只读预览"入口
- [ ] 提示使用桌面浏览器

### P-006 验收标准
- [ ] 导出选项有"Markdown 文档"格式
- [ ] 导出内容包含：限界上下文 + 流程说明 + 组件清单
- [ ] Markdown 可直接粘贴到飞书文档
