## 3. 技术方案

### 3.1 方案 A：添加到 dependencies（推荐）

```bash
pnpm add zustand@4.5.7
```

**修改后 package.json**:
```json
{
  "dependencies": {
    "zustand": "^4.5.7",
    // ... 其他依赖
  }
}
```

**优点**：
- 明确声明依赖，符合最佳实践
- 版本由我们控制，不被 reactflow 绑架
- CI/CD `npm ci` / `pnpm install` 均可正常工作

**缺点**：
- 多一个明确的依赖声明（实际无副作用）

**工作量**：1 分钟

---

### 3.2 方案 B：添加到 devDependencies（不推荐）

```bash
pnpm add -D zustand@4.5.7
```

**问题**：`zustand` 是运行时依赖，不是开发依赖，应在 `dependencies`。

---

### 3.3 方案 C：什么都不做（不推荐）

**理由**：当前构建成功，依赖传递正常工作。

**风险**：
- reactflow 未来可能升级/降级 zustand 版本
- CI/CD 严格模式可能失败
- 代码审查时会被标记为问题
