# Spec: E3-S3.1 — ActionBar 按钮逻辑实现

**Epic**: Epic 3 首页交互功能补全  
**Story**: ActionBar 按钮逻辑实现  
**Agent**: dev  
**状态**: Ready

---

## 概述

实现首页 ActionBar 的 7 个按钮的实际业务逻辑，替换现有的空函数实现。

---

## 功能描述

### 按钮列表
1. **新建分析** — 打开新的 DDD 分析流程
2. **导入文档** — 触发文档上传
3. **导出结果** — 导出分析结果为 JSON/MD
4. **分享** — 生成分享链接
5. **收藏** — 收藏当前分析结果
6. **设置** — 打开设置面板
7. **帮助** — 打开帮助文档

### 实现要求
- 每个按钮有明确的 click handler
- 点击后有视觉反馈（loading/error 状态）
- 错误情况有 toast 提示
- 使用 useHomeGeneration hook 统一处理

---

## 验收标准

| ID | Given | When | Then |
|----|-------|------|------|
| AC1 | 点击新建 | 无分析进行中 | 打开新流程，加载动画 |
| AC2 | 点击导入 | 文档拖拽完成 | 文件处理，显示结果 |
| AC3 | 点击导出 | 分析已完成 | 文件下载触发 |
| AC4 | 点击分享 | 分析已完成 | 分享弹窗，显示链接 |
| AC5 | 点击收藏 | 已登录 | 收藏成功 toast |
| AC6 | 点击设置 | 任意状态 | 设置面板打开 |
| AC7 | 点击帮助 | 任意状态 | 帮助文档打开 |
| AC8 | 网络错误 | API 调用失败 | error toast 显示 |
