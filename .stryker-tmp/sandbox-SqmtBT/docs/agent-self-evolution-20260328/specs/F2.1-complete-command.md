# Spec: F2.1 — task_manager.py complete 子命令

## 问题描述
`heartbeat.sh` 调用 `task_manager.py complete` 但该子命令不存在。
实际需要 `task_manager.py update project stage status` 三参数。

## 修复方案
在 `task_manager.py` 中添加 `complete` 子命令：
```python
# subparsers.add_parser('complete')
# args: project, stage, result (result 写入 logs)
```

## 接口设计
```
task_manager.py complete <project> <stage> <result>
```

## 状态变更
- `status`: `in-progress` → `done`
- `completedAt`: 设为当前 ISO 时间戳
- `logs` 追加: `{event: "status: in-progress → done (by <agent>)"}`

## 验证
```bash
# 创建测试任务
python3 task_manager.py add test-project test-stage "test" --agent dev

# 完成测试任务
python3 task_manager.py complete test-project test-stage "测试完成"

# 验证状态
python3 -c "
import json
with open('test-project.json') as f:
    t = json.load(f)
    stage = t['stages']['test-stage']
    assert stage['status'] == 'done', f'Expected done, got {stage[\"status\"]}'
    assert stage['completedAt'] is not None
    print('✅ F2.1 验证通过')
"
```
