## 执行摘要

**一句话结论**: 首页定位为 VibeX 核心入口，通过 AI 驱动的四步流程（需求录入→需求澄清→业务流程→组件图）实现从需求到可视化设计的无缝体验。

**关键指标**:
- 功能点: 32 个 (P0: 19, P1: 10, P2: 3)
- 布局规格: 1400px×820px，三栏+底部面板
- 核心依赖: Mermaid.js, React 18, Zustand, SSE
- 预估工作量: 22 人日 (P0), 8 人日 (P1), 3 人日 (P2)

## 1. 业务场景分析

### 1.1 产品定位

VibeX 是一个 **DDD (领域驱动设计) AI 分析工具**，核心价值在于将复杂的领域建模流程可视化、智能化，降低技术门槛。

### 1.2 用户画像与痛点

| 用户类型 | 特征 | 核心痛点 | 首页期望 |
|----------|------|----------|----------|
| 产品经理 | 熟悉业务，不懂技术 | 难以清晰表达系统边界 | 快速上手，直观可视化 |
| 开发工程师 | 懂技术，业务理解浅 | 领域模型设计困难 | AI 辅助建模，减少沟通成本 |
| 架构师 | 需要快速验证想法 | 缺乏高效工具 | 探索式设计，随时切换步骤 |
| 初创团队 | 资源有限 | 没有专人做设计 | 一站式完成，降低学习成本 |

### 1.3 核心用户故事

| ID | 用户故事 | 验收条件 |
|----|----------|----------|
| US-001 | 作为产品经理，我希望输入需求后立即看到领域模型 | 输入→10s内看到结果 |
| US-002 | 作为架构师，我希望能够随时切换分析步骤 | 步骤切换<500ms，支持回退 |
| US-003 | 作为开发工程师，我希望 AI 能够追问澄清 | 澄清问题有明确选项 |
| US-004 | 作为用户，我希望将分析结果一键创建为项目 | 创建成功率>99% |

### 1.4 典型使用场景

**场景1: 快速启动** (30秒完成)
```
用户打开首页 → 输入"电商系统" → 看到限界上下文 → 点击创建项目
```

**场景2: 深度分析** (5分钟)
```
输入详细需求 → 展开底部面板 → 使用AI询问追问 → 根据建议优化 → 重新生成 → 创建项目
```

**场景3: 探索设计** (3分钟)
```
输入需求 → 查看限界上下文 → 切换到"业务流程" → 查看流程图 → 切换到"组件图" → 创建项目
```

## 3. 技术方案选项

### 方案A: 完全重构 (推荐)

**思路**: 基于现有代码重构，替换布局和交互逻辑

**优点**:
- 代码可控，可复用现有组件
- 迭代速度快，可分阶段上线
- 风险可控，出问题可快速回滚

**缺点**:
- 需要处理遗留代码兼容
- 部分逻辑可能需要重写

**实现路径**:
```
Phase 1 (P0): 布局重构 → 状态管理 → 基础预览
Phase 2 (P1): SSE集成 → 快捷功能 → AI展示区
Phase 3 (P2): 悬浮模式 → 图表交互
```

### 方案B: 全新开发

**思路**: 从零开始新建页面，保留旧版作为回退

**优点**:
- 架构干净，无历史包袱
- 可并行开发，不影响现有功能

**缺点**:
- 工作量大，需要处理路由切换
- 双维护成本高

**实现路径**:
```
Phase 1: 新页面开发 (独立路由 /v2)
Phase 2: 灰度测试
Phase 3: 切换路由，废弃旧版
```

## 4. 可行性评估

### 4.1 技术可行性

| 技术点 | 可行性 | 说明 |
|--------|--------|------|
| Grid布局 | ✅ 成熟 | CSS Grid 浏览器支持率>95% |
| 拖动调整高度 | ✅ 成熟 | 使用 React DnD 或原生 drag API |
| SSE流式 | ✅ 成熟 | 后端已实现，前端需接入 |
| Mermaid渲染 | ⚠️ 需优化 | 大图可能有性能问题，需懒加载 |
| 状态管理 | ✅ 成熟 | Zustand 已使用，扩展即可 |
| 移动端适配 | ⚠️ 需设计 | 交互模式需单独设计 |

### 4.2 依赖分析

```
前置依赖:
├── 后端 API /api/v1/analyze/stream (SSE) ──┐
├── Mermaid.js v10+                         ├── 必须
└── 状态管理 Zustand ────────────────────────┘

可选依赖:
├── React DnD (拖拽) ──── 可用原生 API 替代
└── 图表导出库 ────────── 可用 html2canvas
```

### 4.3 风险识别

| 风险ID | 风险描述 | 概率 | 影响 | 等级 | 缓解措施 |
|--------|----------|------|------|------|----------|
| R1 | Mermaid渲染大图性能差 | 高 | 中 | P1 | 懒加载+分页+虚拟化 |
| R2 | SSE连接不稳定 | 中 | 高 | P0 | 自动重连3次+本地缓存 |
| R3 | 移动端适配复杂 | 中 | 中 | P2 | 响应式设计+手势优化 |
| R4 | 与现有代码冲突 | 中 | 中 | P1 | 代码审查+测试覆盖 |
| R5 | 动画流畅度差 | 低 | 低 | P3 | CSS硬件加速+requestAnimationFrame |

## 6. 实现方案 (每个需求)

### 6.1 F1 布局框架

| ID | 功能 | 实现方案 | 关键代码 |
|----|------|----------|----------|
| F1.1 | 页面容器 | max-width: 1400px, margin: 0 auto | `.page { max-width: 1400px; }` |
| F1.2 | 网格布局 | CSS Grid 3×3 | `grid-template-rows: 50px 1fr 380px; grid-template-columns: 220px 1fr 260px;` |
| F1.3 | 响应式 | Media Query | `@media (max-width: 1200px) { ... }` |
| F1.4 | 抽屉层叠 | z-index控制 | `.drawer { z-index: 10; } .bottom-panel { z-index: 5; }` |
| F1.5 | 主题变量 | CSS变量 | `:root { --color-primary: #3b82f6; }` |

### 6.2 F2 Header导航

| ID | 功能 | 实现方案 | 关键代码 |
|----|------|----------|----------|
| F2.1 | Logo | Link to="/", styled | `<Link to="/"><span className="logo">VibeX</span></Link>` |
| F2.2 | 导航链接 | React Router | `<Link to="/projects">项目</Link>` |
| F2.3 | 登录按钮 | 触发抽屉 | `onClick={() => setLoginDrawerOpen(true)}` |
| F2.4 | 登录状态 | Zustand authStore | `{user ? <Avatar /> : <LoginButton />}` |

### 6.3 F3 左侧抽屉

| ID | 功能 | 实现方案 | 关键代码 |
|----|------|----------|----------|
| F3.1 | 步骤列表 | map遍历steps数组 | `steps.map(step => <StepItem {...step} />)` |
| F3.2 | 步骤切换 | 更新currentStep | `setCurrentStep(step.id)` |
| F3.3 | 步骤状态 | 条件渲染样式 | `step.status === 'active' ? 'active' : ''` |

### 6.4 F4 预览区

| ID | 功能 | 实现方案 | 关键代码 |
|----|------|----------|----------|
| F4.1 | 空状态 | 条件渲染 | `{!loading && !data && <Placeholder />}` |
| F4.2 | 加载状态 | skeleton组件 | `<Skeleton active />` |
| F4.3 | Mermaid渲染 | mermaid组件 | `<Mermaid code={mermaidCode} />` |

### 6.5 F9 状态管理

| ID | 功能 | 实现方案 | 关键代码 |
|----|------|----------|----------|
| F9.1 | 持久化 | Zustand persist | `persist({ storage: localStorage })` |
