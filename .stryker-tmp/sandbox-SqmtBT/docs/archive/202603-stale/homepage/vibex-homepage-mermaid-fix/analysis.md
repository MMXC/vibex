## 3. 问题定位

### 3.1 可能原因分析

| 可能原因 | 症状 | 检查方法 |
|----------|------|----------|
| Mermaid 代码未生成 | flowMermaidCode 为空 | 检查 store 状态 |
| MermaidManager 未初始化 | 渲染失败 | 检查 console 错误 |
| 预览区未订阅正确状态 | 显示旧数据 | 检查 PreviewArea props |
| 流式输出未完成 | 页面已渲染但内容为空 | 检查 loading 状态 |
| 组件卸载时渲染 | 异步渲染失败 | 检查 Suspense 边界 |

### 3.2 关键代码检查

**StepBusinessFlow 关键逻辑**:

```typescript
// StepBusinessFlow.tsx
const { mermaidCode } = useBusinessFlowStream();

// handleNext - 导航时保存
if (flow.states && flow.states.length > 0) {
  setBusinessFlow(flow);
  if (mermaidCode) {
    setFlowMermaidCode(mermaidCode);  // ← 保存到 store
  }
}
```

**潜在问题**：
1. `mermaidCode` 在 `handleGenerate` 时可能未返回
2. `handleNext` 是手动导航，用户可能未点击"下一步"就返回首页
3. 首页预览区可能订阅的是错误的 store 路径
