## 1. 功能需求 (Functional Requirements)

### 1.1 三步确认流程

#### 1.1.1 Step 1: 限界上下文图确认

**功能描述**: 用户输入需求后，AI 生成限界上下文图（Bounded Context Diagram），用户确认或修改。

**功能点**:
| ID | 功能点 | 优先级 | 描述 |
|----|--------|--------|------|
| F1.1.1 | 需求输入界面 | P0 | 提供文本输入框，用户描述产品需求 |
| F1.1.2 | AI 生成限界上下文 | P0 | 调用 AI 分析需求，生成 Mermaid 格式的限界上下文图 |
| F1.1.3 | 图表渲染 | P0 | 使用 Mermaid.js 渲染限界上下文图 |
| F1.1.4 | 上下文标注 | P1 | 标注核心上下文、支撑上下文、通用上下文 |
| F1.1.5 | 确认/修改操作 | P0 | 提供"确认继续"和"返回修改"按钮 |

**交互流程**:
```
用户输入需求 → AI生成限界上下文图 → 展示图表 → 用户确认/修改 → 进入Step 2
```

#### 1.1.2 Step 2: 领域模型类图确认

**功能描述**: 基于用户在 Step 1 选定的核心上下文，AI 生成领域模型类图（Domain Model Class Diagram），用户确认或修改。

**功能点**:
| ID | 功能点 | 优先级 | 描述 |
|----|--------|--------|------|
| F1.2.1 | 上下文选择 | P0 | 从 Step 1 的上下文列表中选择核心上下文 |
| F1.2.2 | AI 生成领域模型 | P0 | 基于选定上下文，生成聚合根、实体、值对象 |
| F1.2.3 | 类图渲染 | P0 | 使用 Mermaid classDiagram 语法渲染 |
| F1.2.4 | 聚合标注 | P1 | 区分聚合根、普通实体、值对象 |
| F1.2.5 | 属性展示 | P1 | 展示类属性和方法 |
| F1.2.6 | 确认/返回操作 | P0 | "返回Step 1" 和 "确认继续" 按钮 |

**数据依赖**:
- 输入: Step 1 中选定的限界上下文
- 输出: 领域模型类图（Mermaid + 结构化数据）

#### 1.1.3 Step 3: 业务流程图确认

**功能描述**: 基于 Step 1+2 的领域模型，AI 生成业务流程图（Business Flow Diagram），用户确认后创建项目。

**功能点**:
| ID | 功能点 | 优先级 | 描述 |
|----|--------|--------|------|
| F1.3.1 | AI 生成业务流程 | P0 | 基于领域模型生成状态流转图 |
| F1.3.2 | 流程图渲染 | P0 | 使用 Mermaid stateDiagram 语法渲染 |
| F1.3.3 | 状态标注 | P1 | 标注初始状态、中间状态、最终状态 |
| F1.3.4 | 事件标注 | P1 | 标注触发状态变更的领域事件 |
| F1.3.5 | 返回/创建操作 | P0 | "返回Step 2" 和 "确认创建项目" 按钮 |
| F1.3.6 | 项目自动创建 | P0 | 三图数据保存 + 项目记录创建 |

**数据依赖**:
- 输入: Step 1 限界上下文 + Step 2 领域模型
- 输出: 业务流程图 + 项目创建成功

### 1.2 三种编辑方式

#### 1.2.1 图形化编辑

**功能描述**: 将 Mermaid 图表渲染为可交互的 React Flow 图，支持拖拽节点、添加/删除连线。

**功能点**:
| ID | 功能点 | 优先级 | 描述 |
|----|--------|--------|------|
| F1.2.1.1 | Mermaid → React Flow 转换 | P0 | 解析 Mermaid 代码，生成节点和边 |
| F1.2.1.2 | 节点拖拽 | P0 | 节点可自由移动位置 |
| F1.2.1.3 | 连线操作 | P0 | 支持添加、删除、修改连线 |
| F1.2.1.4 | 双向同步 | P0 | 图形化修改后自动同步到 Mermaid 代码 |
| F1.2.1.5 | 节点编辑 | P1 | 双击节点编辑属性（名称、类型等） |

**技术方案**:
- 使用 `mermaid.parse()` 解析 Mermaid 代码
- 转换为 React Flow 的 `Node[]` 和 `Edge[]` 格式
- 使用 Zustand 管理状态，实现双向绑定

#### 1.2.2 代码编辑

**功能描述**: 集成 Monaco Editor，用户直接编辑 Mermaid 代码，实时预览渲染效果。

**功能点**:
| ID | 功能点 | 优先级 | 描述 |
|----|--------|--------|------|
| F1.2.2.1 | Monaco Editor 集成 | P0 | 集成 VS Code 同款编辑器 |
| F1.2.2.2 | Mermaid 语法高亮 | P0 | 支持 Mermaid 语法着色 |
| F1.2.2.3 | 实时预览 | P0 | 编辑时同步渲染图表 |
| F1.2.2.4 | 语法错误提示 | P1 | 错误的 Mermaid 语法高亮显示 |
| F1.2.2.5 | 代码同步 | P0 | 代码变更同步到图形化视图 |

#### 1.2.3 AI 对话编辑

**功能描述**: 提供对话框，用户用自然语言描述修改意图，AI 解析并更新图表。

**功能点**:
| ID | 功能点 | 优先级 | 描述 |
|----|--------|--------|------|
| F1.2.3.1 | 对话输入框 | P0 | 底部固定对话框 |
| F1.2.3.2 | 自然语言解析 | P0 | AI 解析用户意图（添加/删除/修改节点） |
| F1.2.3.3 | 图表更新 | P0 | AI 生成新的 Mermaid 代码并渲染 |
| F1.2.3.4 | 多轮对话 | P1 | 保持上下文，支持连续修改 |
| F1.2.3.5 | 修改预览 | P2 | 显示修改前后的 diff |

**功能点**:

|----|--------|--------|------|

| F1.3.3 | 数据持久化 | P0 | 每步确认后自动保存到 Zustand |
| F1.3.4 | 重新生成提示 | P1 | 返回修改后提示是否重新生成下游步骤 |
| F1.3.5 | 进度保存 | P2 | 支持中途退出，下次继续 |
