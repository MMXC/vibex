## 4. 提案详情

### P001: 状态管理层模块化 (P0)

**问题描述**: `canvasStore.ts` 承担所有状态管理职责，代码超过 900 行，新增功能时难以定位和测试。

**根因分析**: 状态管理未按领域划分，所有节点类型共享同一个 store 实例。

**建议方案**:
```
src/lib/canvas/
├── stores/
│   ├── contextStore.ts    # 上下文节点状态
│   ├── flowStore.ts       # 流程节点状态
│   ├── componentStore.ts  # 组件节点状态
│   └── uiStore.ts        # UI 状态
└── canvasStore.ts         # 聚合层
```

**工时**: 12h（大型重构）

**验收标准**:
- [ ] 每个 store 单元测试覆盖率 > 80%
- [ ] 现有组件无需修改（向后兼容）
- [ ] 新增节点类型只需新增 store 文件

---

### P002: Canvas 虚拟化列表 (P1)

**问题描述**: 节点数量 > 100 时，ComponentTree 和 BusinessFlowTree 出现明显卡顿。

**建议方案**: 引入 `@tanstack/react-virtual` 实现虚拟化列表。

**工时**: 4h

**验收标准**:
- [ ] 100 节点渲染时间 < 100ms
- [ ] 滚动流畅度 > 50fps

---

### P003: TypeScript 严格模式升级 (P1)

**问题描述**: 当前 TypeScript 配置宽松，类型错误可能被掩盖。

**建议方案**: 逐步启用 `strict: true`，逐文件修复类型错误。

**工时**: 8h

**验收标准**:
- [ ] `tsc --strict` 编译通过
- [ ] 无新增 `@ts-ignore`
- [ ] CI 中包含类型检查
