# 首页核心布局调整需求分析

**项目**: vibex-homepage-core-layout
**日期**: 2026-03-14
**状态**: 紧急需求

---

## 1. 用户核心需求

### 需求 A: 预览/录入分离布局

**用户原话**: "实时预览和需求录入拆开，预览占中央区域的60%，需求录入占中央区域下方40%"

**现状问题**:
- 当前布局：三栏（Sidebar 15% + Content 60% + AI Panel 25%）
- Preview 在 Tab 切换中，非固定展示
- 需求录入在 Tab 内容区域

**目标布局**:
```
┌─────────────────────────────────────┐
│      实时预览区域 (60%)              │
│   [Mermaid 图表/节点图渐进渲染]       │
│   [节点可勾选]                       │
├─────────────────────────────────────┤
│      需求录入区域 (40%)              │
│   [需求输入框 + 示例 + 开始按钮]      │
└─────────────────────────────────────┘
```

---

### 需求 B: 节点勾选功能

**用户原话**: "识别出的每一步节点图块需要支持用户勾选"

**功能描述**:
- Mermaid 流程图中的每个节点可勾选
- 用户选择需要的节点/功能
- 仅生成用户勾选的部分

**技术方案**:
- Mermaid 图表支持点击交互
- 节点状态管理（勾选/未勾选）
- 数据传递给后端进行选择性生成

---

## 2. 技术可行性评估

| 需求 | 可行性 | 工作量 |
|------|--------|--------|
| 布局分离 | ✅ 可行 | 1.5天 |
| 节点勾选 | ⚠️ 需评估 | 2天 |

**风险点**:
- Mermaid 渲染后节点交互需要额外处理
- 可能需要使用 React Flow 替代 Mermaid 实现节点交互
