## 1. Problem Statement (核心痛点)

### 1.1 背景分析

VibeX 首页 (`/app/page.tsx`, 1026行) 已实现单页式五步设计流程，但存在以下核心问题：

| 问题类别 | 具体痛点 | 用户影响 |
|----------|----------|----------|
| **功能缺陷** | `/design` 导航链接 404 | 用户点击后看到空白页，困惑且无法继续 |
| **信息冗余** | Step 标题与侧边栏重复 | 视觉噪音，浪费屏幕空间 |
| **交互缺失** | 示例按钮无反馈 | 用户不知道是否成功点击，体验割裂 |
| **布局僵化** | 三栏比例固定 | 不同用户偏好无法满足，大屏用户无法充分利用空间 |
| **功能缺失** | 缺少画布式展示 | 无法直观看到整体设计，需要频繁切换 Tab |
| **体验断层** | 游客无法体验核心功能 | 转化率低，用户流失 |

### 1.2 用户痛点优先级

```
P0: #4 修复 design 404 ──── 阻断用户流程
P1: #2 Step 标题重复 ────── 视觉干扰
P1: #3 移除重复诊断 ─────── 功能冗余
P1: #6 示例交互优化 ─────── 交互反馈缺失
P2: #5 布局可调整 ───────── 个性化需求
P2: #1 中央画布展示 ─────── 可视化需求
P2: #7 页面组件关系图 ───── 信息展示
P2: #8 导入自有项目 ─────── 功能扩展
P2: #10 单页登录抽屉 ────── 体验优化
P3: #9 游客体验优化 ─────── 转化优化
P3: #11 集成测试覆盖 ────── 质量保障
```

## 2. Proposed Solution (解决方案)

### 2.1 分阶段实施策略

#### Phase 1: Bug 修复 (P0-P1) - 预计 2h

| ID | 需求 | 解决方案 | 验收标准 |
|----|------|----------|----------|
| #4 | 修复 design 404 | 重定向到 `/confirm` 流程 | 点击导航"设计"跳转到确认页 |
| #2 | Step 标题重复 | 中央标题改为描述性文字 | 标题显示"需求分析工作台"等描述 |
| #3 | 移除重复诊断 | 检查并移除冗余 DiagnosisPanel | 页面无重复诊断模块 |

#### Phase 2: UX 优化 (P1-P2) - 预计 6h

| ID | 需求 | 解决方案 | 验收标准 |
|----|------|----------|----------|
| #6 | 示例交互优化 | 添加点击反馈、滚动定位、清空按钮 | 点击示例有视觉反馈，自动滚动到输入框 |
| #5 | 布局可调整 | 添加拖拽分隔条，保存用户偏好 | 用户可拖拽调整三栏比例，刷新后保持 |

#### Phase 3: UI 增强 (P2) - 预计 6h

| ID | 需求 | 解决方案 | 验收标准 |
|----|------|----------|----------|
| #1 | 中央画布展示 | 集成 React Flow，支持多图表同屏 | 用户可在画布中查看所有图表，支持拖拽缩放 |

#### Phase 4: Feature 开发 (P2) - 预计 17h

| ID | 需求 | 解决方案 | 验收标准 |
|----|------|----------|----------|
| #7 | 页面组件关系图 | 复用 PageTreeDiagram，扩展树形结构 | Step 5 显示页面-组件树形关系 |
| #8 | 导入自有项目 | 完善导入流程，添加状态反馈 | GitHub/Figma 导入有进度指示和错误处理 |
| #10 | 单页登录抽屉 | 登录后不刷新页面，保留用户状态 | 登录成功后无缝继续操作 |

#### Phase 5: 体验优化 (P3) - 预计 4h

| ID | 需求 | 解决方案 | 验收标准 |
|----|------|----------|----------|
| #9 | 游客体验优化 | 提供 Demo 模式展示预设结果 | 游客可查看 Demo 生成效果 |

#### Phase 6: 测试覆盖 (P3) - 预计 8h

| ID | 需求 | 解决方案 | 验收标准 |
|----|------|----------|----------|
| #11 | 集成测试覆盖 | 补充五步流程、组件渲染、API mock 测试 | 覆盖率 > 70% |

## 4. Core User Stories (核心用户故事)

### 4.1 Epic: Bug 修复

#### US-001: 修复导航 404
```
作为 用户
我想要 点击导航栏"设计"后正常跳转
以便于 继续我的设计流程

验收标准:
- Given: 用户在首页
- When: 点击导航栏"设计"
- Then: 跳转到 /confirm 页面
- And: 页面正常显示
```

#### US-002: 消除标题重复
```
作为 用户
我想要 看到简洁清晰的页面标题
以便于 快速理解当前功能

验收标准:
- Given: 用户在首页
- When: 查看页面标题
- Then: 标题显示描述性文字（如"需求分析工作台"）
- And: 不与侧边栏 Step 标签重复
```

### 4.2 Epic: UX 优化

#### US-003: 示例交互反馈
```
作为 新用户
我想要 点击示例后有明确的视觉反馈
以便于 确认操作成功并看到效果

验收标准:
- Given: 用户在首页
- When: 点击示例按钮
- Then: 按钮有高亮动画
- And: 页面滚动到输入框
- And: 输入框显示示例内容
```

#### US-004: 布局自定义
```
作为 用户
我想要 调整三栏布局的比例
以便于 根据我的屏幕和偏好优化显示

验收标准:
- Given: 用户在首页
- When: 拖拽分隔条
- Then: 三栏比例实时调整
- And: 刷新页面后保持设置
- And: 移动端禁用拖拽
```

### 4.3 Epic: UI 增强

#### US-005: 画布式展示
```
作为 用户
我想要 在画布中同时查看多个图表
以便于 理解整体设计结构

验收标准:
- Given: 用户完成需求输入
- When: AI 生成图表
- Then: 图表在画布中展示
- And: 支持拖拽、缩放、平移
- And: 图表间有连接线
```

### 4.4 Epic: Feature 开发

#### US-006: 页面组件关系图
```
作为 用户
我想要 查看生成的页面和组件关系
以便于 理解项目结构

验收标准:
- Given: 用户完成项目生成
- When: 进入 Step 5
- Then: 显示页面-组件树形图
- And: 支持展开/收起
- And: 点击可查看详情
```

#### US-007: 导入自有项目
```
作为 用户
我想要 从 GitHub 或 Figma 导入现有项目
以便于 在此基础上继续设计

验收标准:
- Given: 用户点击导入选项
- When: 选择 GitHub/Figma 导入
- Then: 显示导入进度
- And: 导入成功后自动填充需求
- And: 导入失败显示错误提示
```

#### US-008: 无缝登录体验
```
作为 未登录用户
我想要 登录后继续之前的操作
以便于 不丢失已输入的内容

验收标准:
- Given: 用户已输入需求
- When: 点击生成触发登录
- Then: 登录成功后不刷新页面
- And: 保留用户输入内容
- And: 自动继续生成流程
```

### 4.5 Epic: 体验优化

#### US-009: 游客 Demo 模式
```
作为 游客
我想要 体验 AI 生成效果
以便于 决定是否注册

验收标准:
- Given: 游客访问首页
- When: 点击"体验 Demo"
- Then: 显示预设的生成结果
- And: 有"注册解锁完整功能"提示
```
