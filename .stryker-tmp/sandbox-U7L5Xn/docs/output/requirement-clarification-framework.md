# 需求澄清与协作设计流程

**版本**: v2.0  
**日期**: 2026-03-10  
**用途**: VibeX 协作式 DDD 建模流程设计

---

## 1. 核心理念

**VibeX 的差异化价值**: 协作式 DDD 建模，让用户深度参与每一步设计

**与竞品的核心差异:**

| 维度 | 竞品（Uxbot/v0/lovable） | VibeX |
|------|--------------------------|-------|
| 生成方式 | 黑盒一键生成 | 分步协作生成 |
| 用户参与 | 被动接受结果 | 主动调整每一步 |
| 输出质量 | 碰运气 | 精确可控 |
| 学习价值 | 无 | 掌握 DDD 思维 |

**流程设计原则:**
- 每一步都生成初始方案 → 用户确认/调整 → 进入下一步
- 不可跳过任何步骤（保证设计质量）
- 每一步支持自然对话修改

### 2.1 必须收集（P0）

| 维度 | 问题模板 | 示例回答 |
|------|----------|----------|
| **产品类型** | 这是一个什么类型的产品？ | 电商平台、社交应用、SaaS 工具... |
| **目标用户** | 谁会使用这个产品？ | C端消费者、企业员工、医生患者... |
| **核心场景** | 用户最常用的功能是什么？ | 下单购物、发布内容、审批流程... |

### 2.2 重要但可推导（P1）

| 维度 | 问题模板 | 说明 |
|------|----------|------|
| **核心实体** | 系统里主要管理哪些东西？ | 可从描述中提取关键词 |
| **业务流程** | 主要操作流程是怎样的？ | 可根据场景生成默认流程 |
| **用户角色** | 有哪些不同类型的用户？ | 可从"谁使用"推导 |

### 2.3 可选/可默认（P2）

| 维度 | 问题模板 | 默认值 |
|------|----------|--------|
| **平台** | 需要支持哪些平台？ | Web + Mobile |
| **风格偏好** | 有设计风格偏好吗？ | 现代简洁风格 |
| **集成需求** | 需要对接第三方服务吗？ | 无 |

## 3. 提问策略（三轮澄清法）

### 第一轮：核心定位
**触发条件**: 用户输入初始需求后

**问题池**:
```yaml
Q1_产品类型:
  question: "这是一个什么类型的产品？"
  type: "single_select"
  options:
    - "电商平台（买卖商品）"
    - "社交应用（用户互动）"
    - "内容平台（发布/浏览）"
    - "SaaS 工具（企业协作）"
    - "其他（请描述）"
  fallback: "从需求描述中提取关键词匹配"

Q2_目标用户:
  question: "谁会使用这个产品？"
  type: "single_select"
  options:
    - "C端消费者"
    - "企业员工"
    - "特定行业从业者（如医生、教师）"
    - "其他（请描述）"
  fallback: "默认为 C端消费者"

Q3_核心场景:
  question: "用户最常用的功能是什么？"
  type: "text"
  placeholder: "例如：浏览商品并下单、发布和分享内容..."
  fallback: "从需求描述中提取动词"
```

**逻辑**:
- 如果用户输入已包含这些信息 → 跳过对应问题
- 最多问 3 个问题
- 用户可随时点击"跳过，直接生成"

### 第二轮：业务细化
**触发条件**: 第一轮完成 + 系统检测到信息不足

**问题池**:
```yaml
Q4_核心实体:
  question: "系统里主要管理哪些东西？"
  type: "multi_select_with_custom"
  options:
    - "用户"
    - "商品/产品"
    - "订单/交易"
    - "内容/文章"
    - "消息/通知"
    - "其他（请输入）"
  auto_extract: true  # 从描述中自动提取实体关键词

Q5_用户角色:
  question: "有哪些不同类型的用户？"
  type: "multi_select_with_custom"
  options:
    - "普通用户"
    - "管理员"
    - "商家/卖家"
    - "审核员"
    - "其他（请输入）"
  fallback: "默认：普通用户 + 管理员"

Q6_业务特色:
  question: "有什么特别的功能或流程吗？"
  type: "text"
  placeholder: "例如：需要审批流程、有会员等级、支持多种支付方式..."
  fallback: "使用行业标准功能"
```

**智能跳过逻辑**:
- 如果检测到「电商」关键词 → 自动填充实体：商品、订单、用户
- 如果检测到「社交」关键词 → 自动填充实体：用户、帖子、评论
- 如果用户说"和淘宝类似" → 自动套用电商模板

### 第三轮：确认与补充
**触发条件**: 第二轮完成 + 准备生成前

**问题池**:
```yaml
Q7_确认摘要:
  question: "我理解您的需求如下，确认无误吗？"
  type: "summary_confirm"
  content: |
    📋 需求摘要
    
    • 产品类型：{产品类型}
    • 目标用户：{目标用户}
    • 核心场景：{核心场景}
    • 主要实体：{核心实体}
    • 用户角色：{用户角色}
    
    系统将为您生成：
    ✓ 领域模型图
    ✓ 业务流程图
    ✓ 可交互原型
    
    [确认，开始生成] [修改某些内容] [重新描述]
```

## 4. Step 2: 领域模型阶段（DDD 核心差异化）

### 4.1 目标
基于需求澄清结果，生成初始领域模型，让用户参与实体设计

### 4.2 交互流程

```
需求确认 → AI 生成初始实体 → 展示领域模型图 → 用户增删调整 → 确认进入下一步
```

### 4.3 生成规则

| 需求关键词 | 自动生成实体 |
|------------|--------------|
| 电商/商城/购物 | User, Product, Order, Cart, Payment |
| 社交/社区/论坛 | User, Post, Comment, Message, Follow |
| 内容/博客/CMS | User, Article, Category, Tag, Comment |
| SaaS/企业工具 | User, Workspace, Team, Permission, AuditLog |
| 教育/课程 | User, Course, Lesson, Enrollment, Progress |
| 预约/挂号 | User, Service, Appointment, Schedule |

### 4.4 用户操作

| 操作类型 | 对话示例 | 效果 |
|----------|----------|------|
| 添加实体 | "添加一个优惠券实体" | 新增 Coupon 实体 |

| 修改属性 | "订单需要加上配送地址" | Order 实体新增 address 属性 |
| 调整关系 | "用户和订单是一对多关系" | 更新关系连线 |

## 5. 特殊场景处理

### 5.1 用户输入已经很清晰
**检测条件**: 输入包含产品类型 + 目标用户 + 核心场景

**处理**: 跳过提问，直接展示摘要确认

```
检测到您的需求描述已经很清晰：
• 产品类型：电商
• 目标用户：C端消费者
• 核心场景：浏览商品、下单购买

是否直接开始生成？[确认] [补充更多细节]
```

### 5.2 用户选择"跳过"
**处理**: 使用默认值或推断值填充，继续流程

```yaml
跳过时的默认值:
  产品类型: "通用 Web 应用"
  目标用户: "C端消费者"
  核心场景: "浏览、搜索、操作"
  核心实体: "用户、内容"
  用户角色: "普通用户、管理员"
```

### 5.3 用户选择"和 xxx 类似"
**处理**: 自动套用对应模板

| 关键词 | 套用模板 |
|--------|----------|
| 淘宝、京东、拼多多 | B2C 电商模板 |
| 美团、饿了么 | O2O 本地生活模板 |
| 微信、抖音、小红书 | 社交媒体模板 |
| 飞书、钉钉、企业微信 | 企业协作模板 |
| Notion、飞书文档 | 文档协作模板 |

### 5.4 用户输入过于模糊
**检测条件**: 输入 < 10 个字，且无关键词

**处理**: 引导用户补充

```
您的需求描述比较简短，能再详细一点吗？
比如：这是一个给谁用的产品？主要解决什么问题？

[重新描述] [看看示例]
```

## 7. 技术实现建议

### 7.1 数据结构

```typescript
interface ClarificationState {
  // 原始输入
  rawInput: string;
  
  // 已收集信息
  collected: {
    productType?: string;
    targetUser?: string;
    coreScenario?: string;
    entities?: string[];
    userRoles?: string[];
    specialFeatures?: string[];
  };
  
  // 当前轮次
  currentRound: 1 | 2 | 3;
  
  // 待问问题
  pendingQuestions: Question[];
  
  // 状态
  status: 'collecting' | 'confirming' | 'generating';
}

interface Question {
  id: string;
  question: string;
  type: 'single_select' | 'multi_select' | 'text' | 'confirm';
  options?: string[];
  fallback?: string;
  priority: 'P0' | 'P1' | 'P2';
}
```
