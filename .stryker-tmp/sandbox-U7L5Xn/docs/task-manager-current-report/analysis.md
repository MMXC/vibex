## 1. 业务场景分析

### 背景
Coord Agent 在日常监控中需要快速了解：
1. 当前活跃项目状态
2. 虚假完成检测（任务标记 done 但实际未产出文件）
3. 服务器基本信息

现有 `task_manager.py` 缺少统一的报告入口，导致 Coord 需要手动执行多个命令才能获取完整信息。

### 目标用户
- **Coord Agent**: 日常监控、决策支持
- **团队负责人**: 项目状态概览

### 核心价值
- **效率提升**: 一条命令获取三类报告，无需多次调用
- **质量保障**: 虚假完成检测防止"纸面完成"
- **运维支持**: 服务器信息快速获取

## 3. 技术方案选项

### 方案 A: 独立 current-report 子命令

```python
# task_manager.py current-report
# 输出: 三段式报告
# - 活跃项目状态 (从 tasks.json 读取)
# - 虚假完成检测 (对比 status=done 与文件存在性)
# - 服务器信息 (psutil 获取)
```

**优点**:
- 命令单一，调用简单
- 符合现有 CLI 风格

**缺点**:
- 需要依赖 psutil 库

### 方案 B: 复用现有子命令组合

```bash
# 组合调用
task_manager.py status
task_manager.py health
# + 手动检测文件存在性
```

**优点**:
- 无新依赖
- 可复用现有代码

**缺点**:
- 需要多次调用
- 无虚假完成检测
- 无服务器信息

### 方案 C: 新增 --report 参数到 status

```bash
task_manager.py status --report  # 包含完整报告
```

**优点**:
- 不新增顶级命令
- 与 status 集成

**缺点**:
- status 已有的输出格式可能冲突
- 职责不单一

---

## 4. 推荐方案

**方案 A** — 独立 `current-report` 子命令

**理由**:
1. 职责单一，每个命令做一件事
2. 符合现有 CLI 风格（现有9个子命令）
3. 便于后续扩展报告类型

## 5. 功能需求

### F1: 活跃项目状态报告

| ID | 功能点 | 描述 | 验收标准 |
|----|--------|------|----------|
| F1.1 | 列出活跃项目 | 显示 status=active 的项目 | `expect(len(active_projects) >= 0)` |
| F1.2 | 显示项目阶段进度 | 每个活跃项目的阶段状态 | `expect(stage in ['analyze', 'prd', 'architecture', 'decision'])` |
| F1.3 | 显示阻塞任务 | pending > 0 的项目 | `expect(pending_count >= 0)` |

### F2: 虚假完成检测

| ID | 功能点 | 描述 | 验收标准 |
|----|--------|------|----------|
| F2.1 | 检测文件缺失 | status=done 但 output 文件不存在 | `expect(file_exists OR no_false_positive)` |
| F2.2 | 报告虚假完成列表 | 列出所有虚假完成项目 | `expect(false_positives <= total_done)` |

### F3: 服务器信息报告

| ID | 功能点 | 描述 | 验收标准 |
|----|--------|------|----------|
| F3.1 | CPU 使用率 | 百分比 | `expect(0 <= cpu <= 100)` |
| F3.2 | 内存使用率 | 百分比 + 绝对值 | `expect(0 <= memory <= 100)` |
| F3.3 | 磁盘使用率 | 百分比 | `expect(0 <= disk <= 100)` |
| F3.4 | 运行时间 | uptime | `expect(uptime > 0)` |

## 7. 初步风险识别

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| psutil 安装失败 | 服务器信息获取失败 | graceful fallback，仅显示可用信息 |
| tasks.json 读取超时 | 阻塞命令 | 设置 5 秒超时 |
| 虚假完成误报 | 用户困惑 | 仅检测 output 字段非空的 done 任务 |
