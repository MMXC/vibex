## 4. 提案详情

### P003: 自检报告路径规范化 (P0)

**问题描述**: 各 agent 自检报告路径不一致，导致 reviewer 审查时需要多轮猜测路径，效率低。

**根因分析**: 无统一的报告存放规范，各 agent 自行其是。

**影响范围**: 所有 agent 自检流程

**建议方案**: 统一路径：`/workspace-{agent}/proposals/YYYYMMDD/{agent}.md`

**工时**: 2h

**验收标准**: 任何 agent 可从单一路径找到所有 agent 的自检报告

---

### P001: 两阶段审查 SOP 文档化 (P1)

**问题描述**: reviewer 执行两阶段审查（功能审查 → 推送验证）但缺乏标准化文档，导致执行方式不统一。

**建议方案**: 创建 `REVIEWER_SOP.md`，固化检查清单。

**工时**: 3h

**验收标准**: 新 agent 可通过阅读 SOP 独立完成审查任务

---

### P002: 重复通知过滤机制缺失 (P1)

**问题描述**: Coord 向 reviewer 发送重复任务通知，导致消息堆积。

**建议方案**: Coord 心跳脚本增加"已处理任务"去重逻辑。

**工时**: 2h

**验收标准**: 同一任务通知 ≤ 2 次
