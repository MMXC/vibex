## 4. 提案详情

### P001: 建立 E2E Playwright 测试规范 (P1)

**问题描述**: Canvas 功能依赖单元测试 + 人工验证，F11/ESC 等键盘快捷键在 useEffect 中实现，单元测试无法覆盖。

**根因分析**: 缺少 headless 浏览器级别的 E2E 测试基础设施。

**影响范围**: Canvas 全流程交互、快捷键、模态框等 UI 行为。

**建议方案**:
1. 在 `vibex-fronted/e2e/` 建立 Playwright 测试目录
2. 覆盖关键路径：F11 全屏、ESC 退出、Click outside 退出、Checkbox toggle
3. 接入 CI，blocking merge on E2E failure

**工时**: 6h

**验收标准**:
- [ ] 至少 5 个 E2E 测试用例覆盖核心交互
- [ ] CI pipeline 中 E2E 测试失败则 blocking merge
- [ ] 测试报告自动生成截图附件

---

### P002: CI 测试质量 Gate 机制 (P1)

**问题描述**: 测试通过率下降时发现滞后（约 4-8h），测试失败传播到生产环境的风险高。

**根因分析**: 缺少 CI 级别的测试质量 gate。

**建议方案**:
1. 任何 suite 失败 → blocking merge + Slack @tester @dev
2. 覆盖率 < 80% → Slack 告警
3. 覆盖率下降 > 5% → blocking merge
4. 每日生成测试健康度报告到 #tester-channel

**工时**: 3h

**验收标准**:
- [ ] CI 中测试失败时 Slack 通知 < 5 分钟
- [ ] 覆盖率 < 80% 自动 blocking merge
- [ ] 每日测试健康度报告

---

### P003: 测试报告标准化与告警 (P2)

**问题描述**: 当前测试报告格式不统一，人工整理报告成本高。

**建议方案**: 建立统一的测试报告模板和自动化报告生成工具。

**工时**: 4h

**验收标准**:
- [ ] 统一报告模板存在
- [ ] 自动化报告生成
- [ ] 支持横向对比历史测试结果
