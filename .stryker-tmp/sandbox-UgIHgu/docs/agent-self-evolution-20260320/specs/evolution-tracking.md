# Feature: 进化效果追踪

## Jobs-To-Be-Done
- 作为一个 **技术负责人**，我希望能查看进化效果统计报告，以便评估系统健康度和团队表现。
- 作为一个 **Coord Agent**，我希望自动追踪每批次的提案数量、质量评分、转化率，以便发现异常。
- 作为一个 **团队成员**，我希望能了解整体改进趋势，以便调整工作重心。

## User Stories
- **US10**: 每批次完成后，Coord 自动生成统计报告（提案数、完成率、转化率）
- **US11**: 系统追踪历史趋势（7天滚动），识别下降信号
- **US12**: 异常告警：连续 2 批无提案的 Agent 自动通知

## Requirements
- [ ] (F4.1) 每批次完成后，Coord 生成 `proposals/YYYYMMDD/REPORT.md`
- [ ] (F4.2) REPORT.md 包含：提案总数、质量评分、转化率、异常 Agent 列表
- [ ] (F4.3) 保留最近 7 天的批次报告，支持趋势对比
- [ ] (F4.4) 异常检测规则：
  - 单批次提案 < 3 个 → 告警
  - 单 Agent 连续 2 批无提案 → 告警
  - 转化率 < 20%（连续 3 批次）→ 告警
- [ ] (F4.5) 告警通过 Slack 发送到 #coord 频道
- [ ] (F4.6) 统计数据可从 team-tasks 自动汇总（无需人工填写）

## Technical Notes
- 报告路径: `proposals/YYYYMMDD/REPORT.md`
- 统计逻辑: 读取 `workspace-coord/team-tasks/projects/` 下相关任务状态
- 告警冷却: 同一告警 24h 内不重复发送
- 历史保留: 报告保留 90 天，之后归档

## Acceptance Criteria
- [ ] **AC16**: 每批次完成后自动生成 REPORT.md（无需人工触发）
- [ ] **AC17**: 报告中包含：提案数、质量达标率、转化率、异常 Agent
- [ ] **AC18**: 异常检测覆盖 3 种场景（见 F4.4）
- [ ] **AC19**: 异常告警在 1 小时内送达 Slack
- [ ] **AC20**: 可查询最近 7 天趋势（文本表格形式）
