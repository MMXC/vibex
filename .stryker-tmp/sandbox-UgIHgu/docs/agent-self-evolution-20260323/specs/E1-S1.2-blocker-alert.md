# Spec: E1-S1.2 — 阻塞任务告警

**Epic**: Epic 1 流程执行监控体系  
**Story**: 阻塞任务告警  
**Agent**: pm (协调 coord/tester 执行)  
**状态**: Ready

---

## 概述

当检测到关键任务被阻塞时，自动通知相关 Agent 和 coord 频道，触发响应和处理流程。

---

## 功能描述

### 触发条件
- 任务状态变为 `blocked` 且持续 > 30 分钟
- 任务依赖项未完成且上游 Agent 无响应

### 通知流程
1. 识别阻塞任务及其依赖关系
2. 查找相关 Agent（阻塞者 + 被阻塞者）
3. 发送 Slack 通知到 #coord 和相关 Agent 频道
4. 记录告警事件到日志

### 通知内容
- 阻塞任务 ID 和名称
- 阻塞原因
- 上游未完成任务
- 建议处理方式

---

## 验收标准

| ID | Given | When | Then |
|----|-------|------|------|
| AC1 | 任务变为 blocked | 立即检测 | 30min 内发送告警 |
| AC2 | 告警已发送 | 后续检查 | 告警不重复发送（去重） |
| AC3 | 阻塞解除 | 任务变为 ready | 发送"阻塞解除"通知 |
| AC4 | 告警失败 | Slack API 异常 | 记录错误，重试 2 次 |
