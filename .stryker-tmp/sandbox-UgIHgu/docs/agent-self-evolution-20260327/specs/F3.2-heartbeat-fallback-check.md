# Spec: F3.2 — 心跳脚本回退检查

## 功能概述
心跳脚本增加本地提案目录检查，API 失败时不阻塞心跳流程，确保心跳在任何情况下都能正常完成。

## 实现方案

```bash
# pm-heartbeat.sh 改造
main() {
  local status="OK"
  
  # Step 1: 扫描本地任务（不依赖 API）
  scan_local_tasks() {
    # 扫描 team-tasks/ 和 workspace 下的任务文件
    # 查找状态为 in-progress/pending/ready 的任务
    return 0  # 始终成功
  }
  
  # Step 2: 写入本地提案（不依赖 API）
  archive_proposal() {
    mkdir -p "proposals/$(date +%Y%m%d)"
    echo "PM 自检: $(date)" > "proposals/$(date +%Y%m%d)/pm.md"
    return 0  # 始终成功
  }
  
  # Step 3: 尝试飞书通知（可失败）
  notify_feishu() {
    openclaw message send ... 2>/dev/null || true
    return 0  # 不管是否成功都返回 0
  }
  
  scan_local_tasks
  archive_proposal  
  notify_feishu
  
  echo "HEARTBEAT_OK"
  return 0
}
```
