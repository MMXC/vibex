# Spec: F5.1 — DAG 拓扑排序心跳领取

## 问题描述
心跳扫描时按字母顺序领取 Epic，而非按 DAG 依赖拓扑序，导致依赖链下游任务被优先领取但无法执行。

## 解决方案
心跳扫描时：
1. 读取所有 active 项目的 `stages[].dependsOn`
2. 构建依赖图
3. 按拓扑序排序，优先领取无上游依赖（或依赖已 done）的任务

## 算法（Kahn's Algorithm）
```python
def topological_sort(tasks):
    # tasks: {task_id: {"dependsOn": [dep_ids], "status": str}}
    in_degree = {tid: len(t["dependsOn"]) for tid, t in tasks.items()}
    ready = [tid for tid, deg in in_degree.items() if deg == 0]
    order = []
    while ready:
        tid = ready.pop(0)
        order.append(tid)
        for next_id, t in tasks.items():
            if tid in t["dependsOn"]:
                in_degree[next_id] -= 1
                if in_degree[next_id] == 0:
                    ready.append(next_id)
    return order
```

## 集成到 heartbeat.sh
```bash
# 替换现有的简单 sort，改为拓扑排序
sorted_tasks=$(topological_sort_tasks "$tasks")
```

## 验证
```bash
# 创建测试依赖链：
# A (无依赖) → B (依赖A) → C (依赖B)
# 验证心跳领取顺序: A → B → C

python3 -c "
# 模拟测试
graph = {'A': {'dependsOn': []}, 'B': {'dependsOn': ['A']}, 'C': {'dependsOn': ['B']}}
order = topological_sort(graph)
assert order == ['A', 'B', 'C'], f'Expected [A,B,C], got {order}'
print('✅ F5.1 拓扑排序验证通过')
"
```
