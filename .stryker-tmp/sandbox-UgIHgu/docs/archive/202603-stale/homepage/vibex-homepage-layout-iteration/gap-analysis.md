# 首页布局差异分析报告

**项目**: vibex-homepage-layout-iteration
**分析师**: Analyst Agent
**日期**: 2026-03-14

---

## 执行摘要

对比 PRD 设计与代码实现，发现 **7 个 Critical/High 级差异**，主要涉及多余区域（Hero/Features）、AI Panel 宽度不符、Input 位置不一致等问题。建议**移除 Hero 和 Features 区域**，调整 AI Panel 宽度至 15%，并重构 Input 布局。

---

## 2. 详细差异分析

### 2.1 🔴 Critical 级差异

#### D1: Hero Section 多余

| 维度 | PRD 设计 | 代码实现 |
|------|----------|----------|
| 存在性 | ❌ 无此区域 | ✅ 有完整 Hero |
| 内容 | - | Badge + 标题 + 副标题 + CTA 按钮 |
| 高度 | - | 约 400px |
| 背景 | - | 粒子特效 + 网格 + 光晕 |

**影响**: 
- 占用首屏大量空间
- 用户需要滚动才能看到核心功能
- 与 PRD 确认的设计意图冲突

**建议**: **移除 Hero Section**，直接展示核心三栏布局

#### D2: Features Section 多余

| 维度 | PRD 设计 | 代码实现 |
|------|----------|----------|
| 存在性 | ❌ 无此区域 | ✅ 有 4 卡片网格 |
| 内容 | - | 你主导、DDD建模、快速生成、实时预览 |
| 高度 | - | 约 200px |

**影响**:
- 占用额外滚动空间
- 与 Hero 功能重复（都在介绍产品）

**建议**: **移除 Features Section**，或移至单独的 Landing Page

#### D3: AI Panel 宽度不符

| 维度 | PRD 设计 | 代码实现 |
|------|----------|----------|
| 宽度 | 15% | 25% |
| 内容 | 点击生成时展示思考过程 | 常驻 AI 助手界面 |

**影响**:
- Preview 区域被压缩
- AI Panel 占据过多空间

**建议**: 调整 AI Panel 宽度至 **15%**

### 2.2 🟠 High 级差异

#### D4: Input 位置不一致

| 维度 | PRD 设计 | 代码实现 |
|------|----------|----------|
| 位置 | Preview 底部固定 | Content 区域内 |
| 布局 | 始终可见 | Tab 切换控制 |
| 与 Preview 关系 | 一体化 | 分离 |

**影响**:
- 用户无法在查看 Preview 时直接输入
- 需要 Tab 切换才能输入

**建议**: 重构为 **Preview 底部固定**，实现输入与预览一体化

#### D5: Header 导航逻辑不一致

| 维度 | PRD 设计 | 代码实现 |
|------|----------|----------|
| 高度 | 8% | ~60px (约 4%) |
| 未登录导航 | [免费开始] [查看演示] | [功能] [价格] [开始使用] |
| 已登录导航 | [项目管理] [个人设置] | - |

**影响**:
- 登录状态切换逻辑缺失
- 导航入口与 PRD 不符

**建议**: 
- 调整 Header 高度至 8%
- 实现登录状态导航切换

### 2.3 🟡 Medium 级差异

#### D6: Preview 画布可调整

| 维度 | PRD 设计 | 代码实现 |
|------|----------|----------|
| 可调整性 | ✅ 画布可调整大小 | ⚠️ 未确认实现 |

**建议**: 确认并实现画布调整功能

#### D7: AI Panel 激活机制

| 维度 | PRD 设计 | 代码实现 |
|------|----------|----------|
| 激活时机 | 点击"发送"后激活 | 常驻显示 |
| 默认状态 | 隐藏/折叠 | 始终显示 |

**建议**: 实现按需激活机制，默认可折叠
