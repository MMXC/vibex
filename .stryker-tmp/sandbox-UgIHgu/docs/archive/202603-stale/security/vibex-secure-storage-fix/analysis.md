## 2. P0: secure-storage.ts 空 catch 块分析

### 2.1 根因

OAuth 刷新 token 和登出时存在空 catch 块，静默吞掉错误：

```typescript
// oauth.ts line 202-204
try {
  await fetch(getApiUrl(`/api/oauth/${provider}/refresh`), { ... });
  // ...
  return true;
} catch {
  return false;  // ❌ 静默失败，无日志
}

// oauth.ts line 214-222
try {
  await fetch(getApiUrl(`/api/oauth/${provider}/revoke`), { ... });
} catch {
  // 忽略错误  // ❌ 静默失败，无日志
}
```

### 2.2 安全风险

| 风险 | 说明 | 影响 |
|------|------|------|
| 错误被吞 | 刷新失败时用户无感知 | ❌ 静默降级 |
| 无可调试性 | 无日志无法排查问题 | ⚠️ 调试困难 |
| token 可能泄露 | revoke 失败但继续删除本地 token | ⚠️ 安全隐患 |

### 2.3 推荐修复

```typescript
// 方案：添加错误日志 + 用户提示

// 刷新 token
try {
  const response = await fetch(...);
  if (!response.ok) {
    console.error(`[OAuth] Token refresh failed for ${provider}: ${response.status}`);
    return false;
  }
  const tokens = await response.json();
  await storeTokens(provider, tokens);
  return true;
} catch (error) {
  console.error(`[OAuth] Token refresh error for ${provider}:`, error);
  return false;
}

// 登出 revoke
try {
  await fetch(getApiUrl(`/api/oauth/${provider}/revoke`), { ... });
} catch (error) {
  console.warn(`[OAuth] Revoke failed for ${provider} (non-critical):`, error);
  // 仍继续删除本地 token，因为登出是幂等的
}
```

### 2.4 验收标准

- [ ] `catch` 块包含 `console.error` 或 `console.warn`
- [ ] 关键错误（如 token 刷新失败）有明确的返回值
- [ ] 日志包含足够上下文（provider、error message）
- [ ] 登出错不阻塞登出流程（最终仍删除本地 token）
