# Bug 提案: 登录状态未正确保持

**Bug ID**: VIBEX-003  
**严重级**: P0  
**状态**: Open  
**发现日期**: 2026-03-11  
**发现者**: 小羊  

---

## 1. 问题描述

用户登录成功后，点击"生成"按钮仍然提示登录。

### 当前行为
1. 点击"生成" → 右侧抽屉弹出登录表单
2. 用户登录成功
3. 再次点击"生成" → 又弹出登录抽屉，提示登录
4. 登录状态未正确保持

### 预期行为
- 登录成功后，再次点击"生成"应正常执行生成流程
- 不应重复弹出登录抽屉

---

## 2. 影响范围

- 用户无法正常使用核心功能
- 登录体验极差

---

## 3. 可能原因

- 登录后 token 未正确保存到 localStorage
- 登录后状态未正确更新
- 登录检测逻辑有问题

---

## 4. 验收标准

| 验收项 | 描述 |
|--------|------|
| 登录成功保存 token | 登录后 localStorage 应有 auth_token |
| 登录状态正确 | 再次点击生成不应弹出登录抽屉 |
| 生成流程正常 | 登录后点击生成应执行生成逻辑 |
