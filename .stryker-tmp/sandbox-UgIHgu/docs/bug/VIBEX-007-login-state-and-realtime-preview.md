# Bug 提案: 登录状态和实时预览问题

**Bug ID**: VIBEX-007  
**严重级**: P0  
**状态**: Open  
**发现日期**: 2026-03-12  
**发现者**: 小羊  

---

## 1. 问题列表

### 1.1 登录状态问题
- **问题**: 点击"开始生成"弹出登录抽屉，登录成功后点击"开始生成"又弹出登录抽屉
- **原因**: 登录后状态未正确保持
- **要求**: 
  - 登录成功后保持登录态
  - 同时保留用户已输入的内容

### 1.2 实时预览缺失
- **问题**: 输入需求（焦点离开输入框）或选择模板后没有实时预览
- **原因**: 需要后端 API 支持
- **要求**:
  - 前端输入时调用后端 API
  - 后端返回 Mermaid 图表数据
  - 前端实时渲染预览

---

## 2. 前后端协作

### 前端
- 监听输入框 blur 事件
- 监听模板点击事件
- 调用后端 API 获取预览数据
- 渲染 Mermaid 图表

### 后端
- 提供预览 API
- 输入需求文本，返回：
  - 上下文图 (Mermaid)
  - 模型图 (Mermaid)
  - 流程图 (Mermaid)
  - UI 节点图 (Mermaid)

---

## 3. 验收标准

| 验收项 | 描述 |
|--------|------|
| 登录态保持 | 登录后点击生成不再弹出登录 |
| 内容保留 | 登录后用户输入内容不丢失 |
| 实时预览 | 输入或选择模板后显示预览 |
| Mermaid 渲染 | 正确渲染上下文图、模型图、流程图、UI节点图 |
