# Vibex JSON 树可视化与模块化重生成 - 可行性分析提案

## 📋 执行摘要

分析 Vibex 生成 JSON 结构的流程图/UI组件树图可视化能力，以及页面区域选择后模块化重生成功能。

**核心发现**：技术可行，但需分阶段实施。建议分 3 个阶段迭代开发。

---

## 🎯 需求理解

### 功能一：图形化节点可视化
- 自定义画布上通过节点树卡片连线描述关系结构
- 用户可在图上勾选想要的流程/组件

### 功能二：区域选择重生成
- 用户选中页面区域
- 系统获取该区域 JSON 内容
- 可控重新模块化生成

## 🔬 可行性分析

### 技术可行性评估

| 功能模块 | 复杂度 | 技术方案 | 风险等级 |
|----------|--------|----------|----------|
| 节点可视化画布 | 中 | ReactFlow + 自定义节点组件 | 🟢 低 |
| 节点多选/勾选 | 低 | ReactFlow SelectionMode + checkbox overlay | 🟢 低 |
| 区域选择提取 | 中 | ReactFlow 选区 API + 路径追踪 | 🟡 中 |
| 模块化重生成 | 高 | JSON Patch + LLM Prompt 重构 | 🔴 高 |
| 双向同步 | 高 | 状态管理 + 反向映射 | 🔴 高 |

### 关键技术挑战

**挑战 1：节点勾选与流程选择**
- ReactFlow 支持节点选择，但需要添加 checkbox overlay
- 需要维护选中的节点 ID 集合
- 勾选后需要输出精简的 JSON 子集

**挑战 2：页面区域选择**
- 传统的 DOM 选区 (Range Selection) 与 ReactFlow 坐标不一致
- 需要将画布坐标转换为 JSON 路径
- 需要可视化选区边框

**挑战 3：模块化重生成**
- 重新生成需要保留上下文（父组件、主题、布局约束）
- 需要设计增量生成 vs 全量生成的策略
- Prompt 设计需要包含足够的上下文信息

---

## 📦 方案设计

### 推荐方案：渐进式增强

**阶段 1：可视化增强（2-3 周）**
- 在 PageTreeDiagram 基础上添加节点勾选能力
- 添加选中高亮和批量选择
- 输出选中节点的 JSON 子集

**阶段 2：选区支持（2-3 周）**
- 实现画布矩形选区
- 将选区映射到组件树路径
- 添加选区预览和确认 UI

**阶段 3：重生成集成（3-4 周）**
- 设计重生成 Prompt 模板
- 实现增量生成 API
- 集成到现有生成流程
