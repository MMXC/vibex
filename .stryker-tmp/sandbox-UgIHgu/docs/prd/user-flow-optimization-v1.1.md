## 1. 需求澄清

### 1.1 用户反馈要点

| 需求项 | 状态 | 说明 |
|--------|------|------|
| 新建项目跳转到需求描述 | ✅ 已完成 | 点击"创建新项目" → `/requirements/new` |
| 点击项目卡片进入单项目管理页面 | ❌ 待实现 | 跳转到 `/project/[id]` 而非 `/editor` |

### 1.2 核心概念澄清

**两个页面框架**：

1. **VibeX 项目管理入口 (Dashboard)**
   - 路径: `/dashboard`
   - 菜单: 项目、模板（当前够用）
   
2. **单项目管理页面框架** (新增)
   - 路径: `/project/[id]`
   - 侧边菜单: **需求 | 模型 | 流程 | 原型**

### 1.3 四个菜单项对应关系

| 菜单项 | 对应路由 | 功能说明 | 现有状态 |
|--------|----------|----------|----------|
| 需求 | `/requirements` 或 `/project/[id]/requirements` | 需求列表/管理 | ✅ 已有 `/requirements` 页面 |
| 模型 | `/domain` 或 `/project/[id]/domain` | 领域模型/实体关系图 | ✅ 已有 `/domain` 页面 |
| 流程 | `/flow` 或 `/project/[id]/flow` | 流程图/业务流程 | ✅ 已有 `/flow` 页面 |
| 原型 | `/prototype` 或 `/project/[id]/prototype` | 原型预览/页面树 | ✅ 已有 `/prototype` 页面 |

**结论**: 四个菜单项对应的页面**均已存在**，无需新建路由。只需创建 `/project/[id]` 项目详情页，集成现有页面即可。

#### 3.2.3 路由映射

| 菜单项 | 实际路由 | 说明 |
|--------|----------|------|
| 需求 | `/project/[id]/requirements` 或 `/requirements?projectId=[id]` | 复用现有页面，添加项目筛选 |
| 模型 | `/project/[id]/domain` 或 `/domain?projectId=[id]` | 复用现有页面，添加项目筛选 |
| 流程 | `/project/[id]/flow` 或 `/flow?projectId=[id]` | 复用现有页面，添加项目筛选 |
| 原型 | `/project/[id]/prototype` 或 `/prototype?projectId=[id]` | 复用现有页面，添加项目筛选 |

**实现建议**: 可以选择两种方案之一：
1. **方案A**: 新建 `/project/[id]/xxx` 子路由，包装现有页面并传入 projectId
2. **方案B**: 在现有页面添加 projectId 查询参数，自动筛选对应项目数据
