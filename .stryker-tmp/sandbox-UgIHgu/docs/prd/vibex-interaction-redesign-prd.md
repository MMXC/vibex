# PRD: VibeX 交互体验重构

**项目名称**: vibex-interaction-redesign  
**版本**: 1.0  
**创建日期**: 2026-03-04  
**负责人**: PM Agent  
**上游文档**: [Brief](./docs/analyze-requirements/brief.md)

---

## 1. 项目目标

重构 VibeX 交互体验，实现：
1. 统一入口：需求录入首页，登录在生成时触发
2. 登录抽屉：半屏展示，顶部导航推开，避免内容丢失
3. 导航设计：登录后显示完整导航项
4. 布局优化：顶部导航可收起，丝滑过渡动画

---

## 2. 功能需求

### 2.1 统一入口 - 需求录入首页

| ID | 功能点 | 验收标准 | 优先级 |
|----|--------|----------|--------|
| F2.1.1 | 首页直接显示需求录入表单 | 访问 `/` 或 `/home` 直接显示表单，无需登录 | P0 |
| F2.1.2 | 需求输入框 | 支持多行文本输入，无字数限制，占位符 "描述您的产品需求..." | P0 |
| F2.1.3 | 生成按钮 | 显示 "生成" 按钮，点击前不触发任何登录检查 | P0 |
| F2.1.4 | 登录触发逻辑 | 点击生成时检查登录态：已登录→继续，未登录→打开登录抽屉 | P0 |
| F2.1.5 | 草稿保存 | 输入内容自动保存到 localStorage | P1 |

**交互细节**:
- 页面加载时检查 localStorage，如有草稿则恢复
- 输入框失焦时自动保存草稿
- 草稿过期时间：7 天

|----|--------|----------|--------|

| F2.2.2 | 抽屉收起动画 | 从底部向上滑出，时长 250ms，缓动 ease-in | P0 |
| F2.2.3 | 抽屉高度 | 50vh，半屏展示 | P0 |
| F2.2.4 | 背景遮罩 | 无全屏遮罩，主内容区同步上移 | P0 |
| F2.2.5 | 毛玻璃效果 | backdrop-filter: blur(20px) | P0 |
| F2.2.6 | 登录方式 | 支持邮箱/密码、GitHub、Google | P0 |
| F2.2.7 | 登录失败处理 | 抽屉保持打开，显示错误提示 "用户名或密码错误" | P0 |
| F2.2.8 | 网络错误处理 | 显示 "网络错误，请重试" 及重试按钮 | P1 |
| F2.2.9 | 第三方登录取消 | 保持抽屉打开，显示其他登录选项 | P1 |

**动画规范**:
```css
/* 抽屉展开 */
@keyframes drawerSlideDown {
  from { transform: translateY(-100%); }
  to { transform: translateY(0); }
}
.drawerOpen {
  animation: drawerSlideDown 300ms ease-out forwards;
}

/* 抽屉收起 */
@keyframes drawerSlideUp {
  from { transform: translateY(0); }
  to { transform: translateY(-100%); }
}
.drawerClose {
  animation: drawerSlideUp 250ms ease-in forwards;
}

/* 内容区同步移动 */
.mainContentShift {
  transition: transform 300ms ease-out;
  transform: translateY(50vh); /* 与抽屉高度同步 */
}
```

**触发条件**:
- 点击 "生成" 按钮时检查登录态
- 点击导航栏 "登录" 按钮
- URL 参数 `?login=true`

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 登录 API 延迟 | 抽屉打开慢 | 预加载登录组件 |

| 第三方登录失败 | 用户流失 | 提供备选登录方式 |
