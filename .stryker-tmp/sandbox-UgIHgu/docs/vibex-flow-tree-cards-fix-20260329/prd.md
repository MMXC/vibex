# Vibex 流程树卡片显示 Bug 修复 PRD

**项目**: vibex-flow-tree-cards-fix-20260329  
**文档类型**: 产品需求文档 (PRD)  
**日期**: 2026-03-29  
**状态**: Draft → Ready for Implementation

---

## 🎯 一、背景与目标

### 1.1 问题描述

用户反馈：vibex 画布页流程树展开子节点后，只能显示约 2 张卡片，下方卡片无法操作。虚线框高度未随子步骤数量自适应扩展。

### 1.2 目标

修复 `.flowCard` CSS `overflow: hidden` 导致的卡片内容截断问题，使展开的流程卡片高度随子步骤数量自然扩展，所有卡片均可正常交互。

---

## 📦 二、功能列表

| ID | 功能点 | 类型 | 说明 |
|----|--------|------|------|
| F1 | `.flowCard` overflow 修复 | Bug Fix | 删除 `canvas.module.css` 中 `.flowCard` 的 `overflow: hidden` |

### 修复范围

**包含**:
- `vibex-fronted/src/components/canvas/canvas.module.css` — `.flowCard` CSS 样式修改

**不包含**:
- `.stepsList` 的 `max-height: 300px` 设计（属合理上限，保留不变）
- 其他画布组件的 CSS 修改
- 流程树组件 BusinessFlowTree.tsx 逻辑修改

## ✅ 三、验收标准

### 3.1 功能验收

| ID | 验收条件 | 验证方式 |
|----|----------|----------|
| AC1 | 展开流程树子节点后，虚线框高度随子步骤数量自适应扩展 | gstack browse 截图验证 |
| AC2 | 超过 2 张卡片时，底部卡片仍可点击/交互 | gstack browse 交互验证 |

### 3.2 修复前/修复后对比验证

| 场景 | 修复前行为 | 修复后期望行为 |
|------|-----------|---------------|
| 展开含 3+ 步骤的流程卡片 | 第 3 张卡片被截断，无法操作 | 虚线框扩展，3 张卡片全部可见可交互 |
| 展开含 6+ 步骤的流程卡片 | 大部分卡片被截断 | 虚线框扩展，.stepsList 内部滚动（max-height: 300px），步骤列表可滚动查看全部 |
| 收起/展开切换 | 无变化 | 无变化 |
