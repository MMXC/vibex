# Analysis: vibex-notification-filter

**Goal**: Coord 通知过滤机制 — 解决重复任务通知问题
**Priority**: P1
**Date**: 2026-03-31
**Analyst**: PM (补写)

---

## 1. 问题描述

Coord 心跳扫描任务时，同一任务被重复通知多次（如 VibexReviewerProposals 每日提醒），导致 Slack 频道消息堆积，噪音严重。

## 2. 根因分析

Coord 心跳脚本无"已处理任务"去重逻辑：
- 每次心跳扫描全量任务
- 状态已为 done/ready 的任务仍被通知
- 无通知冷却机制

## 3. 建议方案

| 功能 | 描述 | 工时 |
|------|------|------|
| S1 | Coord 心跳脚本增加"已通知任务"去重集合 | 1h |
| S2 | 同一任务通知 ≤ 2 次（首次+失败重试） | 0.5h |
| S3 | 冷却期内跳过已通知任务 | 0.5h |

## 4. 验收标准

- [ ] 同一任务通知 ≤ 2 次
- [ ] done/ready 状态不重复通知
- [ ] 日志记录通知次数

**总工时**: 2h
