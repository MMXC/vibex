## 4. 提案详情

### 提案1: Canvas 状态管理规范

**问题描述**: Canvas 存在多重状态冲突（`selectedNodeIds` vs `node.confirmed`），checkbox 行为不可预测。

**建议方案**:
- 节点状态只有两种：未确认（灰色）/ 已确认（绿色）
- 点击 checkbox 可切换状态
- 状态变更有即时视觉反馈

**KPI**:
| KPI | 当前 | 目标 |
|-----|------|------|
| checkbox 操作成功率 | ~70% | ≥ 95% |

**工时**: 3h

**验收标准**:
- [ ] 节点状态只有两种
- [ ] checkbox 点击有即时反馈
- [ ] 操作成功率 ≥ 95%

---

### 提案2: 提案生命周期追踪机制

**问题描述**: 提案提交后缺乏追踪，用户不知道提案是否被评审、执行。

**建议方案**:
- 每条提案有状态：已提交 / 评审中 / 已采纳 / 已执行 / 已拒绝
- 状态变更有通知
- 提案可在 dashboard 查看

**KPI**:
| KPI | 当前 | 目标 |
|-----|------|------|
| 提案执行率 | ~30% | ≥ 60% |

**工时**: 6h

**验收标准**:
- [ ] 提案状态可追踪
- [ ] 状态变更有 Slack 通知
- [ ] 执行率 ≥ 60%

---

### 提案3: 用户引导流程优化

**问题描述**: 新用户首次使用 Canvas 时缺乏引导，不知道如何开始创建 Bounded Context。

**建议方案**:
- 首次登录显示引导教程（可跳过）
- 每个关键操作有 tooltip 说明
- 引导完成后用户能独立完成基本操作

**KPI**:
| KPI | 当前 | 目标 |
|-----|------|------|
| 新用户首次完成率 | ~40% | ≥ 80% |
| 引导教程完成率 | N/A | ≥ 60% |
| 首次使用时间 | > 30min | ≤ 5min |

**工时**: 8h

**验收标准**:
- [ ] 引导教程可显示/跳过
- [ ] tooltip 覆盖关键操作
- [ ] 首次完成率 ≥ 80%

---

### 提案4: PRD 模板标准化

**问题描述**: PRD 模板不统一，不同项目格式差异大。

**建议方案**: 建立统一的 PRD 模板，包含：
- 背景与目标
- 用户故事（格式：作为…我想要…以便…）
- 验收标准（格式：given/when/then）
- DoD 定义

**工时**: 2h

**验收标准**:
- [ ] 模板文档存在
- [ ] 所有新项目使用统一模板
- [ ] 模板包含所有必要字段
