## 当前差距

1. **限界上下文树**：节点之间没有领域关系（依赖/聚合/组合连线）
2. **业务流程树**：没有分支网关、循环回路
3. **UI组件树**：节点不可展开/折叠/交互

## 交互设计简化（Alex 补充）

**原则**：三树节点用 checkbox 选择，移除显式确认/删除按钮

**原因**：用户可能回头勾选之前觉得不需要的节点，不需要强制 commit

**具体方案**：
- 每张卡片左侧加 checkbox
- 移除 ❌ 确认 / ❌ 删除按钮
- 当前 `confirmed: boolean` → 保持不变，但交互改为 checkbox toggle
- 子节点也支持 checkbox 独立勾选
