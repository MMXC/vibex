## 二、各区域详细说明

### 2.1 Header (顶部导航栏)

**位置**: 页面最顶部，固定高度 50px

**组成元素**:
| 元素 | 类型 | 宽度 | 说明 |
|------|------|------|------|
| Logo | 文字 | auto | "VibeX"，主色 #3b82f6 |
| 导航链接 | 链接组 | flex | 首页/项目/模板/更新日志 |
| 登录按钮 | 按钮 | auto | 触发登录抽屉 |

**交互**:
- 登录按钮点击 → 从顶部下推登录抽屉
- 导航链接点击 → 路由跳转

**状态**:
- 默认: 正常显示
- 登录后: 显示用户头像/退出按钮

---

### 2.2 左侧抽屉 (步骤指示器)

**位置**: 预览区左侧，可折叠

**组成元素**:
| 元素 | 说明 |
|------|------|
| 抽屉标题 | "步骤指示器" |
| 步骤项 | 4个步骤，垂直排列 |

**步骤定义**:
| 序号 | 名称 | 状态标识 |
|------|------|----------|
| 1 | 需求录入 | 当前步骤高亮 |
| 2 | 需求澄清 | 待处理 |
| 3 | 业务流程 | 待处理 |
| 4 | 组件图 | 待处理 |

**步骤项结构**:
```
┌─────────────────────────┐
│ ○ 1  需求录入           │  ← 圆角数字 + 文字
└─────────────────────────┘
```

**状态**:
| 状态 | 样式 |
|------|------|
| 默认 | 白底 + 灰边框 |
| 激活 | 浅蓝底(#dbeafe) + 蓝边框 + 蓝字 |
| 完成 | 绿色勾选图标 |

**交互**:
- 点击步骤 → 切换到对应步骤
- 支持随时切换步骤（不强制线性流程）

---

### 2.3 预览区 (核心展示区)

**位置**: 中央区域，占据主要空间

**组成元素**:
| 元素 | 说明 |
|------|------|
| Mermaid图表容器 | 显示DDD分析结果 |
| 预览占位符 | 未生成时显示 |

**内容类型**:
1. 限界上下文图 (Bounded Context)
2. 领域模型图 (Domain Model)
3. 业务流程图 (Flow Chart)
4. 组件结构图 (Component Tree)

**状态**:
| 状态 | 显示内容 |
|------|----------|
| 空状态 | 占位图 + 提示文字 |
| 加载中 | 骨架屏 + 加载动画 |
| 已生成 | Mermaid渲染结果 |
| 错误 | 错误提示 + 重试按钮 |

**交互**:
- 滚动手势 → 缩放/拖动图表
- 双击 → 全屏模式
- 右键 → 导出/复制菜单

---

### 2.4 右侧抽屉 (AI思考过程)

**位置**: 预览区右侧，可折叠

**组成元素**:
| 元素 | 说明 |
|------|------|
| 抽屉标题 | "AI 思考过程" |
| 思考项列表 | SSE流式更新的思考内容 |

**思考项结构**:
```markdown
┌──────────────────────────┐
│ 正在分析需求结构...      │  ← 左侧蓝色边框标识
└──────────────────────────┘
```

**状态**:
| 状态 | 样式 |
|------|------|
| 默认 | 普通样式 |
| 新增 | 呼吸动画 (pulse) |
| 完成 | 静态 |

**交互**:
- 自动接收SSE流式更新
- 点击可展开查看详情

---

### 2.5 底部面板 (Bottom Panel)

**位置**: 页面底部，可拖动收起

**子区域结构** (从上到下):
```
┌─────────────────────────────────────┐
│ ⬆️ 拖动收起 (30px)                  │ ← 收起手柄
├─────────────────────────────────────┤
│ [输入框........................] [发送] │ ← 需求录入区 (80px)
├─────────────────────────────────────┤
│ AI询问 │ 诊断 │ 优化 │ 历史 ││ 💾 │ 🔄 ││ 📁创建项目 │ ← 操作按钮栏 (50px)
├─────────────────────────────────────┤
│ ┌─────────┐ ┌─────────┐ ┌─────────┐ │
│ │智能诊断 │ │应用优化 │ │AI对话澄清│ │ ← AI展示区 (弹性)
│ │         │ │         │ │         │ │
│ └─────────┘ └─────────┘ └─────────┘ │
└─────────────────────────────────────┘
```

---

### 2.5.1 收起手柄

**高度**: 30px  
**背景**: #f3f4f6 (浅灰)  
**内容**: "⬆️ 拖动收起"

**交互**:
- 拖动 → 调整底部面板高度
- 双击 → 完全收起/展开

---

### 2.5.2 需求录入区

**高度**: 80px  
**组成**:
| 元素 | 宽度 | 说明 |
|------|------|------|
| 输入框 | flex: 1 | 多行文本输入 |
| 发送按钮 | 60px | 触发AI分析 |

**输入框属性**:
- placeholder: "输入需求或指令..."
- 最小行数: 2
- 最大行数: 4
- 自动增高

**状态**:
| 状态 | 样式变化 |
|------|----------|
| 空 | placeholder显示 |
| 输入中 | 边框变蓝 |
| 发送中 | 按钮禁用，显示loading |
| 错误 | 红色边框 + 错误提示 |

---

### 2.5.3 操作按钮栏

**高度**: 50px  
**分组**:
| 分组 | 按钮 |
|------|------|
| 快捷功能 | AI询问、诊断、优化、历史 |
| 操作 | 保存、重新生成 |
| 主操作 | 创建项目 |

**按钮样式**:
| 类型 | 背景色 | 边框色 |
|------|--------|--------|
| 默认 | #f3f4f6 | #e5e7eb |
| 悬停 | #dbeafe | #3b82f6 |
| 主按钮 | #3b82f6 | #3b82f6 |

---

### 2.5.4 AI展示区

**布局**: 三列卡片网格，间距 12px  
**可滚动**: 支持垂直滚动

**卡片定义**:
| 卡片 | 图标 | 内容示例 |
|------|------|----------|
| 智能诊断 | 🔍 | 发现 2 个潜在问题需要修复... |
| 应用优化 | ✨ | 推荐优化方案：组件复用、状态简化... |
| AI对话澄清 | 💬 | 需要澄清：用户认证方式？单设备/多设备？ |

**卡片结构**:
```
┌─────────────────────────────┐
│ 🔍 智能诊断                  │ ← 卡片标题
├─────────────────────────────┤
│ 发现 2 个潜在问题需要修复... │ ← 内容区
│                             │
└─────────────────────────────┘
```

**状态**:
| 状态 | 显示 |
|------|------|
| 空 | "暂无分析结果" |
| 加载中 | 骨架屏 |
| 有数据 | 卡片内容 |
| 展开 | 点击卡片展开详情 |

## 三、抽屉交互行为

### 3.1 左右抽屉展开逻辑

**展开条件**:
- 用户手动点击抽屉开关
- 特定操作触发（如点击步骤指示器）

**对底部区域的影响**:
```
┌─────────────────────────────────────────────┐
│ 预览区              预览区    预览区         │
│                    预览区                      │
│                    预览区                      │
│ ┌──────┐           ┌──────┐                 │
│ │左侧  │  预览区    │右侧  │  预览区          │
│ │抽屉  │  预览区    │抽屉  │  预览区          │
│ └──────┘           └──────┘                 │
│ ┌────────────────────────────────────────┐   │
│ │              底部面板                   │ ← 被遮住/压缩
│ └────────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

**遮住效果**:
- 左右抽屉 z-index: 10
- 底部面板 z-index: 5
- 抽屉展开时遮住底部面板边缘

---

### 3.2 底部面板悬浮模式

**触发条件**: 用户滚动预览区时

**悬浮行为**:
```
滚动触发
    ↓
底部面板缩小到悬浮栏高度 (60px)
    ↓
悬浮栏显示: 录入框 + 收起按钮
    ↓
用户点击收起按钮
    ↓
底部面板完全收起
```

**悬浮栏内容**:
| 元素 | 说明 |
|------|------|
| 简化录入框 | 单行输入 |
| 收起按钮 | 展开底部面板 |

## 四、状态管理

### 4.1 全局状态 (designStore)

```typescript
interface DesignStore {
  // 当前步骤
  currentStep: 1 | 2 | 3 | 4;
  
  // 抽屉状态
  leftDrawerOpen: boolean;
  rightDrawerOpen: boolean;
  
  // 底部面板状态
  bottomPanelHeight: number;      // 高度 (0-380px)
  bottomPanelFloating: boolean;   // 是否悬浮
  
  // 分析结果
  boundedContexts: BoundedContext[];
  domainModels: DomainModel[];
  flowCharts: FlowChart[];
  
  // SSE状态
  thinkingMessages: ThinkingMessage[];
  aiDisplayCards: DisplayCard[];
}
```

### 4.2 状态快照

每次步骤切换前保存状态快照，支持回退：
```typescript
interface StateSnapshot {
  step: number;
  data: DesignData;
  timestamp: number;
}
```
