# Spec: E1-S1.1 — 每日进度扫描

**Epic**: Epic 1 流程执行监控体系  
**Story**: 每日进度扫描  
**Agent**: pm (协调 dev/tester 执行)  
**状态**: Ready

---

## 概述

建立每日定时任务，扫描 vibex-simplified-flow 项目状态，生成进度报告并发送到 #coord 频道。

---

## 功能描述

### 输入
- vibex-simplified-flow 项目的 team-tasks JSON 文件
- 当前日期

### 处理逻辑
1. 读取 team-tasks JSON
2. 按 Agent 统计各状态任务数量
3. 识别阻塞任务（status=blocked 或依赖项未完成）
4. 计算整体完成率

### 输出
- 格式化进度报告（Markdown）
- 发送到 Slack #coord 频道

---

## 验收标准

| ID | Given | When | Then |
|----|-------|------|------|
| AC1 | cron 触发 | 每日执行 | 报告送达 #coord 频道 |
| AC2 | 项目无任务 | 扫描时 | 报告显示"无活跃任务" |
| AC3 | 有阻塞任务 | 扫描时 | 阻塞任务列入报告 |
| AC4 | 执行失败 | cron 触发 | 自动重试 1 次，记录错误日志 |

---

## DoD

- [ ] 报告模板符合规范
- [ ] Slack 消息发送成功

- 开发: 1 人/天
- 测试: 0.5 人/天

---

## 实现约束

- 使用现有 team-tasks JSON 格式
- Slack 消息不超过 3000 字符
