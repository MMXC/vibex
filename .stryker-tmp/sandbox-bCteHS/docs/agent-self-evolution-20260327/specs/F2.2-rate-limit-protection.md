# Spec: F2.2 — Feishu API 配额保护

## 功能概述
心跳脚本增加 429 错误检测，超限时自动降级为每日一次通知，防止 API 超限导致的协作中断。

## 实现方案

修改 `/root/.openclaw/scripts/heartbeats/pm-heartbeat.sh` (及其他 agent 的心跳脚本):

```bash
# 在 openclaw message send 调用后增加错误检测
if [ $? -eq 0 ]; then
  echo "✅ 飞书通知已送达"
else
  # 检测是否因配额问题失败
  if grep -q "429\|rate.limit\|exhausted" "$tmp_log" 2>/dev/null; then
    echo "⚠️ Feishu API 配额超限，切换为每日一次模式"
    # 降级: 仅在每天特定时段(如 09:00)发送，其余时间静默
    hour=$(date +%H)
    if [ "$hour" != "09" ]; then
      echo "⏭️ 非每日汇报时段，跳过通知"
      exit 0
    fi
  fi
fi
```
