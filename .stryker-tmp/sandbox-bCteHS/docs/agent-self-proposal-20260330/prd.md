| S1.2 | 现有代码状态审计 | 审计现有 canvas 代码，识别所有状态冲突点 | `grep` 找到的冲突点 ≤ 3 个 | P0 |
| S1.3 | 冲突修复实施 | 修复识别出的状态冲突 | 修复后无 checkbox/selection 相关 bug | P0 |

## 3. UI/UX 流程

### 分析报告工作流（优化后）

```
Analyst 创建分析
    ↓
按模板结构填写（≤100行）
    ↓
PM/Architect 评审
    ↓
如 >120 行，返回重写
```

### 状态管理规范流程

```
代码变更提交
    ↓
Code Review 检查清单验证
    ↓
状态分层检查
    ↓
如有问题，驳回
```
