# VIBEX-006 P0 首页 UI/UX 重构需求分析报告

**Bug ID**: VIBEX-006
**严重级**: P0
**日期**: 2026-03-12
**分析师**: Analyst Agent

---

## 执行摘要

当前首页采用三栏布局，但**点击"开始设计"后跳转到 `/confirm`**，用户体验不连贯。推荐重构为**单页式五步流程**，在同一页面完成需求→上下文→模型→流程→UI 全流程。已有 `confirmationStore` 支持状态管理和持久化。

### 4.2 持久化策略

| 策略 | 说明 | 实现 |
|------|------|------|
| **localStorage 持久化** | 浏览器刷新不丢失 | ✅ 已实现 (Zustand persist) |

| **草稿自动保存** | 防止意外丢失 | ⚠️ 需实现 |

## 6. 技术风险分析

### 6.1 风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| **状态管理复杂度增加** | 高 | 中 | 使用 Zustand 分片，保持职责单一 |
| **Mermaid 渲染性能** | 中 | 中 | 懒加载 Mermaid，缓存渲染结果 |
| **大量状态持久化** | 中 | 低 | 只持久化必要字段，压缩数据 |
| **AI 响应延迟** | 高 | 高 | 显示进度条，支持取消操作 |
| **移动端适配** | 中 | 中 | 响应式布局，简化移动端 UI |

### 6.2 依赖风险

| 依赖 | 风险 | 替代方案 |
|------|------|----------|
| Mermaid.js | 大体积 (~1MB) | 动态导入、CDN 加载 |
| Zustand persist | localStorage 限制 | IndexedDB、后端存储 |
