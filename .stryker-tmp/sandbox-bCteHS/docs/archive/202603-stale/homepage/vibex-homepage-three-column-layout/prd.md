# PRD: 首页三栏布局恢复

**项目**: vibex-homepage-three-column-layout
**产品经理**: PM Agent
**日期**: 2026-03-17
**版本**: 1.0
**状态**: Draft

---

## 1. 执行摘要

### 1.1 背景

当前首页布局为垂直两栏 (60%+40%)，需要恢复为三栏布局以提升用户体验：
- 用户期望在单屏内看到完整的步骤指示器、预览区域和 AI 分析面板
- 三栏布局可提升信息密度和操作效率

### 1.2 目标

| 检查项 | 当前值 | 目标值 |
|--------|--------|--------|
| 布局结构 | 垂直两栏 (60%+40%) | 三栏 (15%+60%+25%) |
| 步骤指示器 | 集成在 InputArea | 独立左侧栏 |
| AI 分析面板 | 不存在 | 右侧独立栏 |
| 单屏展示 | 不完整 | 完整展示 |

### 1.3 预期收益

- 用户可在单屏查看完整流程状态
- AI 分析过程实时可见
- 步骤切换更加直观

---

## 2. 问题陈述

### 2.1 核心痛点

| 问题 | 当前状态 | 影响 |
|------|----------|------|
| 布局过窄 | 垂直两栏挤压内容 | 信息密度低 |
| 步骤指示器隐藏 | 集成在 InputArea | 用户难发现 |
| AI 分析不可见 | 无独立面板 | 无法实时查看 |

### 2.2 根因分析

| 环节 | 问题 | 影响 | 解决方案 |
|------|------|------|----------|
| HomePage | 垂直两栏布局 | 空间利用不足 | 改用三栏布局 |
| InputArea | 包含步骤指示器 | 布局耦合 | 移除到 Sidebar |
| 缺少 AIPanel | 未集成 | AI 过程不可见 | 添加右侧栏 |

## 3. 功能需求

### F1: 三栏布局实现

**描述**: 将首页从两栏改为三栏布局

**验收标准**:
- [ ] AC-F1.1: 左栏宽度为 15% (Sidebar - 步骤指示器)
- [ ] AC-F1.2: 中栏宽度为 60% (Preview + Input)
- [ ] AC-F1.3: 右栏宽度为 25% (AIPanel - AI 分析)
- [ ] AC-F1.4: 整体布局为水平排列 (flex-direction: row)

### F2: 步骤指示器独立

**描述**: 将步骤指示器从 InputArea 移到左侧栏

**验收标准**:
- [ ] AC-F2.1: 步骤指示器显示在左栏 Sidebar
- [ ] AC-F2.2: 点击步骤可切换当前步骤
- [ ] AC-F2.3: 已完成步骤有视觉区分
- [ ] AC-F2.4: 当前步骤高亮显示

### F3: AI 分析面板集成

**描述**: 在右侧栏集成 AIPanel 展示 AI 分析过程

**验收标准**:
- [ ] AC-F3.1: AIPanel 显示在右栏 (25%)
- [ ] AC-F3.2: 实时显示 AI 思考过程 (ThinkingPanel)
- [ ] AC-F3.3: 支持折叠/展开

### F4: 响应式适配

**描述**: 确保布局在不同屏幕尺寸下正常显示

**验收标准**:
- [ ] AC-F4.1: 最小宽度 1024px 时三栏正常显示
- [ ] AC-F4.2: 窗口高度变化时内容自适应
- [ ] AC-F4.3: 无横向滚动条

### F5: 回归测试

**描述**: 确保现有功能不受影响

**验收标准**:
- [ ] AC-F5.1: 图表生成功能正常
- [ ] AC-F5.2: 需求录入功能正常
- [ ] AC-F5.3: 导航功能正常
- [ ] AC-F5.4: 登录功能正常
