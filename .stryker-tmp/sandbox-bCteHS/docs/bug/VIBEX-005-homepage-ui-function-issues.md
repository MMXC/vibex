# Bug 提案: 首页 UI 和功能问题

**Bug ID**: VIBEX-005  
**严重级**: P1  
**状态**: Open  
**发现日期**: 2026-03-12  
**发现者**: 小羊  

---

## 1. 问题列表

### 1.1 Hero 区域布局问题
- **问题**: Hero 不是单行居中主副标题形式，而是靠左
- **预期**: 主副标题应水平居中显示

### 1.2 登录后状态丢失
- **问题**: 登录成功后用户输入被刷新清空
- **预期**: 登录成功后保留用户已输入的内容

### 1.3 模板功能无响应
- **问题**: 点击"使用模板"无任何响应
- **预期**: 点击模板应有响应（填入需求或跳转）

### 1.4 生成流程不符合预期
- **问题**: 点击"开始生成"仍跳转到旧的确认页
- **预期**: 
  - 首页左侧显示步骤导航
  - 直接点亮下一步骤
  - 提供单页调整/撤销/预览
  - 预览内容用 Mermaid 实时渲染：
    - 上下文图
    - 模型图
    - 流程图
    - UI节点图

---

## 2. 核心需求

**单页式生成流程**:
1. 左侧：步骤导航（步骤1 → 步骤2 → 步骤3 → 步骤4）
2. 中间：实时预览区域（Mermaid 渲染）
3. 右侧/底部：当前步骤编辑区域
4. 支持：调整、撤销、预览

---

## 3. 验收标准

| 验收项 | 描述 |
|--------|------|
| Hero 居中 | 主副标题水平居中显示 |
| 状态保持 | 登录后保留用户输入内容 |
| 模板可用 | 点击模板有响应 |
| 单页流程 | 首页完成：步骤导航 + 实时预览 + 编辑 |
| Mermaid 预览 | 上下文图、模型图、流程图、UI节点图实时渲染 |
