# PRD 版本控制规范

> PRD 文档版本管理最佳实践 | 版本: 1.0 | 更新日期: 2026-03-17

---

## 1. 版本号规范

### 格式: MAJOR.MINOR.PATCH

| 层级 | 变更类型 | 示例 |
|------|----------|------|
| MAJOR | 重大变更 (架构调整、核心功能变化) | 1.0 → 2.0 |
| MINOR | 功能新增 (向后兼容) | 1.0 → 1.1 |
| PATCH | 错误修复 (向后兼容) | 1.0 → 1.1 |

---

## 2. 版本历史表格

每个 PRD 必须包含版本历史:

```markdown
## 版本历史

| 版本 | 日期 | 变更内容 | 作者 |
|------|------|----------|------|
| 1.0 | 2026-03-17 | 初始版本 | PM |
| 1.1 | 2026-03-20 | 新增 F2 功能需求 | PM |
| 1.2 | 2026-03-22 | 修正 AC1.3 验收条件 | PM |
```

---

## 3. 变更记录要求

### 必须记录

- 📝 变更日期
- 👤 变更作者
- 📋 变更内容描述

### 建议记录

- 🔗 相关 Issue/Ticket
- ✅ 变更审批人

---

## 4. 变更类型分类

| 类型 | 说明 | 示例 |
|------|------|------|
| 新增 | 添加新功能 | 新增 F3 用户画像功能 |
| 修改 | 调整现有功能 | 修改 AC1.2 验收条件 |
| 删除 | 移除功能 | 移除 F2 旧版支付方式 |
| 修复 | 修正错误 | 修复根因分析错误 |

---

## 5. Git 管理建议

### 分支策略

```
main
├── docs/prd/feature-a-prd.md (当前版本)
└── docs/prd/archived/feature-a-prd-v1.0.md (历史版本)
```

### 提交信息规范

```bash
feat(prd): 添加 F3 功能需求 v1.1
fix(prd): 修正 AC1.2 验收条件 v1.2
```

---

## 6. 审查流程

### 版本发布检查

- [ ] 变更内容已记录在版本历史
- [ ] 相关人员已通知
- [ ] 文档格式检查通过
