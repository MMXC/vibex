## 1. 问题陈述

### 1.1 核心需求

| 维度 | 当前状态 | 目标状态 |
|------|---------|---------|
| 首页画布布局 | Grid/卡片流布局 | 卡片树（Card Tree）布局 |
| 数据来源 | 静态/本地数据 | 后端新 API |
| 交互模式 | 静态展示 + 基础交互 | 支持树形结构导航 |

### 1.2 业务背景

VibeX 首页作为用户入口，需要展示项目/模板等数据的层次关系。卡片树设计可更清晰地表达父子层级关系。

## 3. 技术方案

### 方案 A：树形卡片布局 + API Hook（推荐）

**实现**：
1. 创建 `useProjectTree` Hook，对接后端 `/api/projects` 或 `/api/templates`
2. 将现有 `AIResultCards` 包装为可折叠的树节点
3. 使用 CSS Grid + Flex 实现卡片树布局

```typescript
// 新 Hook
export function useProjectTree() {
  const { data, isLoading } = useSWR('/api/projects');
  return data?.tree ?? data?.flat ?? [];
}
```

**优点**：
- 改动范围可控
- 兼容现有卡片组件
- API 数据驱动

**缺点**：
- 树形布局复杂度高于网格

### 方案 B：全新设计系统

**实现**：完全重新设计首页布局，采用独立的卡片树组件库。

**优点**：设计更一致  
**缺点**：工作量大，风险高

---

## 4. 推荐方案

**选择：方案 A**（树形卡片 + API Hook）

理由：
- 增量改动，风险低
- 复用现有组件
- API 契约清晰后可快速实现
