# 安全审查清单

**项目**: VibeX  
**版本**: 1.0  
**更新日期**: 2026-03-18

---

本清单用于代码审查过程中的安全检查，确保所有安全检查项均已通过。

## 执行命令

```bash
# 依赖漏洞扫描
npm audit
npm run scan:vuln

# 密钥泄露检测
gitleaks detect --source .
```

---

## 1. 依赖安全

- [ ] 运行 `npm audit` 检查依赖漏洞
- [ ] 确认无 **Critical** 漏洞
- [ ] 确认无 **High** 漏洞
- [ ] Moderate/Low 漏洞已评估风险并记录
- [ ] 依赖版本均为最新稳定版

---

## 2. 密钥安全

- [ ] 运行 `gitleaks detect --source .` 检查硬编码密钥
- [ ] 确认 `.env` 文件在 `.gitignore` 中
- [ ] 敏感配置使用环境变量（非硬编码）
- [ ] 无 API Key、Token、密码等硬编码
- [ ] 敏感文件权限设置正确

---

## 3. 代码安全

### 输入验证
- [ ] 用户输入经过验证
- [ ] 用户输入经过清理（转义/过滤）
- [ ] 文件上传有类型和大小限制

### 常见漏洞检查
- [ ] 无 SQL 注入风险（使用参数化查询）
- [ ] 无 XSS 风险（输出编码）
- [ ] 无 CSRF 风险（使用 CSRF Token）
- [ ] 无命令注入风险（避免 shell 执行）

### 权限控制
- [ ] 敏感路由有认证保护
- [ ] 权限检查在服务端执行
- [ ] 用户数据隔离正确

---

## 4. 配置安全

### 网络安全
- [ ] CORS 配置合理（限定可信域名）
- [ ] CSP (Content Security Policy) 头已配置
- [ ] 安全头已设置（X-Frame-Options, X-Content-Type-Options）

### 环境配置
- [ ] 生产环境禁用调试模式
- [ ] 错误信息不暴露敏感细节
- [ ] 敏感配置从环境变量读取
