## 2. 技术方案

### 2.1 现有机制分析
基于 agent-self-evolution 项目历史实现：
- 心跳机制：各Agent通过HEARTBEAT.md执行自检
- 提案存储：proposals/YYYYMMDD/{agent}-proposals.md
- 协调流程：coord统一管理项目生命周期

### 2.2 推荐方案

#### 方案A：扩展心跳协调（推荐）
```
1. coord心跳触发 → 创建 agent-proposal-YYYYMMDD 项目
2. 并行唤醒6个Agent → 各Agent执行自检并提交提案
3. 提案存储到 proposals/YYYYMMDD/ 目录
4. coord收集汇总 → 生成当日提案报告
```

**优点**：复用现有心跳机制，改动最小  
**缺点**：依赖各Agent按时响应

#### 方案B：独立提案收集服务
```
1. 新增 propose-manager 服务
2. 各Agent通过API提交提案
3. 统一存储和展示
```

**优点**：解耦，独立性更强  
**缺点**：需要新增服务，改动大
