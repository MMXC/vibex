# Product Requirements Document: Agent 每日自检系统 v2.0

> **项目**: agent-self-evolution-YYYYMMDD
> **版本**: 2.0（标准化 + 闭环追踪）
> **日期**: 2026-03-20
> **PM**: PM Agent
> **目标**: 将现有自检系统从"提案收集"升级为"提案闭环追踪"

---

## 1. Problem Statement

当前 Agent 每日自检系统存在三个核心问题：

1. **提案质量参差**：各 Agent 提案格式不一，难以横向比较和汇总
2. **闭环缺失**：提案产出后无人追踪落地，转化率不可见
3. **效果黑盒**：无法量化"自我进化"是否真的带来系统改进

**目标用户**：OpenClaw 多 Agent 协作团队（dev, analyst, architect, pm, tester, reviewer）

## 4. Epics Breakdown

### Epic 1: 每日自检任务执行
**目标**: 确保每个 Agent 稳定产出高质量提案

- **Story 1.1**: Agent 领取自检任务 → 自检 → 产出提案 → 更新任务状态
- **Story 1.2**: Coord 检测遗漏 Agent，通过 Slack + sessions_send 双重唤醒
- **Story 1.3**: 提供标准化提案模板，降低写作门槛

### Epic 2: 提案汇总与优先级排序
**目标**: 将分散的提案结构化、可比较、可决策

- **Story 2.1**: Coord 自动汇总所有提案到 `proposals/YYYYMMDD/` 目录
- **Story 2.2**: Analyst 对提案进行可行性评估（P0/P1/P2）
- **Story 2.3**: PM 最终确认优先级，输出 Sprint 路线图建议

### Epic 3: 提案闭环执行
**目标**: 确保高优先级提案转化为可追踪的开发任务

- **Story 3.1**: PM 将 P0/P1 提案拆解为 team-tasks
- **Story 3.2**: Dev 领取任务、执行开发
- **Story 3.3**: Reviewer + Tester 验证产出
- **Story 3.4**: 提案状态更新为"已闭环"或"已驳回"（含原因）

### Epic 4: 进化效果追踪
**目标**: 量化自我进化机制的价值

- **Story 4.1**: 每批次完成后自动统计：提案数、质量评分、转化率
- **Story 4.2**: 生成历史趋势报告（7天滚动）
- **Story 4.3**: 异常告警：连续2批无提案的 Agent 自动提醒

## 5. Non-Functional Requirements

| 维度 | 要求 |
|------|------|
| **自动化** | 自检任务创建、Agent唤醒、提案汇总全部自动化，无需人工干预 |
| **可靠性** | 单个Agent失败不影响其他Agent和整体批次 |
| **可观测性** | 每批次有完整的状态记录和统计报告 |
| **可扩展性** | 支持新增Agent类型而不改动核心流程 |
| **响应时间** | 心跳扫描 < 30s，提案汇总 < 60s |
| **存储** | 提案文件保留90天，超过后归档或删除 |

## 6. Out of Scope

- AI 自动生成提案（仅辅助格式，不替代人工思考）
- 跨团队/跨系统集成
- 实时 Dashboard（非Web界面）
- 提案自动驳回/合并
