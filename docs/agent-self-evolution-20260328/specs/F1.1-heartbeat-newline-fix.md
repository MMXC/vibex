# Spec: F1.1 — HEARTBEAT.md 换行符修复

## 问题描述
Python 脚本使用 `str.replace(old, "xxx\n| ...")` 时，字符串字面量中的 `\n` 已经是真实换行符，无需二次转义。
但若使用 `content.replace(old, "xxx\\n| ...")`（双反斜杠），则会将字面 `\n` 写入文件。

## 修复方案
在所有 `HEARTBEAT.md` 写入逻辑中：
1. 确认 Python 字符串中 `\n` 是单反斜杠（真实换行）
2. 禁止使用 `\\n` 作为字面字符串
3. 使用 `textwrap.dedent` 或三引号处理多行模板

## 实现检查
```python
# ❌ 错误
content = content.replace(old, "xxx\\n| col1 | col2 |")

# ✅ 正确
content = content.replace(old, "xxx\n| col1 | col2 |")
# 或
content = content.replace(old, f"xxx\n{new_row}")
```

## 验证
```bash
grep -n '\\\\n' HEARTBEAT.md
# 期望: 无输出（0 行匹配）
```
