## 开发规范

### 强制规则

1. **禁止破坏性变更**
   - `pageName` 必须为可选字段 (`?`)
   - 现有 `flowId` 分组逻辑不变
   - `getPageLabel` 调用处需兼容新签名

2. **TypeScript Strict Mode**
   - 零新增编译错误
   - 新字段必须声明类型
   - 禁止 `any` 类型滥用

3. **测试覆盖率**
   - `getPageLabel` + `groupByFlowId` ≥ 90% 分支覆盖
   - 新增逻辑必须有对应测试用例

4. **向后兼容**
   - 无 `pageName` 的节点不受影响
   - 通用组件组行为不变

