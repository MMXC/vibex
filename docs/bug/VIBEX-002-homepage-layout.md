## 1. 问题描述

当前首页和生成流程需要优化，用户体验不佳。

### 设计目标
把之前的步骤确认功能集成到单一界面，加上实时预览，提供流畅的用户体验。

### 建议布局
1. **顶部**: 产品一句话功能说明
2. **中部**: 准实时渲染下一步产出的图形（预览区域）
3. **底部**: 需求输入框

### 集成步骤确认
- 把 /confirm/context → /confirm/model → /confirm/flow → /confirm/success 整合到单一界面
- 用户在一个页面完成：输入需求 → 实时预览 → 确认 → 生成
- 减少页面跳转，提升用户体验

## 2. PRD 依据

- AI 对话应伴随用户所有操作
- 用户可随时通过自然对话调整不满意区域
