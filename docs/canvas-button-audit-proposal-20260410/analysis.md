## 二、问题汇总

### 问题 1：TreeToolbar 全选/取消/清空按钮功能不清晰

- **位置**：Context / Flow / Component 三树工具栏
- **现象**：每个树工具栏都有"全选""取消""清空"，但节点是通过 Tab 切换激活的，不是选中的
- **用户困惑**："选中"和"激活"概念混淆
- **建议**：根据实际用户行为决定保留哪些，或改为"激活/取消激活"语义

### 问题 2：清空操作无二次确认

- **位置**：C3, F3, G3（各树的清空按钮）
- **现象**：点击"清空"直接清空节点，无 confirm 对话框
- **风险**：用户误操作导致工作丢失，且 undo 无法覆盖清空操作

### 问题 3：Flow 树删除无 undo 支持

- **位置**：F4（Flow 树删除按钮）
- **现象**：删除 Flow 节点后，undo 栈无法恢复
- **根因**：删除操作未记录到 historySlice

### 问题 4：重新生成按钮语义不清

- **位置**：C5（Context 树的"🔄 重新生成"）
- **现象**：按钮 label 是"重新生成"，但触发的是 `handleRegenerateContexts`，需要非空 requirementText
- **建议**：根据上下文显示不同状态文案

### 问题 5：Context 树的重置按钮

- **位置**：C6（Context 树的"↺ 重置"）
- **现象**：功能不明——是清空所有节点还是重置缩放？
- **建议**：明确重置语义，或移除

### 问题 6：ProjectBar 按钮过于拥挤

- **现象**：12 个按钮挤在同一行，在小屏幕上可能溢出
- **建议**：非核心操作收进下拉菜单（如历史、快捷键帮助）
