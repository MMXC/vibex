# PRD: Canvas 按钮审查与清理

> **项目**: vibex-canvas-button-audit-proposal
> **版本**: 1.0
> **日期**: 2026-04-10
> **角色**: PM
> **输入**: final-analysis.md（Analyst 需求澄清报告）

---

## 一、背景与目标

### 1.1 背景

Canvas 页（/canvas）经多 Epic 迭代后存在按钮功能重叠、UX 不一致、冗余交互问题。当前共有约 35 个按钮分布在 5 个区域，其中 6 项核心问题经 Analyst 识别需要修复。

### 1.2 目标

1. 统一三树（Flow / Component / BoundedContext）的按钮语义与交互模型
2. 消除高危操作的误操作风险
3. 为 Flow 树补充 undo 支持
4. 改善 ProjectBar 按钮过密问题
5. 建立可度量的清理结果

## 三、用户故事

### 故事 1：安全删除 Flow 节点

**为**: Flow 树用户
**我想要**: 删除节点后能通过 Ctrl+Z 撤销
**以便于**: 误删时快速恢复，不丢失工作

**验收标准**:
- [ ] 删除任意 Flow 节点后，Ctrl+Z 可撤销
- [ ] 撤销后节点及其连接关系完整恢复
- [ ] Flow 树和 Component 树行为一致

---

### 故事 2：语义明确的批量选择操作

**为**: Canvas 用户
**我想要**: 全选、取消、清空按钮的行为清晰可预测
**以便于**: 不会因语义混淆导致误操作

**验收标准**:
- [ ] "全选"按钮只选中文本节点，不激活其他节点
- [ ] "取消"按钮清除选中状态，不清除节点
- [ ] "清空"按钮有明确的中文标签，显示在按钮上
- [ ] 三树使用统一的术语和图标

---

### 故事 3：清空操作有确认保护

**为**: Canvas 用户
**我想要**: 清空节点前有二次确认
**以便于**: 避免误点导致大量节点被清除

**验收标准**:
- [ ] 点击"清空"按钮后弹出确认对话框
- [ ] 确认对话框显示将清除的节点数量
- [ ] 确认后才执行清空，取消则保持原状

---

### 故事 4：重新生成按钮语义清晰

**为**: Canvas 用户
**我想要**: 知道点击"重新生成"会发生什么
**以便于**: 避免数据意外丢失

**验收标准**:
- [ ] 按钮文案准确描述实际行为
- [ ] 点击前显示确认（如果涉及数据覆盖）
- [ ] 重新生成后原内容可通过 undo 恢复

---

### 故事 5：Context 树重置功能明确

**为**: Canvas 用户
**我想要**: 知道"重置"按钮的具体作用
**以便于**: 不会因为功能模糊而不敢使用

**验收标准**:
- [ ] 重置按钮有明确的 tooltip 说明功能
- [ ] 重置行为记录到 historySlice
- [ ] 或确认该功能无意义后删除该按钮

---

### 故事 6：ProjectBar 按钮组织清晰

**为**: Canvas 用户
**我想要**: ProjectBar 按钮不过于拥挤，核心操作易发现
**以便于**: 提高操作效率

**验收标准**:
- [ ] ProjectBar 核心按钮（≤5个）直接显示
- [ ] 次要按钮收拢到下拉菜单
- [ ] 收拢后有明确的入口标识
