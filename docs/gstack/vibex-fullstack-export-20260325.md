## 核心设计原则

**画布是主操作区，一切围绕画布展开。**

## UI 架构

```
┌─────────────────────────────────────────────────────────┐
│  顶部工具栏: 项目名 | 保存 | 导出 | 设置 | 全屏切换      │
├───────────────────────────────────────┬─────────────────┤
│                                       │                 │
│           全屏卡片树画布               │   AI 助手面板   │
│     (Canvas - 主要操作区)              │   (可收起)      │
│                                       │                 │
```

## 四步 Pipeline（卡片形式）

### Step 1: 需求卡片（Root Node）
- **外观**: 顶部大卡片，带标题、描述、标签
- **操作**: 编辑需求描述 → AI 自动生成流程节点
- **Checkpoint**: 确认需求描述后，展开下一层

### Step 2: 流程节点卡片（Flow Node Cards）
- **外观**: 子节点卡片，带流程名称、描述、连接线
- **操作**: 勾选确认/编辑/删除/新增
- **Checkpoint**: 每个流程节点确认后，展开组件节点
- **可视化**: 卡片树结构，连接线显示父子关系

### Step 3: 组件节点卡片（Component Node Cards）
- **外观**: 叶子节点卡片，带组件名、属性、API 定义
- **操作**: 勾选确认/编辑/删除/新增
- **Checkpoint**: 所有组件节点确认后，生成原型
- **JSON Schema**:
```json
{
  "componentId": "string",
  "name": "string",
  "type": "page|form|list|detail|modal",
  "props": {},
  "api": { "method": "GET|POST", "path": "/api/...", "params": [] },
  "children": ["componentId"]
}
```

### Step 4: 原型页面卡片（Prototype Card）
- **外观**: 可交互的原型预览卡片
- **操作**: 预览/编辑组件/导出
- **导出触发**: 确认组件节点 JSON → 生成原型

### Step 5: 导出（Export）
- **产物**: Next.js 项目（zip 下载）
- **包含**:
  - `src/app/page.tsx` — 路由页面
  - `src/types/components.ts` — 组件类型定义
  - `src/api/*.ts` — API mock 文件
  - `next.config.js` — Next.js 配置

## Checkpoint 机制

每步生成后：
1. 卡片以"待确认"状态显示（黄色边框）
2. 人类勾选确认 → 卡片变为绿色
3. 确认后解锁下一步生成
4. **不允许跳步** — 必须逐级确认

## 卡片交互状态

| 状态 | 边框颜色 | 操作 |
|------|----------|------|
| 待生成 | 灰色虚线 | — |
| 待确认 | 黄色实线 | 勾选/编辑/删除 |
| 已确认 | 绿色实线 | 查看/重新生成 |
| 错误 | 红色实线 | 编辑修复 |

## 与现有 UI 的区别

| 维度 | 现有 UI | 新设计 |
|------|---------|--------|
| 主操作区 | 右侧 AI 面板 | 全屏卡片树画布 |
| 节点关系 | 文字描述 | 可视化卡片+连接线 |
| 确认方式 | 按钮点击 | 勾选+即时保存 |
| 流程可见性 | 单步显示 | 树状全局视图 |

## MVP 验证目标

1. 需求录入 → 流程节点（卡片+确认） → 组件节点（卡片+确认） → 原型预览 → 导出 Next.js
2. 全流程可走通，无跳步
3. 导出的项目可直接运行
