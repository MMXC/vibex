# Design: VibeX 全栈导出模式 — 三树并行卡片画布

Generated by /office-hours on 2026-03-25
Status: DRAFT
Founder: Alex Yu / 小羊

## 核心设计原则

**画布是主操作区，一切围绕画布展开。**

**交互模型**：全流程可逆，每步可返回修改，重新生成级联下游更新。

## UI 架构 — 三树并行展示

单页内三个卡片树**同时存在**，根据阶段激活不同层。

```
┌──────────────────────────────────────────────────────────────┐
│  顶部工具栏: 项目名 | 保存 | 导出 | 全屏                      │
├────────────────┬────────────────┬─────────────────────────┤
│ 限界上下文树      │ 业务流程树        │ 组件树                   │
│ (需求澄清阶段)    │ (流程确认阶段)    │ (组件确认阶段)            │
│                  │                  │                          │
│ ☑ 用户上下文     │  □ 登录流程     │  □ 登录页卡片             │
│   □ 订单上下文  →  ☑ 注册流程    →  ☑ 注册页卡片             │
│   □ 内容上下文  │  □ 内容管理      │  □ 内容页卡片              │
│                  │                  │  □ 订单页卡片              │
│ [编辑] [确认]    │ [编辑] [确认]    │ [编辑] [确认]             │
├────────────────┴────────────────┴─────────────────────────┤
│  阶段进度: [需求澄清] ──→ [业务流程] ──→ [组件] ──→ [原型]  │
└──────────────────────────────────────────────────────────────┘
```

**阶段激活规则**:
- **需求澄清阶段**: 限界上下文树激活（高亮），可编辑/确认；其他两树暗淡/折叠
- **流程确认阶段**: 业务流程树激活；上下文树锁定；组件树暗淡/折叠
- **组件确认阶段**: 组件树激活；上下文+流程树锁定
- **原型阶段**: 组件树锁定，原型预览展示，导出可用

**可逆规则**:
- 任意阶段可点击"编辑"解锁 → 重新生成该层卡片树
- 重新生成会**级联更新下游**（修改上下文 → 重新生成流程 → 重新生成组件）
- 上游重新生成后，下游需重新确认

## 数据结构

### 限界上下文节点
```json
{
  "contextId": "string",
  "name": "string",
  "description": "string",
  "bounded": true,
  "confirmed": false
}
```

### 业务流程节点
```json
{
  "flowId": "string",
  "contextId": "string",
  "name": "string",
  "steps": [{ "stepId": "string", "name": "string", "actor": "string" }],
  "confirmed": false
}
```

### 组件节点（页面为卡片）
```json
{
  "componentId": "string",
  "flowId": "string",
  "name": "string",
  "type": "page|form|list|detail|modal",
  "props": {},
  "api": { "method": "GET|POST", "path": "/api/...", "params": [] },
  "children": ["componentId"],
  "confirmed": false
}
```

## Checkpoint 机制

- 每步生成后：卡片以"待确认"状态显示（黄色边框）
- 人类勾选确认 → 卡片变为绿色 ✓
- 确认后解锁下一步生成
- 不允许跳步 — 必须逐级确认

## 导出（Export）

- **触发**: 组件树全部确认后，导出按钮可用
- **产物**: Next.js 项目（zip 下载）
- **包含**:
  - `src/app/page.tsx` — 路由页面
  - `src/types/components.ts` — 组件类型定义
  - `src/api/*.ts` — API mock 文件
  - `next.config.js` — Next.js 配置

## MVP 验证目标

1. 需求录入 → 限界上下文树（确认） → 业务流程树（确认） → 组件树（确认） → 原型 + 导出
2. 全流程可逆，可返回重新生成
3. 导出的项目可直接运行
