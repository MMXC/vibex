## How to Migrate a Route

### Step 1: Create App Router Equivalent

Create a new file in `app/api/[resource]/route.ts`:

```typescript
import { NextRequest, NextResponse } from 'next/server';
import { verifyToken } from '@/lib/auth';

export async function GET(request: NextRequest) {
  const auth = await verifyToken(request);
  if (!auth.success) {
    return NextResponse.json({ error: 'Unauthorized', code: 'AUTH_REQUIRED' }, { status: 401 });
  }
  // ... handler logic
}
```

### Step 2: Verify Equivalence

- Same HTTP methods (GET/POST/PUT/DELETE)
- Same request/response shapes
- Same error codes
- Authentication flow matches

### Step 3: Test with gstack

```bash
# Verify new route returns 200 with auth
curl -H "Authorization: Bearer <token>" http://localhost:3000/api/resource

# Verify deprecated route still works (during transition)
curl http://localhost:3000/api/resource
```

### Step 4: Remove Legacy Route

After all consumers are migrated:

```bash
# Delete after E1 completion
rm vibex-backend/src/routes/[resource].ts
```
