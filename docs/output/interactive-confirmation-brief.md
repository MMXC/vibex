# Product Brief: 交互式需求确认流程

## 1. Problem Statement (核心痛点)

### 用户痛点
1. **需求理解偏差**: 用户描述需求后，AI 直接生成结果，缺少中间确认环节，导致理解偏差累积
2. **无法回退修正**: 当前流程是单向的，用户发现某一步有问题，需要从头开始
3. **编辑方式单一**: 只能通过自然语言描述修改，缺少图形化拖拽、代码直接编辑等直观方式
4. **模型难以复用**: 生成的 DDD 模型无法结构化保存，每次都需要重新生成

### 业务痛点
- 返工成本高：后期发现需求理解错误，修改代价大
- 用户信任度低：黑盒式生成，用户对结果缺乏掌控感
- 交付质量不稳定：依赖 AI 单次输出质量，缺少迭代校验

---

## 2. Proposed Solution (解决方案)

### 核心设计理念
**"渐进式确认，可回退修正"** — 将需求确认拆解为三步可交互流程，每一步都允许用户以多种方式编辑和确认。

### 三种编辑方式

| 编辑方式 | 适用场景 | 实现方案 |
|---------|---------|---------|
| **图形化编辑** | 可视化拖拽调整节点、连线 | React Flow 渲染 Mermaid 图 |
| **代码编辑** | 精确控制、批量修改 | Monaco Editor + Mermaid 语法 |
| **AI 对话** | 自然语言描述修改意图 | 调用 AI 解析并更新图表 |

## 4. Core User Stories (核心用户故事)

### US-1: 三步确认需求

> 作为产品经理，我希望描述需求后，系统依次展示限界上下文图、领域模型类图、业务流程图，每步都能确认或修改，最终生成完整的项目结构。

**验收标准:**
- Step 1 展示限界上下文图，用户可确认或修改
- Step 2 基于选定的上下文展示领域模型类图
- Step 3 基于领域模型展示业务流程图
- 三步确认后自动创建项目

### US-2: 图形化编辑图表

> 作为用户，我希望通过拖拽节点、调整连线来直观地修改图表，而不是重新描述。

**验收标准:**
- Mermaid 图表渲染为可交互的 React Flow 图
- 节点可拖拽移动位置
- 连线可添加/删除/修改
- 修改后自动同步到 Mermaid 代码

### US-3: 代码编辑图表

> 作为开发者，我希望直接编辑 Mermaid 代码来精确控制图表，修改后实时预览效果。

**验收标准:**
- Monaco Editor 集成，支持 Mermaid 语法高亮
- 实时预览渲染效果
- 语法错误提示
- 代码变更自动同步到图形化视图

### US-4: AI 对话编辑

> 作为用户，我希望用自然语言描述修改意图，AI 自动理解并更新图表。

**验收标准:**
- 输入框支持自然语言描述
- AI 解析意图并生成 Mermaid 代码变更
- 变更后图形化/代码视图同步更新
- 支持多轮对话，上下文保持

### US-5: 回退修改

> 作为用户，我希望在 Step 2 发现 Step 1 有问题时，能回退到 Step 1 修改，后续步骤自动更新。

**验收标准:**
- 每步都有"上一步"按钮
- 回退后自动加载已确认的内容
- 修改后触发下游步骤的重新生成提示
- 用户可选择重新生成或保留现有内容

### US-6: 项目自动生成

> 作为用户，我希望三步确认完成后，系统自动创建项目并生成 UI 原型，无需手动搭建。

**验收标准:**
- 自动创建项目记录
- 三图数据结构化存储到数据库
- 根据领域模型生成页面骨架（实体 → 页面列表）
- 根据业务流程生成交互逻辑（状态 → 组件状态）
- 跳转到项目详情页查看结果

## 5. 技术方案

### 5.1 核心技术栈

| 技术 | 用途 | 理由 |
|------|------|------|
| Mermaid.js | 图表渲染 | 用户模板要求，语法简洁 |
| React Flow | 图形化编辑 | 已集成，支持拖拽交互 |
| Monaco Editor | 代码编辑 | VS Code 同款，语法高亮 |
| Zustand | 状态管理 | 已使用，轻量高效 |

### 5.2 数据结构设计

```typescript
// 确认流程状态
interface ConfirmationFlow {
  projectId: string;
  requirementText: string;
  currentStep: 1 | 2 | 3;
  
  // Step 1: 限界上下文
  boundedContext: {
    mermaidCode: string;
    contexts: BoundedContext[];
    relationships: ContextRelationship[];
  };
  
  // Step 2: 领域模型
  domainModel: {
    selectedContext: string; // 从 Step 1 选择
    mermaidCode: string;
    aggregates: Aggregate[];
    entities: Entity[];
    valueObjects: ValueObject[];
  };
  
  // Step 3: 业务流程
  businessFlow: {
    mermaidCode: string;
    states: State[];
    transitions: Transition[];
    events: DomainEvent[];
  };
  
  // 元数据
  createdAt: string;
  updatedAt: string;
}

// 限界上下文
interface BoundedContext {
  id: string;
  name: string;
  type: 'core' | 'supporting' | 'generic';
  responsibility: string;
  keyEntities: string[];
}

// 聚合
interface Aggregate {
  id: string;
  name: string;
  root: Entity;
  entities: Entity[];
  valueObjects: ValueObject[];
}

// 业务状态
interface State {
  id: string;
  name: string;
  type: 'initial' | 'intermediate' | 'final';
  description: string;
}
```

### 5.4 Mermaid → React Flow 转换

```typescript
// 将 Mermaid 代码解析为 React Flow 节点和边
function mermaidToReactFlow(mermaidCode: string): {
  nodes: Node[];
  edges: Edge[];
} {
  // 使用 mermaid.js 解析器
  const diagram = mermaid.parse(mermaidCode);
  
  // 转换为 React Flow 格式
  const nodes: Node[] = diagram.nodes.map(node => ({
    id: node.id,
    type: 'custom',
    position: { x: node.x, y: node.y },
    data: { label: node.label }
  }));
  
  const edges: Edge[] = diagram.edges.map(edge => ({
    id: `${edge.source}-${edge.target}`,
    source: edge.source,
    target: edge.target,
    label: edge.label
  }));
  
  return { nodes, edges };
}
```

## 6. 约束条件

### 必须遵守
- **基于用户模板设计**: 三类图的生成规则严格遵循 DDD 模板
- **三种编辑方式可切换**: 图形化/代码/AI对话三种方式在任意步骤都可用
- **确认流程有回退能力**: 每步都可以返回上一步修改
- **输出结构化图数据**: 最终保存的是结构化的图数据，而非纯文本

### 技术约束
- 复用现有 React Flow 集成
- 复用现有 AI 对话能力
- 兼容现有项目创建流程
- 移动端响应式适配

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| AI 输出不稳定 | 图表结构不合理 | 提供示例模板，引导用户补充关键信息 |
| Mermaid 解析复杂 | 转换失败率高 | 使用成熟解析库，提供降级方案（仅预览） |
| 图形化编辑状态同步 | 数据不一致 | 使用单一数据源（Mermaid 代码），双向绑定 |
| 用户理解成本高 | 不熟悉 DDD 概念 | 提供交互式引导、术语解释 |
