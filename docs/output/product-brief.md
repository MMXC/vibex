# VibeX Frontend 产品Brief

## 1. 项目概述

**项目名称**: VibeX Frontend (vibex-playground)  
**项目类型**: React Next.js 前端原型预览与测试框架  
**核心目标**: 展示 VibeX 项目需实现的前端功能，沉淀交互契约

## 2. 原型功能分析

### 2.1 页面清单

| 页面 | 路由 | 核心功能 |
|------|------|----------|
| 落地页 | `/` | 产品介绍、特性展示、CTA引导 |
| 登录注册页 | `/auth` | 用户认证（登录/注册） |
| 用户中心 | `/dashboard` | 个人项目管理、概览 |
| **AI对话页** | `/chat` | **MVP核心** - AI对话、Agent管理 |
| **流程图编辑页** | `/flow` | **MVP核心** - 流程图可视化编辑 |
| 页面列表页 | `/pages` | 项目页面管理 |
| 页面编辑页 | `/editor` | 低代码页面编辑器 |
| 原型预览页 | `/preview` | 设备模拟预览 |
| 导出页面 | `/export` | 导出项目产物 |
| 项目设置页 | `/settings/project` | 项目配置 |
| 模板市场页 | `/templates` | 模板库 |
| 用户设置页 | `/settings/user` | 用户偏好设置 |

### 2.2 控件清单

| 控件 | 功能 |
|------|------|
| Button | 多态按钮（primary/secondary/ghost/danger） |
| Card | 可悬停卡片 |

### 2.3 布局模式

- **固定头部**: 64px/56px 高度
- **侧边栏**: 200px 宽度
- **三栏布局**: 左侧组件库 + 中间画布 + 右侧属性面板
- **响应式**: 支持桌面/平板/移动设备切换

## 3. MVP 阶段需求

基于 VibeX 产品定位，MVP 阶段应聚焦：

### 3.1 核心功能（必须实现）

1. **AI对话页** (`/chat`)
   - 对话流展示
   - 用户输入
   - AI 回复渲染
   - Agent 切换/管理

2. **流程图编辑页** (`/flow`)
   - 节点库
   - 画布拖拽
   - 连线操作
   - 属性面板

3. **用户中心** (`/dashboard`)
   - 项目列表
   - 创建项目
   - 项目概览

### 3.2 支撑功能（优先实现）

- 登录注册页
- 页面列表页
- 原型预览页

### 3.3 后续功能

- 页面编辑页（低代码编辑器）
- 模板市场
- 导出功能
- 项目/用户设置

## 4. 技术约束

- **框架**: React + Next.js
- **路由**: Next.js App Router
- **样式**: CSS Modules
- **控件体系**: 控件与页面分离，支持动态加载

## 5. 产出要求

1. **交互契约文档**: 页面间跳转、数据流、事件定义
2. **控件规范**: 控件属性定义、复用机制
3. **API 契约**: 前后端接口定义（待后端补充）
