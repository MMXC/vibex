# PRD: PRD 自动化验证工具

**项目**: prd-validation-tool
**产品经理**: PM Agent
**日期**: 2026-03-18
**版本**: 1.0
**状态**: Draft

---

## 1. 执行摘要

### 1.1 背景

当前 PRD 文档缺少自动化验证机制，人工审查效率低且容易遗漏。需要构建自动化验证工具。

### 1.2 目标

| 指标 | 当前值 | 目标值 |
|------|--------|--------|
| 验证覆盖率 | 0% | 100% |
| 验证时间 | 30min | 2min |
| 错误检出率 | 手动 | 90%+ |

---

## 2. 功能需求

### F1: PRD 格式验证

**描述**: 验证 PRD 文档格式完整性

**验收标准**:
- [ ] AC-F1.1: 检查必需章节存在（目标、Epic、验收标准）
- [ ] AC-F1.2: 检查 Epic 编号连续
- [ ] AC-F1.3: 检查验收标准格式

### F2: 验收标准检查

**描述**: 验证验收标准可测试性

**验收标准**:
- [ ] AC-F2.1: 检测无可测试验收标准
- [ ] AC-F2.2: 检测缺少 DoD 定义
- [ ] AC-F2.3: 检测 Epic-Story 映射完整

### F3: 依赖检查

**描述**: 验证任务依赖关系

**验收标准**:
- [ ] AC-F3.1: 检测循环依赖
- [ ] AC-F3.2: 检测悬空依赖（依赖不存在任务）

### F4: 报告生成

**描述**: 生成验证报告

**验收标准**:
- [ ] AC-F4.1: 输出 JSON 格式报告
- [ ] AC-F4.2: 输出 Markdown 格式报告
- [ ] AC-F4.3: 支持 CI 集成

## 5. 技术方案

### 5.1 技术栈

- Node.js + TypeScript
- Cheerio (Markdown 解析)
- Ajv (JSON Schema 验证)

### 5.2 验证规则

```typescript
interface ValidationRule {
  id: string;
  name: string;
  check: (prd: PRD) => ValidationResult;
  severity: 'error' | 'warning';
}
```
