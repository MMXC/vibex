### Proposal A: 测试隔离检查清单（P1）

**问题描述**:
`homepageAPI.test.ts` 在 `beforeAll` 中设置 `global.fetch` 但从未在 `afterAll` 中恢复，导致后续测试文件（如 `theme-binding.test.tsx`）运行时调用泄漏的 fetch mock，返回 `ok:undefined` → 12 个测试失败。

**根因**: Jest 测试文件中修改全局状态后忘记恢复，是常见但容易被忽视的问题。

**建议方案**:
1. 创建 `test-quality-checklist.md`，包含：
   - [ ] 检查 `beforeAll` 中是否修改了 `global.*`
   - [ ] 检查是否有对应的 `afterAll` 恢复
   - [ ] 检查 `jest.clearAllMocks()` vs `jest.resetAllMocks()` 的选择
   - [ ] 检查 `localStorage`、`matchMedia` 等浏览器 API mock 的清理

2. 将检查清单集成到分析报告模板中（作为"代码质量"检查节）

**工作量**: 0.5 天（文档编写 + 模板更新）

**预期收益**: 
- 减少因测试隔离问题导致的重复分析
- Analyst 能在第一时间识别测试文件设计问题

**验收标准**:
- [ ] 检查清单存在于 `~/.openclaw/skills/analysis-patterns/test-quality-checklist.md`
- [ ] 在下一个测试相关分析中应用并记录效果

    dependency-conflict.md       # 依赖问题

```bash

    echo "发现未分析问题: $dir"

  fi
done
```
