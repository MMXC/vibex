# PRD: PM 自主提案收集 (self-proposal-20260330)

## 1. 执行摘要

**项目目标**: 收集 PM Agent 今日自主提案，识别可执行的自我改进点

**成功指标**:
- 产出至少 3 条具体可执行的提案
- 每条提案包含问题描述、改进方案、预期收益
- 提案存储在指定位置

**范围**:
- 产出: `docs/proposals/20260330/pm-proposals.md`
- 不涉及代码开发

---

## 2. 功能需求

### F1: 提案收集

| ID | 功能点 | 描述 | 验收标准 | 页面集成 |
|----|--------|------|----------|----------|
| F1.1 | 日常反思 | 从心跳日志、任务产出、团队反馈中识别改进点 | expect(改进点数 ≥ 3) | 否 |
| F1.2 | 提案撰写 | 按模板格式撰写每条提案 | expect(每条提案包含 problem/solution/impact) | 否 |
| F1.3 | 提案存储 | 保存到 `docs/proposals/20260330/pm-proposals.md` | expect(file_exists) | 否 |

### F2: 提案评审

| ID | 功能点 | 描述 | 验收标准 | 页面集成 |
|----|--------|------|----------|----------|
| F2.1 | 提案自审 | 检查提案完整性、可执行性 | expect(所有提案通过自审检查清单) | 否 |

## 3. Epic 拆分
