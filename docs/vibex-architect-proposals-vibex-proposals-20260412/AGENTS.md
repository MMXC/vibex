## 开发规范

### 强制规则

1. **@vibex/types 引用规则**
   - 所有新增 API 类型必须先写入 `packages/types/src/api/`
   - backend/frontend 禁止独立定义已存在于 `@vibex/types` 的类型
   - 引用前先 `pnpm install @vibex/types` 确认包可用

2. **ErrorBoundary 隔离规则**
   - 每个 TreePanel 必须独立包裹 ErrorBoundary
   - fallback UI 必须包含重试按钮
   - `onError` 必须调用 `canvasLogger.error`
   - 禁止在 ErrorBoundary 内部捕获所有错误（保留 React 错误上报）

3. **v0→v1 迁移规则**
   - 迁移后旧路由必须保留（添加 Deprecation Header）
   - frontend 调用方必须同步更新到 `/api/v1/`
   - 迁移前后均需运行 e2e 测试回归

4. **types.ts 重构规则**
   - 必须先备份 `types.ts` 再修改
   - re-export 格式: `export type { X } from '@vibex/types'`
   - 修改后必须 `pnpm tsc --noEmit` 验证

### ErrorBoundary fallback

```tsx
// ✅ 正确：完整 fallback，包含重试
<ErrorBoundary
  fallback={({ error, reset }) => (
    <div>
      <p>{error.message}</p>
      <button onClick={reset}>重试</button>
    </div>
  )}
  onError={(error) => canvasLogger.error('[Panel]', error)}
>
  <TreePanel />
</ErrorBoundary>

// ❌ 错误：空 fallback 或无 onError
<ErrorBoundary>{children}</ErrorBoundary>
```
