# PRD：Canvas 按钮文案优化

**项目**: vibex-canvas-btn-rename-20260328  
**状态**: Draft  
**PM**: pm  
**创建时间**: 2026-03-28 16:37  
**目标**: 将 Canvas 页面按钮文案「AI生成上下文」改为「重新执行」

---

## 1. 背景与目标

### 1.1 问题描述

当前 Canvas 页面存在一个功能按钮，其文案为「AI生成上下文」，该文案不符合用户心理预期：

- 用户对「AI生成上下文」功能认知模糊，无法判断点击后会发生什么
- 「重新执行」更符合用户心智：按钮点击后重新执行已有的生成逻辑

### 1.2 目标

将 Canvas 页面中所有出现「AI生成上下文」文案的按钮，替换为「重新执行」，包括空闲状态和加载状态。

---

## 2. 功能范围

### 2.1 涉及范围

| 范围 | 说明 |
|------|------|
| 目标页面 | `/canvas` 页面 |
| 目标按钮 | 包含「AI生成上下文」文本的按钮组件 |
| 按钮状态 | 空闲态文案 + 加载态文案（若有） |

### 2.2 排除范围

- 其他页面（如 `/`、`/design` 等）的按钮不受影响
- 与按钮文案无关的逻辑变更
- Zustand store 或 API 层面的修改

---

## 3. 功能点

| ID | 功能点 | 描述 | 验收标准 | 页面集成 |
|----|--------|------|----------|----------|
| F1.1 | 搜索目标按钮 | 在 `src/app/canvas/**/*.{tsx,ts}` 中全局搜索「AI生成上下文」文本，定位所有需要修改的按钮 | `expect(searchResults.length > 0).toBe(true)` 确认至少找到 1 处 | ❌ |
| F1.2 | 替换按钮文案 | 将所有「AI生成上下文」替换为「重新执行」 | `expect(buttonText).toBe('重新执行')` | 【需页面集成】 |
| F1.3 | 替换加载态文案 | 若按钮有 loading 状态文案，同步替换（如「AI生成上下文中...」→「重新执行中...」） | `expect(loadingText).toBe('重新执行中...')` | 【需页面集成】 |
| F1.4 | 验证无副作用 | 全局搜索「AI生成上下文」确认无残留；搜索「重新执行」确认未误改其他按钮 | `expect(remainingCount).toBe(0)` 确认原文案已全部替换 | ❌ |
