# Spec: F1.1 — contextsToSend 空数组前置校验

**Story ID**: F1.1
**所属 Epic**: E1
**文件**: `vibex-frontend/src/components/canvas/BusinessFlowTree.tsx`
**改动位置**: `handleContinueToComponents` 函数内部，`contextsToSend` 构建之后

---

## 现状

当前 `handleContinueToComponents` 中构建 `contextsToSend` 的逻辑：

```typescript
const activeContexts = contextNodes.filter((ctx) => ctx.isActive !== false);
const selectedContextSet = new Set(selectedNodeIds.context);
const contextsToSend = selectedContextSet.size > 0
  ? activeContexts.filter((ctx) => selectedContextSet.has(ctx.nodeId))
  : activeContexts;
```

当 `activeContexts` 为空数组时（所有节点 `isActive === false`），`contextsToSend` 也为空，API 返回 400 但 toast 不够具体。

---

## 改动规格

### 改动点 1：在 contextsToSend 构建后增加空数组校验

**位置**: `BusinessFlowTree.tsx`，`handleContinueToComponents` 函数内，`contextsToSend` 赋值语句之后

**代码**：

```typescript
// contextsToSend 构建后新增
if (contextsToSend.length === 0) {
  toast.showToast('请先确认至少一个上下文节点后再生成组件树', 'error');
  return;
}
```

### 改动点 2：toast 变量确认

**前置条件**: 确认 `toast` 变量（`useToast()`）在组件顶部已存在（analysis.md 第 724 行已确认 ✅）

## 测试场景

| # | 场景 | 前置条件 | 操作 | 预期结果 |
|---|------|---------|------|---------|
| T1 | 所有 contextNodes isActive === false | contextNodes 有数据但全 inactive | 点击"继续·组件树" | toast 显示"请先确认至少一个上下文节点后再生成组件树"，API 未调用 |
| T2 | selectedNodeIds.context 为空且无 active 节点 | contextNodes 有数据但全 inactive | 点击"继续·组件树" | 同 T1 |

| T4 | 回归：contextNodes 为空 | contextNodes === [] | 点击"继续·组件树" | 触发第 767 行的原有校验"请先生成上下文树"，本条规则不触发 |
