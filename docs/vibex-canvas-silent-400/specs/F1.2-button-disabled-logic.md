# Spec: F1.2 — 按钮 disabled 逻辑优化

**Story ID**: F1.2
**所属 Epic**: E1
**文件**: `vibex-frontend/src/components/canvas/BusinessFlowTree.tsx`
**改动位置**: "继续·组件树"按钮的 disabled 属性

---

## 现状

当前按钮 disabled 逻辑：

```tsx
{flowNodes.length > 0 && (
  <button
    disabled={componentGenerating}
    onClick={handleContinueToComponents}
  >
    继续·组件树
  </button>
)}
```

只检查 `componentGenerating` loading 锁，不检查 `contextsToSend` 是否有效。

---

## 改动规格

### 改动点：扩展按钮 disabled 判断逻辑

**原代码**：

```tsx
disabled={componentGenerating}
```

**新代码**：

```tsx
disabled={
  componentGenerating ||
  contextsToSend.length === 0
}
```

### 变量依赖说明

`contextsToSend` 的计算依赖于：
- `contextNodes`：组件状态，需确认按钮渲染时 `contextsToSend` 已在 scope 内
- `selectedNodeIds.context`：组件状态

若 `contextsToSend` 在按钮渲染时不在 scope 内，需在组件顶层计算 `isContextsToSendEmpty` 布尔值：

```tsx
// 组件顶层（按钮渲染区同一 scope）
const activeContexts = contextNodes.filter((ctx) => ctx.isActive !== false);
const selectedContextSet = new Set(selectedNodeIds.context);
const contextsToSend = selectedContextSet.size > 0
  ? activeContexts.filter((ctx) => selectedContextSet.has(ctx.nodeId))
  : activeContexts;
const isContextsEmpty = contextsToSend.length === 0;

// 按钮处
disabled={componentGenerating || isContextsEmpty}
```

**验收确认**：
- [ ] 按钮 disabled 状态与 `contextsToSend.length === 0` 同步变化
- [ ] `flowNodes.length === 0` 时按钮不渲染（保持原逻辑）
- [ ] regression：正常路径按钮可点击性不受影响
