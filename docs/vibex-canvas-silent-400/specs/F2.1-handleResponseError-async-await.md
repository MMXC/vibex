# Spec: F2.1 — handleResponseError async/await 修复

**Story ID**: F2.1
**所属 Epic**: E2
**文件**: `vibex-frontend/src/lib/canvas/api/canvasApi.ts`
**改动位置**: `handleResponseError` 函数（第 145-166 行附近）

---

## 现状（Bug 代码）

```typescript
function handleResponseError(res: Response, defaultMsg: string, returnTo?: string): never {
  // ...401, 404 handling...

  const err = res.json().catch(() => ({ error: `HTTP ${res.status}` })); // BUG: 无 await
  throw new Error((err as { error?: string }).error ?? defaultMsg); // err 是 Promise，永远是 undefined
}
```

`res.json()` 返回 `Promise<any>`，没有 `await` 导致 `err` 是 Promise 对象而非解析后的对象，`err.error` 永远为 `undefined`。

---

## 改动规格

### 改动点：增加 async/await

**原代码片段**：

```typescript
const err = res.json().catch(() => ({ error: `HTTP ${res.status}` }));
throw new Error((err as { error?: string }).error ?? defaultMsg);
```

**新代码**：

```typescript
const err = await res.json().catch(() => ({ error: `HTTP ${res.status}` }));
throw new Error((err as { error?: string }).error ?? defaultMsg);
```

### 副作用检查

由于函数签名返回 `never`，调用方需确保所有 `throw` 路径都正常：

```typescript
// 函数签名保持不变
function handleResponseError(res: Response, defaultMsg: string, returnTo?: string): never {
  // ...
  throw new Error(...); // never 到达
}
```

**验收确认**：
- [ ] `res.json()` 前有 `await` 关键字
- [ ] 函数返回类型 `never` 不变
- [ ] 其他 401/404 分支不受影响
- [ ] 其他调用 `handleResponseError` 的 API（如 fetchFlowTree）回归正常
