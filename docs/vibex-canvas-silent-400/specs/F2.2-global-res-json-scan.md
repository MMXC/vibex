# Spec: F2.2 — 全局 res.json() 调用扫描

**Story ID**: F2.2
**所属 Epic**: E2
**工作目录**: `vibex-frontend/src/`

---

## 目的

防止其他位置存在相同 `res.json()` 未加 `await` 的 bug，作为 F2.1 的推广验证。

---

## 扫描步骤

### 1. 搜索所有 res.json() 调用

```bash
grep -rn "res\.json()" vibex-frontend/src/
```

### 2. 对每处结果验证

对每条 grep 输出，检查上下 2 行内是否包含 `await`：

```bash
# 示例：验证某处 res.json() 有 await
grep -rn "res\.json()" vibex-frontend/src/ | while read line; do
  file=$(echo "$line" | cut -d: -f1)
  linenum=$(echo "$line" | cut -d: -f2)
  echo "=== $file:$linenum ==="
  sed -n "$((linenum-2)),$((linenum+2))p" "$file"
  echo ""
done
```

### 3. 分类处理

| 分类 | 处理方式 |
|------|---------|
| 有 await ✅ | 无需改动，记录在案 |
| 无 await ❌ | 按 F2.1 模式修复，补充测试 |

### 4. 输出

创建扫描报告 `specs/F2.2-scan-report.md`，包含：
- 扫描命令及执行时间
- 每处 res.json() 的位置、上下文截图、是否合规
- 修复记录（如有）
- reviewer 确认签名栏

**reviewer 确认**：

- [ ] reviewer 已审查扫描报告，无遗漏
- [ ] 所有 res.json() 调用均已正确 await（或已修复）
