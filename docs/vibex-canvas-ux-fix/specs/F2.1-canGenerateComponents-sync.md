# F2.1 — canGenerateComponents 与 handler 逻辑同步

**文件**: `vibex-fronted/src/components/canvas/BusinessFlowTree.tsx`
**行号**: 825–832（当前 useMemo）
**优先级**: P0

## 当前有 Bug 的代码

```typescript
// F1.2: Derive canGenerateComponents — shared logic for button disabled + validation
const canGenerateComponents = useMemo(() => {
  const activeContexts = contextNodes.filter((ctx) => ctx.isActive !== false);
  const selectedContextSet = new Set(selectedNodeIds.context);
  const validContexts = selectedContextSet.size > 0
    ? activeContexts.filter((ctx) => selectedContextSet.has(ctx.nodeId))
    : activeContexts;
  return validContexts.length > 0 && flowNodes.length > 0;
}, [contextNodes, selectedNodeIds.context, flowNodes.length]);
```

**Bug**: 
1. 只检查 `validContexts.length > 0` 但不检查 `flowsToSend.length > 0`（`flowNodes.length > 0` 不等于 `flowsToSend.length > 0`，因为 flows 也可能被 deactive 或 selection 过滤）
2. `canGenerateComponents` 与 `handleContinueToComponents` 内部的 `contextsToSend`/`flowsToSend` 计算逻辑不一致

## 修复方案

`canGenerateComponents` useMemo 完全镜像 handler 内部的计算逻辑：

```typescript
const canGenerateComponents = useMemo(() => {
  // Mirror exactly what handleContinueToComponents computes for contextsToSend
  const activeContexts = contextNodes.filter((ctx) => ctx.isActive !== false);
  const selectedContextSet = new Set(selectedNodeIds.context);
  const contextsToSend = selectedContextSet.size > 0
    ? activeContexts.filter((ctx) => selectedContextSet.has(ctx.nodeId))
    : activeContexts;

  // Mirror exactly what handleContinueToComponents computes for flowsToSend
  const activeFlows = flowNodes.filter((f) => f.isActive !== false);
  const selectedFlowSet = new Set(selectedNodeIds.flow);
  const flowsToSend = selectedFlowSet.size > 0
    ? activeFlows.filter((f) => selectedFlowSet.has(f.nodeId))
    : activeFlows;

  // Button enabled only when both are non-empty
  return contextsToSend.length > 0 && flowsToSend.length > 0;
}, [contextNodes, flowNodes, selectedNodeIds.context, selectedNodeIds.flow]);
```

**改动点**:
1. 增加 `flowsToSend` 计算（与 handler 完全一致）
2. `return` 条件改为 `contextsToSend.length > 0 && flowsToSend.length > 0`
3. `useMemo` deps 补充 `flowNodes`（完整对象）和 `selectedNodeIds.flow`
