# F2.2 — componentGenerating unmount cleanup

**文件**: `vibex-fronted/src/components/canvas/BusinessFlowTree.tsx`
**行号**: 758（handler 定义处）+ 新增 useEffect
**优先级**: Medium

---

## 当前有 Bug 的代码

```typescript
const handleContinueToComponents = useCallback(async () => {
  // 防重复提交：flowData 为空或正在生成时禁用
  if (componentGenerating || flowNodes.length === 0) return;
  setComponentGenerating(true);

  try {
    // ... API call ...
  } catch (err) {
    canvasLogger.BusinessFlowTree.error(' handleContinueToComponents error:', err);
    toast.showToast(err instanceof Error ? err.message : '生成组件树失败', 'error');
  } finally {
    setComponentGenerating(false);
  }
}, [componentGenerating, flowNodes, contextNodes, selectedNodeIds, projectId, setComponentNodes, setPhase, toast]);
```

**Bug**: 
- `finally` 块在正常 unmount 场景下不执行（React 组件 unmount 时不会等待 async 函数完成）
- 若 API 调用进行中组件 unmount，`componentGenerating` 状态粘滞在 `true`
- 下次 mount 时按钮永久 disabled（`disabled={!canGenerateComponents || componentGenerating}`）

---

## 修复方案

新增 `useEffect` cleanup：

```typescript
// Clean up: 组件 unmount 时重置 componentGenerating 状态，防止 stale state 导致按钮永久失效
useEffect(() => {
  return () => {
    setComponentGenerating(false);
  };
}, []);
```

**改动点**: 
- 在 `handleContinueToComponents` 的 `useCallback` 之前新增一个 `useEffect`
- cleanup 依赖数组为空（`[]`），确保只在 unmount 时执行一次

---

## 副作用说明

此 cleanup 不影响正常流程：
- API 调用成功后 → `finally` 执行 → `setComponentGenerating(false)` → cleanup 不再执行（已 unmount 了）
- API 调用进行中 unmount → `finally` 可能不执行 → cleanup 执行 → 状态正确重置

**注意**: cleanup 中直接调用 `setComponentGenerating(false)` 是幂等的，重复调用无副作用。

## 测试场景

| # | 场景 | 初始状态 | 操作 | 预期结果 |
|---|------|---------|------|----------|
| T1 | 组件正常 unmount（无进行中请求） | `componentGenerating = false` | unmount | 状态仍为 `false` |
| T2 | 组件 unmount（请求进行中） | `componentGenerating = true`（请求未完成） | unmount | cleanup 执行，`componentGenerating` 重置为 `false` |
| T3 | 组件 remount | `componentGenerating` 已由 cleanup 重置 | remount 后 render | 按钮 disabled 状态由 `canGenerateComponents` 决定 |
| T4 | 快速 mount/unmount 多次 | 任意 | 多次 unmount | cleanup 多次执行（幂等，无问题） |
