# F3.1 — hasAllNodes 要求 isActive !== false

**文件**: `vibex-fronted/src/components/canvas/ProjectBar.tsx`
**行号**: 160–161
**优先级**: P0

---

## 当前有 Bug 的代码

```typescript
const hasAllNodes = hasNodes(contextNodes) && hasNodes(flowNodes) && hasNodes(componentNodes)
  && contextNodes.length > 0 && flowNodes.length > 0 && componentNodes.length > 0;
```

其中 `hasNodes` 定义为 `nodes.length > 0`（来自 `cascade/CascadeUpdateManager.ts`）。

**Bug**:
- 只检查节点存在性（`length > 0`），不检查节点激活状态
- 当组件树因 Issue #1 静默失败时，`componentNodes` 为空 → `hasAllNodes` 永远 `false` → 按钮永久 disabled
- 即使修复 Issue #1 后，若节点存在但 `isActive === false`，按钮也会误导用户（显示可以创建但实际条件不满足）

---

## 修复方案

```typescript
const contextReady = contextNodes.length > 0 && contextNodes.every((n) => n.isActive !== false);
const flowReady = flowNodes.length > 0 && flowNodes.every((n) => n.isActive !== false);
const componentReady = componentNodes.length > 0 && componentNodes.every((n) => n.isActive !== false);
const hasAllNodes = contextReady && flowReady && componentReady;
```

**改动点**:
1. 将 `hasAllNodes` 拆分为三个独立的就绪检查
2. 每个树的检查：`length > 0 && every(isActive !== false)`
3. 最终 `hasAllNodes = contextReady && flowReady && componentReady`

---

## 降级路径（可选）

若需保留 `nodes.length > 0` 作为 fallback 兜底：

```typescript
const contextReady = contextNodes.length > 0 && contextNodes.every((n) => n.isActive !== false);
const flowReady = flowNodes.length > 0 && flowNodes.every((n) => n.isActive !== false);
const componentReady = componentNodes.length > 0 && componentNodes.every((n) => n.isActive !== false);
// Fallback: if any tree has zero nodes, still treat as ready if length check passes
const hasAllNodes = (contextReady || contextNodes.length > 0)
  && (flowReady || flowNodes.length > 0)
  && (componentReady || componentNodes.length > 0);
```

**推荐**: 不做 fallback，直接要求 `every(isActive !== false)`，与产品意图一致（用户看到绿色"已确认"节点时才能创建项目）。
