# Spec: F4.2 — handleConfirmAll 原子设置双字段

**Story ID**: F4.2
**所属 Epic**: E4
**文件**: `vibex-frontend/src/components/canvas/BoundedContextTree.tsx`
**改动位置**: `handleConfirmAll` 函数

---

## 现状（Bug）

```typescript
const handleConfirmAll = useCallback(() => {
  // Advance phase (no confirm gating in Epic 3)
  advancePhase();
}, [contextNodes, advancePhase]);
```

- 不设置 `isActive` 字段
- 不调用 `confirmDialogStore`
- 注释明确说 "no confirm gating in Epic 3" — 这是 Issue 3 的根因

---

## 改动规格

### 改动点：handleConfirmAll 原子设置 status + isActive

**原代码片段**：

```typescript
const handleConfirmAll = useCallback(() => {
  advancePhase();
}, [contextNodes, advancePhase]);
```

**新代码**：

```typescript
const handleConfirmAll = useCallback(() => {
  // 原子设置所有节点的确认状态
  contextNodes.forEach((node) => {
    toggleContextNode(node.nodeId); // 设置 status: 'confirmed'
  });
  // 同时确保 isActive 为 true（与 toggle 联动）
  // 若 toggleContextNode 已同时设置 isActive，则无需额外操作
  // 若 toggleContextNode 只设置 status，需补充 isActive 设置
  advancePhase();
}, [contextNodes, advancePhase]);
```

**依赖确认**：需确认 `toggleContextNode`（通过 store）的行为：
- 若已同时设置 `status: 'confirmed'` 和 `isActive: true` → 无需额外改动，spec 改为验证 `toggleContextNode` 被每个节点调用
- 若只设置 `status` → 需在 `handleConfirmAll` 中补充设置 `isActive: true`

### 验收确认

- [ ] `handleConfirmAll` 调用后，每个 `contextNodes` 的 `status === 'confirmed'`
- [ ] `handleConfirmAll` 调用后，每个 `contextNodes` 的 `isActive !== false`
- [ ] `advancePhase()` 在状态更新后调用
- [ ] `allConfirmed` 在 `handleConfirmAll` 调用后立即为 `true`
