## 开发规范

### 强制规则

1. **dedup API 可用性**
   - `scripts/dedup_api.py` 必须监听端口 8765
   - coord 集成使用 try/except，不阻塞派发流程
   - similarity 阈值 0.7（可调整）

2. **flaky-detector 参数传递**
   - 必须使用 `flaky-params.txt` 文件传递参数
   - 禁止使用命令行参数（shell 转义问题历史 bug）

3. **npm scripts 清理**
   - 删除操作前必须先 `grep` 确认无 CI 引用
   - 必须备份 `package.json.bak` 再修改
   - `test:notify` 改为 `scripts/test/notify.js` 独立脚本

4. **TypeScript / 代码质量**
   - 新增 Python 文件需有 `if __name__ == '__main__'` guard
   - npm scripts 修改后必须运行 `npm run` 验证

## 文件修改规则

### scripts/dedup_api.py

```python
#!/usr/bin/env python3
# ✅ 正确：完整错误处理
def run_server(port=8765):
    try:
        server = HTTPServer(('localhost', port), DedupHandler)
        print(f"Dedup API running on port {port}")
        server.serve_forever()
    except OSError as e:
        if e.errno == 98:  # Address in use
            print(f"Port {port} already in use, dedup check skipped")
        else:
            raise

# ❌ 错误：无错误处理，端口占用时崩溃
```

### coord/scheduler.py

```python
# ✅ 正确：非阻塞集成
def dispatch_project(project_data: dict):
    try:
        resp = requests.post('http://localhost:8765/dedup', json={...}, timeout=5)
        # ... 写入 dedup 字段
    except Exception as e:
        print(f'[Dedup] Non-blocking failure: {e}')
    _do_dispatch(project_data)  # 不阻塞

# ❌ 错误：阻塞派发
def dispatch_project(project_data: dict):
    resp = requests.post(...)  # 超时或错误时整个派发失败
    _do_dispatch(project_data)
```
