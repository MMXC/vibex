## 2. 问题列表

### Bug 1: OPTIONS 预检 CORS 500（P0 | 0.5h）

**根因**: `protected_.options` 在 `authMiddleware` 之后注册，所有跨域预检被 401 拦截。

**修复**: 将 `protected_.options` 移到 `authMiddleware` 之前。

**验收标准**:
- `curl -X OPTIONS -I /v1/projects` 返回 204（非 401）
- `Access-Control-Allow-Origin: *` header 存在
- GET/POST/DELETE 请求不受影响

---

### Bug 2: Canvas Context checkbox 不生效（P0 | 0.3h）

**根因**: `BoundedContextTree.tsx` checkbox `onChange` 绑定到 `toggleContextNode`（右击菜单）而非 `onToggleSelect`（Zustand store）。

**修复**: 将 checkbox `onChange` 改为 `onToggleSelect`。

**验收标准**:
- 点击 checkbox → `selectedNodeIds` 在 Zustand store 中更新
- 右击 `toggleContextNode` 仍正常工作
- 回归测试通过

---

### Bug 3: generate-components flowId = unknown（P0 | 0.3h）

**根因**: AI schema 缺少 `flowId` 字段定义，prompt 未要求输出。

**修复**: schema 添加 `flowId: string`，prompt 明确要求输出。

**验收标准**:
- `expect(flowId).toMatch(/^flow-/)`
- `expect(flowId).not.toBe('unknown')`
- API 响应包含有效 flowId

---

### Bug 4: SSE 超时 + 连接清理（P1 | 1.5h）

**根因**: `aiService.chat()` 无超时控制，10s 无响应时 Worker 挂死；`cancel()` 未清理 `setTimeout`。

**修复**: `AbortController.timeout(10000)` 包装 aiService 调用，`ReadableStream.cancel()` 清理所有 timers。

**验收标准**:
- SSE 流 10s 无响应 → `controller.close()`，前端收到流结束
- `stream.cancel()` 后 `clearTimeout` 被调用
- jest 测试覆盖

---

### Bug 5: test-notify 去重（P1 | 1h）

**根因**: Python 版已有去重，JS 版遗漏。

**修复**: 实现 `dedup.js`（`checkDedup` + `recordSend`），状态持久化到 `.dedup-cache.json`。

**验收标准**:
- 首次调用 → `{ sent: true }`
- 5min 内重复 → `{ skipped: true }`
- jest 测试通过
