## P0 提案详情

### P001: 需求智能补全（AI 主动澄清追问）

**用户价值**: 用户输入模糊需求时，AI 主动提问澄清，而非直接生成错误结果，避免返工。

**20260410 状态**: 已列入 PRD，但未开发。

**现状**:
- AI 直接生成结果，用户无法中途干预
- 模糊输入（如"我要做个电商"）生成质量差的领域模型
- 用户不知道 AI 需要哪些信息

**建议方案**:
1. 输入过程中实时检测关键词（实体名、动词、业务术语）
2. 触发澄清追问：「检测到您提到"订单"，请补充：订单包含哪些核心字段？」
3. 多轮澄清后生成，提高准确率
4. 澄清历史可折叠查看，不打断用户思路

**验收标准**:
- [ ] 输入 ≥50 字时触发智能检测
- [ ] 追问响应时间 < 1s
- [ ] 经澄清后的分析质量评分高于直接生成 20%
- [ ] 澄清历史可折叠/展开

### P002: 项目搜索与过滤

**用户价值**: 项目数量增加后，无法快速找到目标项目，多项目用户强烈需求。

**20260410 状态**: 已列入 PRD，但未开发。

**现状**:
- Dashboard 项目列表无搜索功能
- 无过滤/排序能力
- 10+ 项目时体验极差

**建议方案**:
1. 全局搜索栏：支持项目名模糊匹配
2. 过滤栏：按时间（最近 7 天/30 天/全部）、按状态（草稿/完成）
3. 最近项目快捷入口（Dashboard 顶部）
4. 响应时间 < 200ms（前端过滤或索引搜索）

**验收标准**:
- [ ] 搜索响应时间 < 200ms
- [ ] 支持按名称、时间、状态过滤
- [ ] 搜索结果实时更新（debounce 300ms）
- [ ] 空搜索框显示最近项目

### P004: 项目版本对比

**用户价值**: 当前无法查看需求变更历史，用户无法追溯"之前是什么样"。

**20260410 状态**: 已列入 PRD，但未开发。

**现状**:
- 每次保存不生成版本快照
- 无版本历史 API
- 无对比视图

**建议方案**:
1. 每次保存生成版本快照（snapshotId + timestamp + content hash）
2. 版本列表 API：`GET /api/v1/projects/:id/versions`
3. 版本对比视图：两版本差异高亮（文字 diff + 结构 diff）
4. 版本回滚：选择历史版本恢复到当前

**验收标准**:
- [ ] 版本历史列表可查看
- [ ] 支持两个版本对比
- [ ] 差异内容高亮显示
- [ ] 支持回滚到历史版本

### P005: Tree 组件按钮样式统一

**用户价值**: Toolbar 按钮样式不一致影响视觉一致性和用户信任。

**来源**: Dev/20260408（P005），与 P1 功能点 tree-toolbar-consolidation 相关。

**现状**:
- Tree 面板 Toolbar 按钮（展开/折叠/生成/删除）样式不统一
- 部分按钮使用 icon-only，部分使用 icon+文字
- hover/active 状态样式不一致

**建议方案**:
1. 统一 Toolbar 按钮组件：`<TreeToolbarButton icon={} label={} onClick={} />`
2. 统一按钮尺寸、间距、圆角
3. 统一 hover/active/disabled 状态
4. 参照现有 design system 规范

**验收标准**:
- [ ] Tree 面板所有按钮使用统一组件
- [ ] 按钮样式通过 Playwright 截图回归测试
- [ ] icon+文字按钮和 icon-only 按钮视觉一致

**验收标准**:

- [ ] 提示条文案明确
