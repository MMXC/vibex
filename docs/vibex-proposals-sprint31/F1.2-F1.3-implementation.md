# E02 项目导入导出补全 — 实现方案

## 背景

Sprint30 的 E02-U1/U2/U3 完成了 backend API（`export` + `import` endpoints），但前端 Dashboard 的导出按钮仍 `alert('导出功能开发中')`，且缺少导入功能入口。

## 分析

### 当前问题

1. **F1.1 Schema 不对齐**: `projectExporter.export()` 输出 `{projectId, projectName}`，但 `VibexExportSchema` 期望 `{project: {name, description}}`
2. **F1.2 Dashboard 导出**: 单项目导出和批量导出都是 alert 占位，未调用 backend API
3. **F1.3 ImportModal 缺失**: 没有导入入口和组件

### 影响范围

- `vibex-backend/src/lib/services/projectExporter.ts` — 输出格式
- `vibex-backend/src/lib/schemas/vibex.ts` — exportedAt optional
- `vibex-backend/src/__tests__/import-export.test.ts` — roundtrip 测试
- `vibex-fronted/src/app/dashboard/page.tsx` — 导出重构 + 导入按钮
- `vibex-fronted/src/components/dashboard/ImportModal.tsx` — 新建

## 方案设计

### F1.2 — Dashboard 导出重构

**方案**: 调用 Backend API `GET /api/projects/:id/export`，下载 `.vibex` 文件

**实施**:
1. `handleSingleExport(projectId, projectName)` → `fetch('/api/projects/${id}/export')` → `downloadBlob(blob, filename)`
2. `handleBulkExport()` → `Promise.all(selectedProjectIds.map(id => handleSingleExport(id, name)))`
3. Dashboard header 添加「导入项目」按钮

### F1.3 — ImportModal

**方案**: 新建 `ImportModal.tsx` 组件，支持拖拽/点击上传

**实施**:
1. 创建 `ImportModal.tsx`，状态: `isOpen`, `isLoading`, `errorMessage`
2. Dashboard header 添加「导入项目」按钮，渲染 `<ImportModal>`
3. `onSuccess` 中 `queryClient.invalidateQueries` 刷新列表

## 实施步骤
