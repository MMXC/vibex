# F1.4 同步状态可视化 — 四态规格

**Epic**: F1.4 同步状态可视化
**涉及文件**: `vibex-fronted/src/components/canvas/OfflineBanner.tsx` + `OfflineBanner.module.css`
**集成页面**: CanvasPage（`vibex-fronted/src/components/canvas/CanvasPage.tsx`）

---

## 状态机概览

```
                    ┌──────────────────────┐
                    │   HIDE（默认态）      │
                    │  hidden = true        │
                    │  → return null       │
                    └──────────┬───────────┘
                               │ !hidden && (isOffline || pendingCount > 0 || isSyncing)
                               ▼
┌────────────────┐    ┌────────────────┐    ┌────────────────┐
│ OFFLINE（离线） │───▶│ SYNCING（同步中）│───▶│ SUCCESS（完成） │
│ 📡 离线模式     │    │ 🔄 进度条显示    │    │ ✅ 2s后消失     │
│ 待同步计数显示   │    │ 待同步计数+进度条 │    │ hidden=true    │
└────────────────┘    └────────────────┘    └────────────────┘
                               │
                               ▼
                    ┌────────────────┐
                    │ ERROR（错误态） │
                    │ ⚠️ 错误消息     │
                    │ 含retryCount   │
                    └────────────────┘
                               │
                               │ 重试成功后
                               ▼
                    ┌────────────────┐
                    │ SYNCING 或 SUCCESS │
                    └────────────────┘
```

---

## 态 1：理想态（SUCCESS）

**触发条件**: `detail.type === 'complete'` 事件触发，banner 显示 ✅，2s 后 `hidden = true`

**视觉规范**:
- 同通用 banner 样式（见下方通用规范）
- 图标: `✅`
- 文案: 无（或 `已同步完成`，可选）

**时序**:
1. `detail.type === 'complete'` → `pendingCount = 0` + `isSyncing = false` + `syncError = null`
2. 触发 `setTimeout(() => { setHidden(true) }, 2000)`
3. 2s 后 `hidden = true` → `return null`

**E2E 验收断言**:
```javascript
// 同步完成后 2s 内 banner 消失
await context.setOnline()
await waitFor(() => {
  expect(screen.queryByTestId('offline-banner')).toBeNull()
}, { timeout: 3000 })
```

**引导文案**: "已同步完成"（出现后约 2s 消失，不需用户操作）

---

## 态 2：空状态（HIDE）

**触发条件**: `hidden = true` 或 `showSyncBanner = false`（`isOffline=false && pendingCount=0 && !isSyncing && syncError=null`）

**视觉规范**:
- `return null`，不渲染任何 DOM

**E2E 验收断言**:
```javascript
// 网络正常且无待同步项时 banner 不显示
expect(screen.queryByTestId('offline-banner')).toBeNull()
```

**引导文案**: 无（不应出现任何空状态文案）

---

## 态 3：加载态（OFFLINE / SYNCING 初始）

**SYNCING 初始阶段**: `pendingCount > 0 && isSyncing = true`，进度条正在更新

**视觉规范**:
- 图标: `🔄`
- 文案: `{pendingCount} 项操作待同步`
- 进度条显示（见下方通用规范）
- 进度条初始宽度: `0%`，随着 `detail.completed` 更新逐渐增长

**E2E 验收断言**:
```javascript
// 待同步计数正确显示
expect(screen.getByTestId('offline-banner')).toHaveTextContent(/\d+ 项操作待同步/)
// 进度条存在
const progressBar = screen.getByTestId('offline-banner').querySelector('[data-sync-progress]')
expect(progressBar).toBeVisible()
// 进度条有 aria 属性
expect(progressBar).toHaveAttribute('aria-valuenow')
expect(progressBar).toHaveAttribute('aria-valuemin', '0')
expect(progressBar).toHaveAttribute('aria-valuemax')
expect(progressBar).toHaveAttribute('aria-label', expect.stringContaining('同步进度'))
```

---

## 态 4：错误态（ERROR）

**触发条件**: `detail.type === 'error'` 事件触发

**视觉规范**:
- 图标: `⚠️`
- 文案: `syncError` 内容，`color: #f87171`（红色）
- **必须包含 retryCount**: 错误消息格式为"同步失败（第 N 次），请检查网络"，N 为重试次数

**引导文案**:
- "同步失败（第 {retryCount} 次），请检查网络" — 告知用户已重试 N 次，引导检查网络
- 不需要手动重试按钮（本期暂缓）

**E2E 验收断言**:
```javascript
// 错误消息包含 retryCount
expect(screen.getByTestId('offline-banner')).toHaveTextContent(/第 \d+ 次失败/)
// 错误消息有错误样式
const errorText = screen.getByTestId('offline-banner').querySelector('[class*="errorText"]')
expect(errorText).toBeVisible()
```

## 通用 Banner 样式规范

**定位**: `position: fixed; bottom: 24px; left: 50%; transform: translateX(-50%)`（居中底部）

**容器**:
- 背景: `#1c1c1c`
- 边框: `1px solid #333`
- 错误态边框: `1px solid #ef4444`（通过 `data-sync-error="true"` 属性选择器）
- 圆角: `24px`（胶囊形）
- 内边距: `12px 20px`
- 阴影: `0 4px 16px rgba(0,0,0,0.4)`
- 动画: `slideUp 0.3s ease`
- 最大宽度: `90vw`

**data 属性**:
- `data-testid="offline-banner"`
- `data-sync-error={syncError !== null}`（布尔字符串）
- 进度条: `data-sync-progress="true"`

**无障碍属性**:
- `role="alert"`（实时通知）
- `aria-live="polite"`（非阻塞性通知）

**进度条**:
- 宽度: `200px`
- 高度: `4px`
- 背景: `#333`
- 填充: `#6366f1`
- 圆角: `2px`
- 动画: `transition: width 0.3s ease`

## E2E 完整验收脚本

```javascript
describe('F1.4 同步状态可视化', () => {
  beforeEach(() => {
    // 清除 IndexedDB 和 localStorage
  })

  it('网络正常且无待同步项时 banner 不显示（空状态）', () => {
    expect(screen.queryByTestId('offline-banner')).toBeNull()
  })

  it('离线时显示离线提示（加载态-离线）', async () => {
    await context.setOffline()
    expect(screen.getByTestId('offline-banner')).toBeVisible()
    expect(screen.getByTestId('offline-banner')).toHaveTextContent(/离线模式/)
  })

  it('待同步项计数正确显示（加载态-同步前）', async () => {
    // 模拟离线写入 3 项
    await offlineQueue.enqueueRequest({ url: '/api/test', method: 'POST', body: '{}' })
    await offlineQueue.enqueueRequest({ url: '/api/test2', method: 'POST', body: '{}' })
    await offlineQueue.enqueueRequest({ url: '/api/test3', method: 'POST', body: '{}' })
    expect(screen.getByTestId('offline-banner')).toHaveTextContent(/3 项操作待同步/)
  })

  it('进度条有 data-sync-progress 属性和 aria 属性（同步中）', () => {
    const banner = screen.getByTestId('offline-banner')
    const progressBar = banner.querySelector('[data-sync-progress]')
    expect(progressBar).toBeVisible()
    expect(progressBar).toHaveAttribute('aria-valuenow')
    expect(progressBar).toHaveAttribute('aria-valuemax')
    expect(progressBar).toHaveAttribute('aria-label', expect.stringContaining('同步进度'))
  })

  it('同步完成后 2s 内 banner 消失（理想态）', async () => {
    await context.setOnline()
    // 触发同步完成事件
    window.dispatchEvent(new CustomEvent('offline-replay-progress', {
      detail: { type: 'complete' }
    }))
    expect(screen.getByTestId('offline-banner')).toHaveTextContent(/✅/)
    await waitForElementToBeRemoved(screen.queryByTestId('offline-banner'), { timeout: 3000 })
  })

  it('同步失败显示错误消息且包含 retryCount（错误态）', async () => {
    window.dispatchEvent(new CustomEvent('offline-replay-progress', {
      detail: { type: 'error', lastError: '同步失败（第 2 次），请检查网络' }
    }))
    const banner = screen.getByTestId('offline-banner')
    expect(banner).toHaveTextContent(/第 2 次失败/)
    expect(banner).toHaveTextContent(/请检查网络/)
  })

  it('无障碍：role=alert + aria-live=polite', () => {
    const banner = screen.getByTestId('offline-banner')
    expect(banner).toHaveAttribute('role', 'alert')
    expect(banner).toHaveAttribute('aria-live', 'polite')
  })
})
```
