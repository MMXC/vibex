# Spec: F1.2 AI 评审 CI Gate

## 概述

GitHub Actions CI 在 PR 创建/更新时自动触发 AI 设计评审，评审结果写入 JSON 文件，并在 PR 评论中发布摘要。

## 技术方案

### 文件
- 新建 `.github/workflows/ai-review.yml`
- 复用 `vibex-fronted/src/lib/mcp-bridge.ts` 的 `spawnAgent`

### CI Flow

```
PR opened/updated
  → trigger ai-review workflow
  → clone repo
  → run: node scripts/ai-review.js {pr_number}
  → spawnAgent({ type: 'design-review', pr: { number, diff } })
  → writeFile: ai-review-results/{pr_number}.json
  → github.rest.issues.createComment(summary)
```

### AI 评审 JSON 格式

```json
{
  "pr": 42,
  "status": "complete",
  "score": 78,
  "feedback": "设计一致，建议优化响应式布局...",
  "issues": [
    { "severity": "warning", "file": "CanvasPage.tsx", "line": 42, "message": "..." }
  ],
  "timestamp": "2026-05-08T12:00:00Z"
}
```

### 评审内容

基于 PR diff 分析：
- 组件命名一致性
- 样式规范遵循（Tailwind 类名）
- 响应式布局覆盖
- 组件复用情况

## 验收标准

```
expect(github.context.payload.pull_request).toBeDefined();
expect(spawnAgent).toHaveBeenCalled();
const result = JSON.parse(fs.readFileSync('ai-review-results/42.json'));
expect(result.score).toBeGreaterThanOrEqual(0);
expect(result.feedback).toBeDefined();
```

## DoD

- [ ] `ai-review.yml` 配置 `on: [pull_request]`
- [ ] 评审结果写入 `ai-review-results/{pr}.json`
- [ ] PR 评论发布评审摘要
- [ ] 手动 re-run 正常
- [ ] 已有 PR 测试通过
