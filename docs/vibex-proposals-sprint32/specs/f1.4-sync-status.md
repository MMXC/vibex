# Spec: F1.4 同步状态可视化

## 概述

OfflineBanner 显示待同步操作计数和进度，让用户知道有多少离线操作正在等待同步。

## 技术方案

### 文件
- 修改 `vibex-fronted/src/components/dds/OfflineBanner.tsx`

### UI 设计

```
┌─────────────────────────────────────────┐
│ 🔄 3 项操作待同步...  [████████░░] 2/3  │
└─────────────────────────────────────────┘
```

### 状态

| 状态 | 显示 |
|------|------|
| 在线，无队列 | 不显示 |
| 在线，有队列 | 显示进度 |
| 离线，有队列 | 显示"离线，X 项待同步" |
| 同步中 | 显示进度条 + 递减计数 |
| 同步完成 | 2s 后消失 |
| 同步失败 | 显示错误 + 重试按钮 |

### 与 F1.3 的集成

- 读取 IndexedDB `offline-queue` 的 `length` 显示计数
- 监听 `sync` 事件更新进度
- 同步完成/失败通过 `CustomEvent` 通知 Banner

## 验收标准

```
expect(screen.getByText(/3 项操作待同步/)).toBeInTheDocument();
expect(page.locator('[data-sync-progress]')).toHaveAttribute('aria-valuenow', '1');
await flushPromises();
expect(screen.queryByTestId('offline-banner')).not.toBeInTheDocument();
```
