# Spec — 原型画布错误态规格

**文件**: spec-prototype-canvas-error-state.md
**状态**: v1.0
**日期**: 2026-04-17

---

## 1. 错误态原则

- 错误信息必须**可操作**，不只报告问题，要引导下一步
- 网络异常/权限不足/数据超长/接口超时 四种场景必须覆盖
- 错误态使用红色/橙色视觉标识，不使用灰色

---

## 2. 错误态 — 网络异常

### 触发条件

- 用户网络断开
- API 请求超时
- WebSocket 断开

### UI 描述

- 画布顶部：固定 Banner（红色背景）
  - 图标：⚠️ 或断开连接图标
  - 文案：`"网络连接已断开，数据已暂存本地"`
  - 副文案：`"检查网络后自动重连"`
  - 右侧按钮：`"重试"` — 手动触发重连

### 数据策略

- 画布数据自动保存到 localStorage（每 30s 或每次操作后）
- 重连后自动同步到服务器
- 冲突时提示用户选择版本

### 禁止事项

- ❌ 不显示 error stack
- ❌ 不清空画布数据
- ❌ 不阻止用户继续编辑（离线模式）

---

## 3. 错误态 — 权限不足

### 触发条件

- 用户无编辑权限（只读项目）
- Token 过期
- 跨域无权限

### UI 描述

- **场景 1: 只读项目**
  - 画布显示遮罩（半透明灰色）
  - 提示：`"当前项目为只读权限"`
  - 按钮：`"联系管理员获取编辑权限"`

- **场景 2: Token 过期**
  - 模态框居中弹出
  - 标题：`"登录已过期"`
  - 内容：`"请重新登录以继续编辑"`
  - 按钮：`"重新登录"` — 跳转登录页

- **场景 3: 跨域权限**
  - Toast 提示：`"无权限访问，请检查项目设置"`
  - 按钮：`"刷新"` — 重新获取权限

### 禁止事项

- ❌ 不显示 401/403 HTTP 状态码
- ❌ 不让用户在不明确原因的情况下卡住

---

## 4. 错误态 — 数据超长

### 触发条件

- Mock 数据 JSON 超过 50KB
- 导出 JSON 超过 5MB（Canvas 限制）
- 组件数量超过 500 个

### UI 描述

- **场景 1: Mock数据超长**
  - Mock数据 Tab 顶部：红色警告 Banner
  - 文案：`"数据过长（>50KB），可能导致性能问题"`
  - 建议：`"建议精简或移除部分数据"`
  - 仍可保存，但不推荐

- **场景 2: 导出超限**
  - 导出前校验失败
  - Toast 提示：`"导出内容过大（>5MB），无法导出"`
  - 建议：`"减少组件数量或精简 Mock 数据"`
  - 按钮：`"查看优化建议"` — 展开优化建议面板

- **场景 3: 组件数量超限**
  - 画布右侧：警告 Badge（橙色）
  - 文案：`"组件数量已达上限（500）"`
  - 建议：`"建议拆分为多个页面"`

### 禁止事项

- ❌ 不强制阻止用户保存（可警告但不阻止）
- ❌ 不静默截断数据

---

## 5. 错误态 — 接口超时

### 触发条件

- API 请求 30s 内无响应
- 后端服务不可用
- 数据库连接超时

### UI 描述

- **场景 1: 单次操作超时**
  - Toast 提示：`"请求超时，请稍后重试"`
  - 自动重试 1 次
  - 重试失败后显示 `"重试失败"` + `"重试"` 按钮

- **场景 2: 批量操作超时**
  - 模态框居中
  - 标题：`"操作超时"`
  - 内容：`"批量保存超时，已保存部分数据"`
  - 列表：显示哪些数据保存成功，哪些失败
  - 按钮：`"重试失败项"` / `"知道了"`

- **场景 3: 持续超时（后端不可用）**
  - 画布顶部：红色 Banner
  - 文案：`"服务暂时不可用，数据已暂存本地"`
  - 副文案：`"正在等待服务恢复..."`
  - 预计时间（如有）：`"预计 5 分钟后恢复"`

### 禁止事项

- ❌ 不显示 HTTP 超时细节
- ❌ 不让用户重复点击导致更多超时

---

## 6. 错误态 — 组件加载失败

### 触发条件

- 组件拖入画布时，组件定义加载失败
- 组件类型未知

### UI 描述

- 画布节点显示：红色边框 + 错误图标
- 文案：`"组件加载失败"`
- 节点内容：显示组件 ID（而非崩溃）
- 右键菜单：`"移除"` / `"重试"`

### 禁止事项

- ❌ 不让整个画布崩溃
- ❌ 不显示 error stack

---

## 7. 错误态 — JSON导入失败

### 触发条件

- 用户导入的 JSON 文件格式错误
- JSON 版本不兼容（v1.0 导入 v2.0）
- JSON 数据损坏

### UI 描述

- **场景 1: 格式错误**
  - 模态框居中
  - 标题：`"导入失败"`
  - 内容：`"JSON 格式错误，请检查文件内容"`
  - 按钮：`"选择其他文件"`

- **场景 2: 版本不兼容**
  - 模态框居中
  - 标题：`"版本不兼容"`
  - 内容：`"该文件为 v2.0 格式，当前系统支持 v1.0"`
  - 按钮：`"了解详情"`（跳转文档）/ `"选择其他文件"`

- **场景 3: 数据损坏**
  - 模态框居中
  - 标题：`"文件已损坏"`
  - 内容：`"无法读取文件内容，请确认文件完整性"`
  - 按钮：`"选择其他文件"`

### 禁止事项

- ❌ 不静默忽略导入错误
- ❌ 不覆盖现有数据（导入失败时保留原数据）

---

## 验收标准

```
expect(网络异常时显示红色Banner+重试按钮).toBe(true)
expect(网络异常时数据暂存localStorage).toBe(true)
expect(只读项目显示遮罩+联系管理员按钮).toBe(true)
expect(Token过期显示模态框+重新登录按钮).toBe(true)
expect(Mock数据超50KB时显示警告Banner).toBe(true)
expect(导出超5MB时Toast+优化建议).toBe(true)
expect(单次超时Toast+自动重试).toBe(true)
expect(批量超时显示模态框+重试选项).toBe(true)
expect(组件加载失败显示错误节点+移除选项).toBe(true)
expect(导入失败保留原数据).toBe(true)
```
