# Spec — DDS Canvas 空状态（Empty State）

**文件**: `spec-dds-canvas-empty-state.md`
**组件**: DDSScrollContainer / DDSFlow
**Epic**: Epic 5 — S5.2
**状态**: 进行中

---

## 1. 理想态

用户进入 Spec Canvas 后，如果某个章节（requirement/context/flow）内没有卡片，应显示引导型空状态——不是留白，而是清晰的引导，告诉用户"这里可以做什么"。

每个章节的空状态文案和插图各不相同，体现章节特征：

| 章节 | 插图主题 | 主文案 | 副文案 | CTA |
|------|---------|--------|--------|-----|
| 需求 (requirement) | 3 张便签纸叠加 | 添加你的第一个用户故事 | 将需求拆解为角色、行为、收益的结构化卡片 | + 新建用户故事 |
| 上下文 (context) | 2 个圆环交叉（DDD 限界上下文图示） | 建立你的限界上下文 | 划分领域边界，明确职责和依赖关系 | + 新建限界上下文 |
| 流程 (flow) | 3 个箭头串联的节点 | 描绘你的领域流程 | 从步骤.actor 和前置条件开始，构建可执行的流程 DAG | + 新建流程步骤 |

## 2. 空状态定义

### 2.1 需求章节空状态（requirement）

**触发条件**: `chapters.requirement.cards.length === 0`

**布局**:
```
┌─────────────────────────────────────────────┐
│                                             │
│          [插图: 3张便签纸叠加图标]            │
│                                             │
│     添加你的第一个用户故事                    │
│                                             │
│   将需求拆解为角色、行为、收益的结构化卡片      │
│                                             │
│      [ + 新建用户故事 ]  [ AI 辅助生成 ]     │
│                                             │
│   💡 快捷键: 按 N 新建 · 按 A 打开 AI 草稿   │
│                                             │
└─────────────────────────────────────────────┘
```

**插图规格**:
- SVG 内联，尺寸 80x80px
- 颜色: `var(--text-secondary, #6b7280)`
- 主题: 3 张倾斜的便签纸，带虚线边框和文字轮廓
- 不可用纯色块或 emoji 替代

**CTA 按钮**:
- 主按钮: "+ 新建用户故事" — `bg: var(--accent-primary)`, hover 提亮 10%
- 次按钮: "AI 辅助生成" — `border: 1px solid var(--border)`, 文字色
- 按钮间距: 12px
- 点击主按钮 → 打开卡片编辑弹窗（新卡片）
- 点击次按钮 → 打开 AIDraftDrawer

**快捷键提示**:
- 字号: 12px, `var(--text-secondary)`
- 位置: CTA 下方，间距 16px
- 键盘图标用 `<kbd>` 标签包裹

## 3. 行为规则

| 行为 | 处理 |
|------|------|
| 首次进入且无数据 | 显示空状态 + 加载骨架屏消失后替换为空状态 |
| 章节有数据但被筛选为空 | 显示"没有匹配的结果"而非空状态引导 |
| AI 草稿生成中 | 临时隐藏空状态，显示生成中骨架 |
| 章节内只有 1 张卡片 | 不显示空状态，显示卡片本身 |
