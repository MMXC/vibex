# Spec: FlowTreePanel「添加连线」按钮 — E1-QA

**关联功能**: F-Q1.1
**页面**: FlowTreePanel.tsx
**状态**: 草稿

---

## 四态定义

### 理想态（Default）
- 按钮文本：「添加连线」
- 样式：`bg-blue-500 text-white px-3 py-1.5 rounded`
- 位置：FlowTreePanel 工具栏区域，与其他操作按钮对齐
- 状态：默认可用，无徽标/角标

### 空状态（Empty — 无页面可连）
- 条件：FlowTreePanel 中页面列表为空时
- UI：**禁止只留白**
- 插图：简笔画虚线框 + 连线图标（SVG，约 48×48px，fill: #CBD5E1）
- 文案：「还没有页面」副文案：「添加页面后再来创建跳转连线」
- 按钮：禁用态，`opacity-50 cursor-not-allowed`

### 加载态（Loading — 连线创建中）
- 条件：点击按钮后，等待用户选择源/目标节点时
- 方案 A（弹窗式）：Modal 显示「选择源页面」+ 页面列表，loading 时按钮 text 变为「连接中...」
- 方案 B（内联式）：FlowTreePanel 顶部显示步骤指示器「① 选择源页面 → ② 选择目标页面 → ③ 确认」
- 骨架屏：页面列表项用灰色占位条（而非 spinner）
- 按钮：disabled + spinner（禁止抖动的圆圈）

### 错误态（Error）
| 错误类型 | 表现 |
|---------|------|
| 网络异常 | Toast:「连接失败，请重试」+ 重试按钮 |
| 页面不存在 | 红色边框高亮选中项 + 「该页面已被删除」提示 |
| 循环引用检测 | 红色警告文案「不能跳转到自己或形成循环」 |
| 权限不足 | Toast:「无权限创建连线」 |
| 超时（30s） | Toast:「请求超时，请重试」 |

---

## 交互流程

1. **点击按钮** → 进入连线创建流程（方案 B 内联式）
2. **步骤1**: 高亮 FlowTreePanel 中的页面列表，等待点击选择源页面
3. **选中源页面**: 高亮消失，出现下一步指示
4. **步骤2**: 再次高亮页面列表，等待选择目标页面
5. **选中目标页面**: 出现「确认连线」按钮
6. **点击确认**: 调用 `prototypeStore.addEdge()` → 成功后 Toast「连线已创建」
7. **取消**: 任意步骤点击「取消」或按 Esc，返回默认态

## 开发交接标注（神技6）

- 按钮对齐：`items-center justify-between`
- 响应式：`w-full` 保证移动端可点击（最小 44×44px 触控区域）
- 禁止硬编码颜色：使用 design token（`--color-primary-500` 等）
- 禁止硬编码间距：8px 倍数（`gap-2`, `p-3` 等）
