# Spec: E5 — 状态与错误处理规格

**对应 Epic**: E5 状态与错误处理
**目标文件**: `vibex-fronted/src/components/delivery/` 全部 Tab 组件
**相关**: `vibex-fronted/src/stores/deliveryStore.ts`

---

## 1. 空状态规范（通用）

### 引导页设计

每个 Tab 无数据时显示统一的引导页：

| Tab | 引导文案 | 副文案 | 按钮 | 跳转目标 |
|-----|---------|--------|------|---------|
| 章节 | "还没有限界上下文" | "请先在详设画布中创建上下文" | "去详设画布" | /design/dds-canvas |
| 流程 | "还没有业务流程" | "请先在详设画布中创建流程" | "去详设画布" | /design/dds-canvas |
| 组件 | "还没有组件" | "请先在原型画布中拖拽组件" | "去原型画布" | /prototype/editor |
| PRD | "还没有足够数据生成 PRD" | "请先在画布中创建内容" | "去原型画布" | /prototype/editor |
| API | "还没有 API 端点" | "请先在详设画布的 API 章节中添加" | "去详设画布" | /design/dds-canvas |

### 引导页样式

- 居中显示
- 插图：文件夹/文档 SVG（48x48px）
- 标题：16px bold
- 副文案：14px，灰色
- 按钮：primary 样式

### 禁止

- 空状态只留白
- 空状态只显示 "无数据" 文字

## 2. 骨架屏规范（通用）

### 使用场景
- Tab 切换时数据加载中
- 页面初始加载
- 导出数据生成中

### 骨架屏样式

- 灰色占位块（`var(--color-skeleton)`）
- 圆角：4px
- 高度：与实际内容一致（模拟真实布局）
- 宽度：填充容器
- 动画：渐变闪烁（shimmer）

### 禁止

- 使用转圈（会抖动且无法预期加载时长）
- 骨架屏内容与实际布局不一致

## 3. 错误态规范

### Toast 错误提示

| 场景 | 文案 | 时长 |
|------|------|------|
| 数据加载失败 | "数据加载失败，请检查网络连接" | 5s |
| 导出失败 | "导出失败，请重试" | 5s |
| 跳转失败 | "无法跳转，请检查目标页面" | 5s |
| 复制失败 | "复制失败，请手动复制" | 3s |

### 错误处理原则

- 不崩溃：任何错误不能导致页面不可用
- 不丢失数据：操作失败时用户已输入的内容必须保留
- 可重试：所有失败操作都有重试入口
- 友好文案：错误信息要告诉用户下一步怎么办

---

## 4. 导出状态规范

### 导出进度 Modal

- 显示进度条 + 百分比
- 列出正在导出的文件
- 当前处理中的文件高亮
- 完成后显示成功提示

### 导出失败处理

- Modal 显示失败状态
- 列出成功/失败的文件数
- 失败文件有单独重试按钮
- "全部重试" 按钮
- "关闭" 按钮（失败后仍可关闭）

---

## 样式约束

- 间距：8 的倍数
- 颜色：使用 Token
- 禁止硬编码
- 骨架屏动画：`background: linear-gradient(90deg, var(--color-skeleton) 25%, var(--color-skeleton-light) 50%, var(--color-skeleton) 75%)`
- Toast：固定在页面底部，居中
