## 3. 技术方案选项

### 方案 A: D3.js 可视化增强（推荐）

**现状技术栈**: 限界上下文树和流程树使用 D3.js 力导向图渲染
**增强思路**: 在现有 D3.js 基础上扩展节点类型和连线

**上下文树增强**:
- 增加 `dependency` 边类型（箭头连线）
- 增加 `aggregate` 边类型（聚合连线，粗线）
- 拖拽节点调整布局

**流程树增强**:
- 增加 XOR Gateway 节点（菱形，标注条件）
- 增加 OR Gateway 节点（菱形，标注 OR）
- 增加循环回路（虚线箭头指向前置步骤）
- 分支条件显示在连线旁

**工作量**: ~6h（Dev）

### 方案 B: 全新可视化库迁移

**思路**: 迁移到更专业的流程图库（如 React Flow、BPMN.js）

**优点**: 功能更强大（BPMN 标准支持）
**缺点**: 迁移成本高（~20h），破坏性改动
