## 一、产品概述

### 1.1 产品定位

VibeX 是一个 DDD (领域驱动设计) 分析工具，通过 AI 辅助帮助用户快速完成领域建模和业务流程设计。

**核心价值**: 将复杂的 DDD 分析流程可视化、智能化，降低领域建模门槛。

### 1.2 用户画像

| 用户类型 | 特征 | 痛点 |
|----------|------|------|
| 产品经理 | 熟悉业务，不懂技术 | 难以清晰表达系统边界 |
| 开发工程师 | 懂技术，业务理解浅 | 领域模型设计困难 |
| 架构师 | 需要快速验证想法 | 缺乏高效工具 |
| 初创团队 | 资源有限 | 没有专人做设计 |

### 1.3 首页定位

首页是 VibeX 的**主入口**，承担以下职责：
1. **需求输入**: 用户输入业务需求
2. **即时反馈**: 实时展示 AI 分析结果
3. **快速操作**: 一站式完成从需求到项目创建

## 二、用户故事与使用场景

### 2.1 核心用户故事

#### US-001: 快速分析需求
```
作为一名产品经理
我希望输入业务需求后立即看到领域模型
以便于快速验证我的想法
```

#### US-002: 探索式设计
```
作为一名架构师
我希望能够随时切换分析步骤
以便探索不同的设计方向
```

#### US-003: 边问边改
```
作为一名开发工程师
我希望 AI 能够追问澄清
以便于完善不完整的需求
```

#### US-004: 快速建项目
```
作为一名用户
我希望将分析结果一键创建为项目
以便后续继续完善
```

### 2.2 典型使用场景

#### 场景1: 快速启动
```
1. 用户打开 VibeX
2. 输入需求："做一个电商系统"
3. 看到 AI 生成的限界上下文
4. 点击创建项目
5. 进入项目管理页继续完善
```

#### 场景2: 深度分析
```
1. 用户打开 VibeX
2. 输入详细需求
3. 展开底部面板
4. 使用 AI 询问功能追问细节
5. 根据 AI 建议优化需求
6. 重新生成分析结果
7. 创建项目
```

#### 场景3: 探索设计
```
1. 用户打开 VibeX
2. 输入需求
3. 看到初步分析结果
4. 切换到"业务流程"步骤
5. 查看流程图
6. 切换到"组件图"步骤
7. 查看组件结构
8. 创建项目
```

### 3.2 布局规格

| 参数 | 值 |
|------|-----|

## 五、数据需求

### 5.1 状态管理 (designStore)

```typescript
interface DesignStore {
  // 布局状态
  leftDrawerOpen: boolean;
  rightDrawerOpen: boolean;
  bottomPanelHeight: number;
  bottomPanelFloating: boolean;
  
  // 步骤状态
  currentStep: 1 | 2 | 3 | 4;
  
  // 分析结果
  boundedContexts: BoundedContext[];
  domainModels: DomainModel[];
  flowCharts: FlowChart[];
  componentTrees: ComponentTree[];
  
  // AI 状态
  thinkingMessages: ThinkingMessage[];
  aiDisplayCards: DisplayCard[];
  
  // SSE 连接
  sseConnected: boolean;
  sseReconnecting: boolean;
}
```

### 5.2 状态持久化

| 存储内容 | 存储方式 | 有效期 |
|----------|----------|--------|
| 分析结果 | localStorage | 7天 |
| 布局状态 | sessionStorage | 会话 |
| 用户偏好 | localStorage | 永久 |

### 5.3 状态快照

```typescript
interface StateSnapshot {
  id: string;
  step: number;
  data: Partial<DesignStore>;
  timestamp: number;
  label?: string;
}
```

每次切换步骤前自动保存，支持回退。

## 六、API 需求

### 6.1 核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/v1/analyze | POST | 提交需求分析 |
| /api/v1/analyze/stream | GET | SSE 流式分析 |
| /api/v1/clarify/stream | GET | SSE 对话澄清 |
| /api/v1/quick-action | POST | 快捷功能 |
| /api/v1/projects | POST | 创建项目 |
| /api/v1/projects | GET | 获取项目列表 |

### 6.2 请求示例

**POST /api/v1/analyze**
```json
{
  "requirement": "做一个电商系统",
  "context": {
    "currentStep": 1
  }
}
```

**响应**
```json
{
  "success": true,
  "data": {
    "boundedContexts": [...],
    "domainModels": [...],
    "flowCharts": [...],
    "componentTrees": [...]
  }
}
```
