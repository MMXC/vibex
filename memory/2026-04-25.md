### Phase2 Detection False Positives (Critical)

The Step 0 "Phase2 missing" detection in the heartbeat is broken by design. It only checks for `dev-epic`, `tester-epic`, `reviewer-epic`, `reviewer-push-epic` patterns, but actual phase2 tasks use at least 3 naming conventions:

1. `dev-epic1` — small "epic" (the pattern only catches this)
2. `dev-E1` — capital "E" (NOT caught)
3. `dev-p001-typescript债务清理` — p00N format (NOT caught)

**Impact**: All "PHASE2-MISSING" results are false positives. When I scanned all completed projects, every single one that appeared as "PHASE2-MISSING" actually had a complete phase2 chain — just with non-epic naming.

**Fix needed**: Update the heartbeat's Phase2 detection to check for all three patterns.

### current-report JSON parsing: `'idle'` key missing

When parsing `current-report --json` output, the resulting dict may not have an `'idle'` key. The `current-report` command returns different keys depending on what it finds. Always check `list(d.keys())` before accessing.

**Safe parsing pattern**:
```python
python3 -c "
import json
with open('/tmp/cr.json') as f:
    content = f.read()
if content.startswith('CALLING MAIN'):
    content = content[len('CALLING MAIN'):]
d = json.loads(content)
# Check available keys
print('Keys:', list(d.keys()))
"
```
