### P001: Canvas API 完整实现

**用户价值**: 91.7% API 端点缺失导致 AI 生成能力无法释放。用户无法完成从 Context → Flow → Components 的完整工作流。

**产品影响**:

**ROI 评估**:

|------|-----|
| 用户影响 | 高 — 核心功能缺失 |
| 开发成本 | 中 — 约 8h（API 实现）+ 2h（集成）|
| 商业价值 | 高 — 差异化竞争力 |

**优先级**: P0

### P002: Zustand 重构 ROI

**用户价值**: Zustand store 与 React 组件耦合导致状态管理混乱，影响：
- 开发体验：组件状态不可预测
- 用户体验：界面闪烁/状态不一致

**ROI 评估**:
| 维度 | 当前 | 重构后 |
|------|------|--------|
| 开发效率 | 低 — 状态调试困难 | 高 — 可预测状态流 |
| 用户体验 | 中 — 偶发闪烁 | 高 — 流畅响应 |
| 迁移成本 | - | 高 — 全局状态改动 |
| 风险 | - | 中 — 可能引入 regression |

**建议**: 
- 优先级 P1（低于 P0 API 修复）
- 渐进式迁移：先拆分，再重构
- 验收标准：状态更新后 100ms 内 UI 响应

**优先级**: P1

### P003: 流程改进优先级排序

基于今日提案，流程改进优先级：

| 优先级 | 改进项 | 用户价值 | 开发效率 | 影响范围 |
|--------|--------|----------|----------|----------|
| P0 | subagent 超时策略 | 无直接影响 | 高 | 全团队 |
| P1 | 提案去重机制 | 无直接影响 | 中 | 决策效率 |
| P2 | Mock/Real 边界可视化 | 无直接影响 | 中 | 开发体验 |

**P0 核心问题**: subagent 5 分钟超时导致已完成工作丢失（2/3 案例）。

**建议**:
1. 增加 commit checkpoint（代码修改后立即 commit）
2. Parent agent 定期检测 subagent 进度
3. 超时后保留 partial work

**ROI**: 
- 开发效率提升 30%（减少重复工作）
- 心理负担降低（工作不会丢失）
