## 2. 提案详情

### P001: 画布性能强化——500+节点虚拟化 + 缩略图导航

**问题描述**：
当前虚拟化支持 100+ 节点，但真实用户项目常见 300-800 节点规模，全览缩放时帧率下降；大规模画布缺乏缩略图导航，用户定位节点效率低。

**影响范围**：
Canvas 核心模块；影响所有中大型项目用户。

**验收标准**：
- 500 节点项目缩放/平移帧率 ≥ 50fps
- 提供缩略图导航面板，支持点击跳转
- Group 节点折叠/展开不影响周围节点布局

---

### P002: AI 评审自动化——从人工触发到持续 Gate

**问题描述**：
当前 AI 设计评审依赖人工触发，评审结果不自动追踪、无法对比历史；Code Generation 产出无质量评分机制。

**影响范围**：
AI 能力模块；影响设计→代码的全流程质量闭环。

**验收标准**：
- PR/Commit 触发自动 AI 评审，结果写入 CI 报告
- 评审结果结构化存储，支持历史对比
- Code Generation 质量评分（可读性/复杂度/覆盖率）

---

### P003: 协作感知增强——冲突可视化 + 操作历史追溯

**问题描述**：
当前 LWW 仲裁解决了数据冲突，但用户看不到冲突发生位置和被覆盖的内容；多人编辑同一区域缺乏操作意图感知。

**影响范围**：
实时协作模块；影响团队协作核心体验。

**验收标准**：
- 冲突发生时高亮显示冲突区域，提示"某用户覆盖了你的修改"
- 操作历史时间线：可回看任意时间点的画布快照
- 协作者意图气泡（如"正在编辑节点A"）

---

### P004: 离线模式增强——冲突离线合并 + 后台同步队列

**问题描述**：
当前 PWA 离线模式可在离线时编辑，但重新上线后冲突处理机制不完善；长时离线后数据合并依赖 LWW，可能丢失关键修改。

**影响范围**：
Canvas 核心 + 基础设施；影响高频断网用户（移动端/出差场景）。

**验收标准**：
- 离线修改重新上线后自动检测冲突，生成冲突diff
- 用户可选择保留本地/远程/手动合并
- 同步队列可视化：显示待同步项数量和同步进度

---

### P005: 测试覆盖率升级——组件级 Snapshot Testing + 自动化视觉回归

**问题描述**：
当前 Playwright E2E 覆盖主流程，组件级别单元测试缺失；Canvas 渲染的视觉变化无法自动检测，依赖人工 Review。

**影响范围**：
质量保障模块；影响前端组件重构安全性。

**验收标准**：
- 核心组件（Canvas/DDS/节点）快照测试覆盖率 ≥ 80%
- Playwright 截图对比集成 CI，视觉diff自动报告
- 94 个 API 模块测试新增组件级 Mock 测试

## 根因分析

### 根因

基于 Sprint 1-31 交付盘点，核心问题集中于三点：

1. **性能基座存在瓶颈**：Sprint 27 虚拟化解决了 100 节点问题，但真实项目规模远超测试数据，Sprint 32 需要面向 500+ 节点场景。
2. **AI 能力闭环未形成**：Sprint 28 AI 辅助需求解析 + Sprint 29 AI 设计评审 + Sprint 20 AI Agent 均为独立能力，缺乏与 CI/CD 的深度集成，用户感知价值分散。
3. **协作体验存在盲区**：Sprint 27-29 的实时协作基础（RTDB/Cursor/Presence）已就绪，但冲突处理和意图感知仍是空白，协作用户体验存在"黑盒"感。

### 证据

- CHANGELOG.md 显示 S27-30 持续交付协作基础设施，但均未涉及冲突可视化
- Sprint 28 E02 已完成 Design Output 性能优化（react-window 虚拟化），但未涉及 Group/Folder 层级抽象
- Sprint 20 Workbench AI Agent 已接入，但无质量评分机制
- Sprint 29 E05 离线模式已上线 PWA，但无冲突合并队列

## 建议方案

### 方案 A（推荐）

**P001 + P002 + P003 组合**：性能基座 + AI 质量闭环 + 协作体验三角支柱

- **P001**：ReactFlow viewport 优化 + 层级抽象 Group/Folder + 缩略图导航面板
- **P002**：AI 评审写入 CI Gate + 评审结果结构化存储 + Code Generation 评分
- **P003**：冲突高亮 + 操作历史时间线 + 协作者意图气泡

**实施成本**：高（涉及 Canvas 核心重构 + CI 集成）
**风险**：中（Group/Folder 变更影响现有节点数据模型）
**回滚计划**：Group 节点标记为可选 feature flag

### 方案 B

**P001 + P004 + P005 组合**：性能基座 + 离线体验 + 质量保障

- **P001**：同方案 A
- **P004**：IndexedDB Change Queue + 冲突diff UI + 同步进度可视化
- **P005**：Vitest Snapshot + Playwright screenshot diff + 覆盖率报告集成 CI

**实施成本**：中（基于现有 PWA 基础设施扩展）
**风险**：低（离线队列为新增模块，不改动核心数据模型）
**回滚计划**：同步队列可禁用回退到 LWW

### 方案 C

**单一 P001 聚焦**：最紧急的 Pain Point

**实施成本**：中
**风险**：低
**回滚计划**：Group 节点可选
