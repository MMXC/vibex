## 3. 🔴 发现的 Bug（必须修复）

### Bug #1: unwrappers.ts 行为变更破坏测试

**文件**: `vibex-fronted/src/services/api/unwrappers.ts`

**问题**: Dev 在修复 TS strict 错误时，改变了 `unwrapField` 函数的返回值行为：

```diff
- return (response as Record<string, T>)[field as string];
+ return (response as Record<string, T>)[field as string] ?? null;

- return response as T;
+ return (response as T) ?? null;
```

**影响**: 当 field 不存在时，函数现在返回 `null` 而非 `undefined`。

**测试失败**:
```
FAIL tests/unit/unwrappers.test.ts > unwrappers > unwrapField > should handle undefined response
AssertionError: expected null to be undefined
  Expected: undefined
  Received: null
```

**根因**: Dev 添加 `?? null` 的目的是消除 TS 错误（`T` 可能为 `undefined`），但这改变了函数的语义，导致已有测试失败。

**修复建议**: 保持 `?? null` 修复 TS 错误，同时更新测试用例：
```ts
it('should handle undefined response', () => {
  const result = unwrapField<string[]>(undefined, 'agents');
  expect(result).toBeNull(); // 修改为 null
});
```
