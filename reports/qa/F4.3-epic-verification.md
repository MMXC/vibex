## 2. 审计验证

F4.3 的核心问题是确认 `inactivePanel` 与 `allConfirmed` 标志一致。审计链路追踪：

### 组件链路
```
CanvasPage
  → FlowTreePanel (passes isActive={flowActive})
    → BusinessFlowTree (receives isActive prop, defaults to true)
      → inactivePanel (renders when isActive === false)
```

### 审计发现
- `inactivePanel` 使用 `isActive` prop（来自 `BusinessFlowTreeProps`），默认值 `true`
- `flowActive` = `activeTree === 'flow'` — 这是 UI tab 可见性，不是完成状态
- `CanvasPage` 确实未传 `allConfirmed` 到 `BusinessFlowTree`
- `inactivePanel` **永不显示**（因为 `BusinessFlowTree` 只在 Flow tab 时渲染，而 Flow tab 时 `isActive=true`）
- Panel lock 功能当前未实际使用
